# ADR-0037: ISO (isolated per-source) + Program (composited-output) recording — faithful, segmented, retained, disk-pressure-gated capture that can never stall the engine

- **Status:** Proposed
- **Area:** Output / Input / Engine / Config / Telemetry / Control / Web
- **Date:** 2026-06-08
- **Source brief:** [iso-program-recording.md](../research/iso-program-recording.md)
- **Builds on:** [ADR-0030](ADR-0030.md) (GP-7 guarded passthrough + GP-6
  `RestampAccumulator`), [ADR-0032](ADR-0032.md) (HLS live segmenter — rolling
  window, atomic publish, definable locations), [ADR-0026](ADR-0026.md)
  (encode-once-mux-many), [ADR-0035](ADR-0035.md) (self-aware placement —
  the `HealthWarning` model + `GET /api/v1/health` + `HealthBanner`),
  [ADR-R003](ADR-R003.md) (`backon` exponential-with-jitter back-off),
  [ADR-R009](ADR-R009.md) (`/livez`/`/readyz` semantics — a full disk must not
  restart-loop the container).
- **Relates to:** invariant #1 (output clock never stalls), invariant #7
  (encode-once-mux-many), invariant #3 (timing), invariant #10 (isolation — a
  best-effort consumer cannot back-pressure the engine).

> **Terminology (load-bearing — see the brief §0).** **ISO = Isolated**: a continuous,
> independent recording of a single input, captured regardless of on-air state (the
> *"raw input to disk"* half). It is **not** ISO-639 language codes nor ISO-BMFF/`mp4`.
> **Program / PGM**: a recording of the composited output (the *"raw output to disk"*
> half). **Clean** (overlay-free) vs **Dirty** (graphics burned in) — v1 records the
> **dirty** program; clean is a future axis.

> **ADR number is fixed at 0037.** ADR-0036 is reserved for a concurrent
> codec-capabilities design; do not renumber.

---

## Context

The operator requires faithful capture of every ingested input ("raw input to disk", all
elementary streams) **and** of the composited output ("raw output to disk"), with rotation
(keep 30 min / 1 h / 7 days …), a disk-availability policy (stop at e.g. 80 % full), and a
hard rule: it must **warn + back off**, never crash or stall — *"a failing write must not
take out core functionality."*

This maps onto the standard production-truck model: an **ISO** recording per source
(continuous, independent of on-air state) and a **Program** recording per output/rendition,
both **segmented + retained + disk-pressure-gated**. The cardinal architectural constraint is
that a recorder is a **best-effort sink that is physically incapable of back-pressuring the
engine or the live transports** (invariants #1 and #10). A slow/full/unmounted/erroring disk
must make the recorder **drop (bounded queue) → warn → back off → auto-resume**, never block
the data plane, never unbounded-buffer into RAM, never panic.

The codebase already supplies most of the pieces, and an adversarial review pinned down
exactly which "it already does X" assumptions are true and which are net-new (full table in
the brief §3). Load-bearing facts:

- **All-stream read exists, but ingest is video-only.** `Demuxer::read_packet()` returns a
  `ReadPacket` for any stream; `Demuxer::inventory()` enumerates every elementary stream
  (`StreamInventory`). But `FileSource` reads `read_packet_for(video_index)` and
  `read_packet_for` **silently discards** all other packets — and a libav demuxer is
  `Send + !Sync` and **read-once**, so a second demuxer is not viable for live single-
  connection transports.
- **The copy primitive is real; the ISO wiring is not.** `Muxer::add_stream_from_parameters`
  does `avcodec_parameters_copy` onto an encoder-less stream (genuine stream-copy, no
  re-encode), but the only `StreamCodecParameters` constructors snapshot an **opened
  encoder**, and `StreamInventory` carries descriptors, not `AVCodecParameters`. A
  `from_parameters` constructor + a per-inventory registration loop are net-new.
- **The Program file sink exists as a single file, not rotating.** `PacketMuxSink` holds no
  encoder; `maybe_prepend_program_ts` already feeds a `PacketMuxSink::file("program.ts")` the
  **same** single encode (inv #7). But its fan-out coupling is `send_bounded` (blocks the
  bake-consumer up to ~2 s, then marks the sink dead) — **not** drop-oldest auto-resuming.
- **The keyframe-rotated segmenter exists; prune is count-only and segments aren't fsync'd.**
  `segment_live` rotates on a video keyframe; `close_current` does `<seg>.tmp → rename`;
  `LivePlaylist` evicts beyond a count window and fsyncs the **manifest**. The **segment media
  file is not fsync'd** (`finish` → `write_trailer` is an AVIO flush, not `fsync`).
- **Disk sensing does not exist** (`rg statvfs|f_bavail|free_space crates/` → 0;
  `availability.rs` is uptime-ratio).
- **The health-warning model is fully implemented** (`HealthWarning`/`WarningCode`/store/
  ingest/`GET /api/v1/health`/`HealthBanner`); `WarningCode` is `#[non_exhaustive]` with one
  variant today.
- **`backon` is the mandated back-off crate** (`backoff` is RUSTSEC-2025-0012) and is **not
  yet a workspace dep**.

## Decision

Ship **ISO** and **Program** recording as best-effort, bounded, isolated sinks, reusing the
existing segmenter/muxer/health machinery and building only the named gaps.

1. **The bulletproof write path comes first.** Every recorder couples to the engine **only**
   through a **bounded drop-oldest ring + wait-free `try_send`**, drained by a **dedicated
   off-hot-path writer task**. Built on the genuinely non-blocking `BoundedPacketQueue`
   try-push pattern — **never** the 2 s-blocking `send_bounded`. A per-recorder **state
   machine** `Disarmed → Recording → Paused(DiskPressure|WriteError|Manual) → Recording`,
   with **`backon` capped-exponential + full-jitter** back-off (1 s → 30 s) on
   `ENOSPC`/`EIO`/`ENODEV`/`EROFS`, **draining + dropping the ring while backing off**, and a
   **reopen-and-probe-write auto-resume**. One ring per recorder, depth **and** byte cap,
   allocated once at arm; **DROP-OLDEST** for Program, **DROP-FORWARD-TO-NEXT-IDR** for ISO.
   Muxers opened with a bounded `max_interleave_delta` so libav's interleaver cannot grow
   RAM outside the ring's accounting. A **held-out chaos gate** proves zero output-clock gap
   and bounded RSS while a disk goes full/unmounts/throws `EIO`.

2. **Program recording = a rotating file sink in the encode-once fan-out (inv #7).**
   Generalize the `maybe_prepend_program_ts` anchor into a `RunnableOutput::ProgramRecording`
   carrying a `PacketMuxSink::segment_live`, fed the **same** single encode (per-sink
   `av_packet_ref` copy — not a second encode), behind the recorder's **own** ring. v1 records
   the **dirty** program at zero extra cost; **clean** is a deferred separate encode under a
   distinct `RenditionId`.

3. **ISO recording = faithful copy-remux of ALL elementary streams.** Replace
   `read_packet_for(video_index)` with a **packet-router on the single input read thread**:
   route the video AU to the existing decode path **unchanged** (the unblocked primary), and
   `av_packet_ref`-clone **every** packet into the ISO writer's ring. A new
   `StreamCodecParameters::from_parameters` + a dedicated `IsoMuxSink` (keyed by source
   `stream_index`, one output stream per `StreamInventory` row) stream-copy every stream into
   a rotating `mkv`/`ts` container. **ISO preserves original input PTS/DTS** in each stream's
   own timebase (inv #3 — `f(tick)` is rejected for the copy path); a **per-ISO-stream
   `RestampAccumulator`** is the `av_interleaved_write_frame` abort-guard across segment
   seams. Rotate on a configured video stream's **IDR**. **Byte-exact on-wire passthrough is
   deferred** (`IsoContainer::ByteExact`, Ts/Srt only) — it needs a raw-AVIO tee that does not
   exist and is wrong for RTMP/HLS/RTSP.

4. **Segmentation + retention reuse the ADR-0032 segmenter**, extended to **dual-axis
   (time AND size)** retention (prune while *any* of time / size / count bounds hold; ≥ 1
   bound required), with an index (`MediaPlaylist` for `.ts`, JSON manifest for `mkv`). Segment
   durability is stated honestly; optionally `sync_all` the segment before rename (on the
   writer task) to close the crash-truncation window.

5. **Disk-pressure sensing** is a new `statvfs(2)` sensor in a dedicated `*sys` crate
   (precedent: `multiview-ntpsys`/`multiview-i915pmu`), consumed by `multiview-telemetry`
   (which stays `forbid(unsafe_code)`). Free is computed against **`f_bavail`** (df-honest).
   Checked at each **segment OPEN** on the writer task — never the output-clock thread. A
   sensor error **fails safe → disarm**. Disarm below `stop_at_pct`/`min_free`; auto-resume
   above a hysteresis band (reuse `degradation::Hysteresis`). Disk pressure overrides
   retention.

6. **Config** is additive (`iso: Option<IsoRecording>` on `Source`, `program:
   Option<ProgramRecording>` per `Output`); validation rejects `ByteExact` on
   non-byte-stream transports, requires `stop_at_pct ∈ 1..=99`, `duration_seconds > 0`, and
   ≥ 1 retention bound when armed.

7. **API/realtime/UI**: arm/disarm ops (`POST …/recording/arm|disarm`, **Class-1 hot**,
   `202` + op-id, shed-to-`503`) + off-engine reads (`GET /recordings`, `…/segments`,
   `/recordings/disk-status`); a re-snapshotable `Event::RecordingStatus`; **four additive
   `WarningCode` variants** (`iso/program-disk-pressure`, `recording-write-failing`,
   `recording-disarmed`, `recording-dropping`) on the **already-implemented** health-warning
   model (do **not** interim-ship as `Alert`); a `RecordingsPage` + the existing
   `HealthBanner`.

## Consequences

**Positive.**
- Faithful per-source ISO + composited Program capture with rotation, dual-axis retention, and
  disk-pressure gating — the operator's full ask in standard broadcast vocabulary.
- The recorder is **structurally isolated** (bounded drop-oldest ring + wait-free `try_send`,
  engine never awaits), proven by a chaos/RAM acceptance gate — *ISO = isolated* in the
  literal invariant-#10 sense.
- Program is **zero extra encode** (inv #7); ISO is **zero re-encode** (`avcodec_parameters_
  copy`). Maximum reuse of the ADR-0032 segmenter and the ADR-0035 health surface.

**Negative / costs.**
- Net-new code with real footguns: `StreamCodecParameters::from_parameters`, the
  `IsoMuxSink`, the dual-axis prune, the `statvfs` `*sys` crate, the bounded ring + back-off
  state machine, and a `backon` workspace dep (feature-gated, `cargo deny`-clean,
  `cargo deny check` in the same PR).
- The RAM ceiling is **ring byte-cap + (`max_interleave_delta` × max_pkt_bytes ×
  stream_count)**, not the bare `depth × max_pkt_bytes`; multi-stream ISO interleave buffering
  is new territory and must be bounded + tested.
- v1 is copy-remux (not byte-exact): re-containers, loses TS null packets / PID assignments /
  exact PCR/SCTE-35 byte placement. The SCTE-35 *stream* is copied; bit-exact splice forensics
  await `ByteExact`.
- Without the optional segment `sync_all`, a crash before writeback can leave a torn segment
  under its final name (only the manifest is durable today) — documented honestly.

**Neutral.**
- Arm/disarm is **Class-1 hot** (flips a best-effort sink's state — safer than `Source.enabled`).
- Warnings surface at `GET /api/v1/health`, **not** `/livez`/`/readyz` (ADR-R009 — a full disk
  must not restart-loop the container).

## Alternatives considered

1. **Byte-exact on-wire passthrough in v1 (raw 188-byte TS/SRT tee).** *Rejected for v1.*
   Needs a new raw-AVIO/socket write tee that does not exist (only `avio_fetch.rs`, read-only
   text), and is impossible for RTMP/HLS/RTSP where on-wire bytes are not 1:1 recoverable.
   Deferred behind `IsoContainer::ByteExact` (Ts/Srt only, validated).

2. **A second independent demuxer for ISO.** *Rejected.* Live single-connection transports
   (SRT/RTSP/RTMP) cannot be opened twice, and a second demuxer breaks the
   single-read-thread / `!Sync` posture. ISO must fork at the single `read_packet()` call.

3. **Re-stamp ISO from the output tick counter (`out_pts = f(tick)`).** *Rejected.* Collapses
   B-frame DTS reorder on a copy path (ADR-0030, streaming-gotchas). ISO is an as-is archive:
   preserve source PTS/DTS; use `RestampAccumulator` only as the monotonic abort-guard.

4. **A second encode for the Program recording.** *Rejected* (violates inv #7). Program is a
   mux-only fan-out sink fed the existing single encode. (A *clean* feed is the lone
   inv-#7-sanctioned separate encode — pixels differ — and is deferred.)

5. **Reuse the `program.ts` `send_bounded`/block-then-mark-dead coupling for the rotating
   recorder.** *Rejected.* It paces the bake-consumer for the grace window on every frame and
   **cannot auto-resume** (it permanently marks the sink dead). The recorder must interpose
   its own bounded drop-oldest ring + writer task that backs off and auto-resumes.

6. **Unbounded buffer / block on a slow disk; or panic on write error.** *Rejected* —
   violates invariants #1/#10 and the never-panic rule. The ring drops, the writer warns +
   backs off, the state machine pauses; the engine never notices.

7. **Count-window retention only (FFmpeg `hls_list_size`).** *Rejected* — the operator wants
   time *and* size retention; extend to dual-axis (strict superset). Disk pressure overrides
   retention.

8. **Put the `statvfs` `unsafe` directly in `multiview-telemetry`.** *Rejected* — that crate
   is `forbid(unsafe_code)`. The syscall lives in a dedicated `*sys` crate; telemetry consumes
   its safe API.

9. **Interim-ship the recorder warnings as `Alert` with a dedupe key.** *Rejected* — the
   `HealthWarning` model is already implemented and tested; `Alert` lacks the
   `code`/`subsystem`/`remediation` fields operators need. Just add four `#[non_exhaustive]`
   `WarningCode` variants.

10. **Use the `backoff` crate.** *Rejected* — RUSTSEC-2025-0012 (unmaintained). Use `backon`
    (mandated by the resilience brief), feature-gated and `cargo deny`-clean.
