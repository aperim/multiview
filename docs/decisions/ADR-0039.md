# ADR-0039 — SRT (Secure Reliable Transport) input + output: tiered transport seam

**Status:** Proposed (2026-06-08)
**Area:** streaming/timing (T) — contribution/distribution transport (implementation)
**Drivers:** the SRT-as-a-first-class-transport requirement (SRT is named in CLAUDE.md §1 as
an ingest **and** egress protocol) and the LGPL-clean-default + open-standards mandate
(CLAUDE.md §7, [conventions §7](../architecture/conventions.md)).
**Research:** [srt-transport](../research/srt-transport.md).
**Related:** [ADR-0031](ADR-0031.md) (our-built LGPL-clean FFmpeg already ships
`--enable-libsrt`), [ADR-0028](ADR-0028.md) (own-the-FFI-binding pattern — the Tier-2 model),
[ADR-0034](ADR-0034.md) (decoupled routing / `StreamInventory` — stream-id routing target),
[ADR-0035](ADR-0035.md) (capability detection + `HealthWarning` catalog — the SRT-stats health
surface), [ADR-T003](ADR-T003.md) (unified timing — never float, re-stamp from tick),
[ADR-R001](ADR-R001.md)/[ADR-R003](ADR-R003.md) (continuous-output guarantee + supervised
reconnect). **SRT here = the network transport (`libsrt`), not SubRip subtitles.**

## Context

SRT bytes **already flow** through FFmpeg's `libsrt` protocol: ingest `SourceKind::Srt { url }`
(`schema.rs:269`) opened as a `srt://` AVIO URL (`pipeline.rs:3545` → `demux.rs:191`), and
egress `Output::Srt` (`schema.rs:635`) → `PushProtocol::Srt` → muxer `mpegts` (`sink.rs:1039`)
→ `PacketMuxSink::push` (`sink.rs:1233`), fanned by the tolerant `run_push_output`
(`pipeline.rs:2307`). A fully-built **but orphaned** typed connection model already exists —
`SrtMode`/`KeyLength`/`StreamId`/`SrtConfig` with `validate`/`to_url`/`to_url_redacted` in
`crates/multiview-input/src/srt.rs` (`pub mod srt` at `lib.rs:110`, tested by
`tests/srt_config.rs`) — but **nothing on the run path references it** (verified).

So this is a wiring + tiering problem, not a greenfield build. The current passthrough is
**blind and single-mode**: no typed config on the run path, **caller-only** egress, **no link
statistics** (verified: `libavformat`'s `libsrt` protocol never calls `srt_bstats`/`srt_bistats`),
no listener multiplexing (verified: FFmpeg's listener is `srt_listen(fd, 1)` + one
`srt_accept`), no stream-id Access-Control routing, and no bonding. Licensing is **already
settled**: libsrt is **MPL-2.0** (weak, file-level copyleft) in FFmpeg's `EXTERNAL_LIBRARY_LIST`
(verified — `--enable-libsrt` forces neither `--enable-gpl` nor `--enable-nonfree`), and
`MPL-2.0` is already on the `deny.toml` allow-list, so the default build stays LGPL-clean.

Two latent defects in the orphaned model were verified and must be fixed while wiring: the
passphrase bound is `10..=79` (`srt.rs:228`) but libsrt accepts **`10..=80`** inclusive
(`HAICRYPT_SECRET_MAX_SZ = 80`); and `to_url` emits `latency=<ms>` (`srt.rs:266`) but FFmpeg's
`libsrt` AVOptions are in **microseconds** — a latent **1000×** bug (libsrt's native API is ms;
the unit flips only inside `libavformat`).

## Decision

1. **Tier-0 = FFmpeg `srt://` is the default byte path** for **caller-mode** SRT ingest and
   egress, under the **existing `ffmpeg` feature** — **not** a license-escalating flag. It
   already works end-to-end (`pipeline.rs:3545` ingest, `sink.rs:1039` egress muxer `mpegts`),
   is MPL-2.0 license-clean (libsrt in FFmpeg's `EXTERNAL_LIBRARY_LIST` — verified), and is the
   widest-interop **conformance oracle**. Respects the LGPL-clean-default discipline
   (CLAUDE.md §7) and encode-once-mux-many (inv #7 — the SRT sink is mux-only, no encoder).

2. **Wire the orphaned `SrtConfig` into the schema as a backward-compatible typed extension.**
   Add an optional `srt: Option<SrtOptions>` to **both** `SourceKind::Srt` and `Output::Srt`,
   **mirroring `Rtsp`'s `Option<RtspOptions>`** (`schema.rs:245`),
   `#[serde(default, skip_serializing_if = "Option::is_none")]`, internally-tagged unions,
   **never `untagged`**. `SrtOptions` is the serializable projection of `SrtConfig`
   (`mode`, `latency_ms`, `encryption {key_length, passphrase_ref}`, `stream_id`, `listener`,
   `oheadbw_pct`, `maxbw`, timeouts, `backend`). When `srt` is absent, behaviour is
   byte-identical to today's `{ url }` passthrough — **no schema version bump** (purely
   additive). Respects the typing/serde conventions (§5) and keeps existing configs valid.

3. **Carry latency as typed `latency_ms: u32` and own the unit conversion in the seam:**
   `×1000` to microseconds at the FFmpeg/`libsrt` boundary (fixing the `srt.rs:266` ms bug),
   milliseconds native to srt-tokio. Internal time is i64 ns / exact rationals, **never float**
   (inv #3). Telemetry surfaces the **negotiated** effective latency
   (`max(local RCVLATENCY, peer PEERLATENCY)` per direction, since SRT 1.3), not just the
   requested value.

4. **Treat the SRT TSBPD receive buffer strictly as an input jitter buffer** feeding the
   last-good-frame store — **never** the output cadence. The output clock **samples** the
   SRT-fed tile's latest frame each tick; no data-plane code awaits `srt_recv` or the next
   TSBPD release; **TLPKTDROP stays enabled** (default true) so a too-late packet becomes a gap
   the tile state machine absorbs. Respects inv #1 (output-clock) and inv #2
   (LIVE→STALE→RECONNECTING→NO_SIGNAL, keyed on **freshness, not socket liveness** — a
   connected-but-dropping link must go STALE).

5. **Store the passphrase as a reference only** (`SourceAuth.secret_ref`, `schema.rs:185`) —
   never a plaintext `String` in serde config, never in the SRT URL query — resolved to
   plaintext only at socket setup, zeroized after, redacted from all logs/telemetry/exported
   config (keep + extend `to_url_redacted`'s `***` masking; config-as-code round-trips the
   reference only, mirroring libsrt's write-only `SRTO_PASSPHRASE`). **Widen passphrase
   validation `10..=79` → `10..=80`** to match the libsrt source, documenting the FFmpeg-path
   79 cap when Tier-0 is the backend. Respects the repo secret rule (1Password / gitleaks).

6. **Own SRT statistics behind the seam, because FFmpeg structurally cannot provide them** —
   `libavformat` never calls `srt_bstats`/`srt_bistats` (verified). A detached ~1–2 Hz
   per-link poller publishes a conflated latest-value snapshot via a `watch` channel (Tier-1
   srt-tokio `SocketStatistics`; Tier-2 FFI `srt_bistats(clear=0)` + instantaneous gauges); the
   engine/compositor never read it and never await it. Map `…Total` fields to Prometheus
   counters (let `rate()` difference them — never clear), live buffer/RTT to gauges,
   **role-masked** (ingest hides sender fields). Do **not** gate admission/degradation (inv #9)
   on the noisy `mbpsBandwidth` packet-pair estimate — use send/recv rate + loss % + drops +
   RTT. Respects inv #10 (isolation) and feeds inv #9 + inv #2.

7. **Surface SRT link degradation as health warnings, not A/V alarms.** Extend the
   [ADR-0035](ADR-0035.md) `WarningCode` catalog (`event.rs:284`) with
   `srt-link-packet-loss-sustained`, `srt-link-latency-spike`, `srt-connection-unstable`,
   `srt-rtt-ceiling-hit`, emitted as `Event::HealthWarningRaised`/`Cleared`
   (`event.rs:573`/`577`) on `Topic::Alerts` via the **same drop-oldest publisher** as
   `SystemMetrics`, with **hysteresis** (latch + dwell, no flapping). Read-only
   `GET /api/v1/health`; **never `/readyz`** (a degraded link must not restart-loop the
   container). **No new `AlarmKind`** — SRT link degradation is operator-actionable infra
   health, not an A/V *content* alarm. Respects inv #10 + inv #9.

8. **Tier-1 = pure-Rust `srt-tokio` behind a socket-free `SrtTransport` seam** (off-by-default
   `srt` feature) for the two FFmpeg-impossible capabilities — **listener-mode multiplexing**
   (one port → one `Source` per accepted connection, keyed by stream-id) and **statistics**.
   The seam is modelled on the preview WHEP seam (`SessionId` newtype, explicit `SessionState`
   Created→Connecting→Connected→Closed, `SampleFeed`/`SampleSink` bounded drop-oldest). The
   accept/Access-Control decision is an in-memory `O(1)`/`O(log n)` lookup from a
   control-plane-populated `watch` snapshot — **never** blocking on SQLite/the command bus/any
   lock (a slow accept callback is literal ingest back-pressure, inv #10) — rejecting with
   precise `SRT_REJX` codes (1401/1404/1409/1402/1400). srt-tokio is FFI-free
   (`unsafe_code = forbid`-clean), **Apache-2.0** (verified — not dual-MIT), but "NOT PRODUCTION
   READY"; it is therefore gated behind the feature and **validated per-endpoint against the
   FFmpeg/libsrt baseline** before production trust. Respects inv #10 and inv #11
   (route/passphrase changes classified Class-1/Class-2).

9. **Parse the stream-id Access-Control string as a typed grammar** (prefix `#!::`, reserved
   keys `u`/`r`/`h`/`s`/`t`/`m` with `m=request` default, namespaced `user_*`/`multiview_*`
   extensions) with `TryFrom` + exhaustive `match` in `multiview-config`/`-input` — never
   string-munging — and integrate routing with [ADR-0034](ADR-0034.md)'s
   `RoutingTable`/`StreamInventory` (a `StableStreamId` per accepted stream-id). Reject
   unparseable stream-ids with `SRT_REJX_BAD_REQUEST` (1400); apply explicit, live-
   reconfigurable duplicate (1409) and capacity (1402) policies. Respects the absolute-typing
   pillar and inv #11.

10. **Tier-2 = an owned `multiview-srt-sys` libsrt FFI leaf is DEFERRED**, built **only** when a
    hard requirement appears: connection **bonding** (broadcast/backup — only in libsrt ≥ 1.5
    with `ENABLE_BONDING`, absent from FFmpeg **and** srt-tokio — verified), HSv4/legacy-peer
    interop, or proven-at-scale throughput. It would follow [ADR-0028](ADR-0028.md) exactly:
    runtime-probed, **all `unsafe` confined to the leaf** so `multiview-input`/`-output` stay
    `forbid(unsafe_code)`, off-by-default `srt-libsrt` feature, crypto backend
    `USE_ENCLIB=openssl-evp` **or** `mbedtls` (both Apache-2.0). The supervised-reconnect ladder
    (inv #2) extends down into the group (libsrt does not auto-restore members). Bonded send is
    wrapped in a bounded queue the engine never awaits (inv #1/#7/#10). **RIST (VSF TR-06) is
    out of scope** this cycle (reopened only on a customer ST 2022-7-hardware requirement).

11. **An SRT chaos gate is a blocking CI test** proving inv #1/#10 for **both** directions:
    drop/lag/loss-inject an SRT ingest **and** an SRT egress peer (a killable loopback
    co-process + a lossy UDP relay — no special hardware) while asserting the output clock emits
    exactly one frame per tick throughout, the tile rides the failure ladder, no engine queue
    grows, and a wedged SRT egress sink is **detached at `SINK_WEDGE_GRACE`** (`pipeline.rs:138`)
    rather than awaited. Reuses the existing tolerant `run_push_output` (`pipeline.rs:2307`) and
    `consumer_main` `fan_packets`/`send_bounded` path.

## Consequences

- SRT becomes a **first-class, typed, observable** transport for ingest and egress in every
  connection mode, with **no license escalation** (Tier-0/1 stay LGPL-clean,
  `unsafe_code = forbid`) and the existing `{ url }` configs unchanged.
- The default build gains nothing new to compile: Tier-0 is already there; the typed schema +
  validation is pure Rust. Tier-1 (`srt`) adds the `srt-tokio` dep only when enabled.
- **FFmpeg remains the conformance oracle.** Any non-FFmpeg tier is validated against it; the
  "NOT PRODUCTION READY" srt-tokio risk is bounded by keeping Tier-0 as the baseline and gating
  Tier-1 behind real per-endpoint interop tests.
- Statistics and listener multiplexing — the two things operators most need from SRT and the
  two things FFmpeg structurally cannot give — are delivered by Tier-1 without touching the
  engine's isolation guarantees.
- **Bonding is honestly deferred** (it requires the Tier-2 FFI leaf, which is build-heavy and
  trigger-gated). The deferral is revisitable; the brief states the trigger.
- Two verified defects in the orphaned model (passphrase `10..=79` off-by-one; latency ms-as-µs
  1000× bug) are fixed as part of SRT-0/SRT-1, with property tests guarding both boundaries.
- The implementation backlog (`SRT-0..SRT-12`) lives in
  [docs/development/srt-transport-backlog.md](../development/srt-transport-backlog.md).

## Alternatives rejected

- **Ship the owned `multiview-srt-sys` FFI binding now as the single SRT path.** Rejected: it
  pulls cmake + a C++ toolchain + libclang/bindgen + an OpenSSL/mbedTLS crypto backend at build
  time, complicating cross-compile/static-link/CI; the only existing FFI crates (`libsrt-sys`
  1.4.13, `srt-rs` 0.2.3) are ~6 years stale and pre-bonding; and FFmpeg already delivers
  license-clean caller in/out today. Speculative debt with no triggering requirement. Deferred
  to Tier-2.
- **Adopt pure-Rust srt-tokio as the *sole* SRT backend, replacing FFmpeg `srt://`.** Rejected:
  its README says "NOT PRODUCTION READY", it last shipped 2024-05 with frequent breaking
  changes, it does not support SRT < 1.3.0 (HSv4), its keysize-mismatch negotiation is
  unfinished, the reference `srt-c-unittests` "do not pass yet", and there are no published
  interop/perf benchmarks. Betting the only SRT path on it would risk silent interop failures
  against real encoders/decoders. Kept as Tier-1 behind a feature, baseline-validated.
- **Express the new SRT options inside the URL query string** (`srt://host:port?mode=…&
  passphrase=…`), avoiding any schema change. Rejected: it cannot type-check/validate at
  admission (mode/encryption/latency/stream-id become opaque strings), it leaks the passphrase
  into a URL that gets logged/traced/exported (violates the secret rule), and SRT's richer
  options don't all fit a query string safely. A typed `SrtOptions` is required; the URL is
  assembled **from** it (as `SrtConfig::to_url` already does) with the secret redacted.
- **Make `srt` a license-escalating feature behind a copyleft gate** (like `gpl-codecs`/`ndi`).
  Rejected: libsrt is MPL-2.0 (file-level copyleft, combinable, no binary taint) and FFmpeg
  classifies it as an ordinary LGPL-compatible external library (verified). Adding SRT does
  **not** escalate the license; the Tier-0 byte path correctly lives under the ordinary
  `ffmpeg` feature. Only the optional native FFI leaf gets its own off-by-default flag — for
  build-burden reasons, not licensing.
- **Drive resource-adaptive degradation (inv #9) and tile state (inv #2) from the SRT
  `mbpsBandwidth` estimate.** Rejected: it is a receiver-side packet-pair estimate (every 16th
  packet), documented noisy/unreliable on mobile/asymmetric links — advisory only. Use measured
  send/recv rate + loss % + drop/belated counts + RTT with hysteresis.
- **Build connection bonding on the FFmpeg or srt-tokio path.** Rejected as impossible:
  FFmpeg's `libsrt.c` has no group API (verified — single-socket) and srt-tokio has no socket
  groups. Bonding exists only in libsrt ≥ 1.5 with `ENABLE_BONDING` — it strictly requires the
  Tier-2 owned FFI leaf, confirming it must be deferred, not attempted on the available
  backends.
- **Add SRT-specific `AlarmKind` variants** (e.g. `SrtLinkDown`) to `multiview-core`. Rejected:
  SRT link degradation is operator-actionable infrastructure health, not an A/V content alarm
  (Black/Freeze/Silence). It belongs in the [ADR-0035](ADR-0035.md) `HealthWarning`/
  `WarningCode` catalog (richer: code/subsystem/remediation), reusing the existing drop-oldest
  publisher.
- **Implement RIST (VSF TR-06) in parallel** as the redundancy/contribution transport. Rejected
  for this cycle: the codebase/bibliography already commit to SRT, SRT 1.5 bonding covers RIST's
  main multi-path differentiator, and a second transport doubles the surface with no current
  requirement. Listed as a future standards-track option only.
