# ADR-0016: Efficient overlay rendering (GPU glyph-atlas text + libass subtitles + cached/dirty-region compositing)

- **Status:** Proposed
- **Area:** Media / Compositor · Overlays
- **Date:** 2026-06-03
- **Source brief:** [media/overlay-rendering.md](../media/overlay-rendering.md)
- **Relates to:** [ADR-R008](ADR-R008.md) (overlay layer stack + portable atlas/quad contract — this ADR makes its renderer concrete), [ADR-R007](ADR-R007.md) (subtitle ingest → libass burn-in off the hot path), [ADR-E006](ADR-E006.md) (dirty-region recompositing precedent), [ADR-C003](ADR-C003.md) (composite in linear light, premultiplied alpha), [ADR-E002](ADR-E002.md) (NV12-throughout), [ADR-E005](ADR-E005.md) (bounded frame pools), [ADR-E009](ADR-E009.md) (per-tier efficiency budgets in CI), [ADR-R006](ADR-R006.md) (R128 metering cadence), [ADR-I002](ADR-I002.md) (wgpu compositor behind a feature, SSIM/PSNR-gated)

## Decision

Render Multiview overlays through **three paths that all converge on one premultiplied source-over blend in
the engine's `Rgba16Float` linear-light canvas** (invariant #8), placed **between the existing tile
composite pass and the NV12 encode pass** in `multiview-compositor` — never through a third-party renderer's
own render pass (which would blend in the framebuffer's sRGB/8-bit color space and bypass invariants #5/#8):

1. **Text** (labels, UMD, digital clocks/timecode, numeric meter/scope readouts) via a **raster glyph
   atlas** — `cosmic-text` + `swash` + `etagere` (the `glyphon` stack) used for **CPU shaping +
   rasterization + atlas management only**, with the cached glyph quads drawn through Multiview's own
   [`DrawList`/`DrawQuad`](../../crates/multiview-overlay/src/resolve.rs) contract (ADR-R008). The **same**
   `swash` bitmaps feed the GPU atlas upload **and** the CPU reference blit — one text engine, two
   consumers, no metric divergence.
2. **Vector primitives** (audio meter bars / peak-hold, goniometer + correlation, confidence scopes,
   safe-area + center-cross markers, tally borders, IDENTIFY flash, alert-card rounded-rect chrome) as
   **uniform-driven quads / lines / points evaluated analytically in-shader** — "meters are geometry, not
   pictures" — with a hand-written **SDF** reserved only for rounded chrome. No per-frame bitmap.
3. **Subtitle burn-in** off the hot path: **ASS/SSA → `libass`** (behind the off-by-default `libass`
   feature, ADR-R007) and **SRT/VTT → the pure-Rust `cosmic-text` path**, both rasterized on a dedicated
   worker into a lock-free latest-rendered-overlay slot the compositor samples (the synchronous
   libavfilter `subtitles`/`ass` filter is never in the live path).

**Efficiency is enforced by construction:** raster atlas (not SDF/MSDF) because overlays render at a known
fixed canvas size (invariant #6); **per-glyph caching** so a clock tick re-rasterizes nothing already
resident; **change-detection** gating all `set_text()`/re-`prepare()`/upload on a content
revision/hash (never per frame); **dirty-region sub-texture uploads** into a **persistent, byte-capped,
LRU** atlas (never full-canvas re-upload); **uniform-driven primitives** that upload only a few floats;
**meter/scope sample rates conflated to ~10–30 Hz at the source** (invariant #10, ADR-R006); **zero
per-frame heap allocation** and no `unwrap`/`panic` on the path; and **batched** draws (one gather +
one sub-pass per frame, cost ≈ constant in N tiles). Overlays are **input-decoupled** — rendered purely
from local state + the media clock — and critical assets (SIGNAL LOST card, slate) are **atlas-resident
at startup** so they draw with zero upload at the instant of failure. Color: overlays are linearized to the
canvas working space (sRGB → BT.2020 PQ/HLG when the program is HDR) and premultiplied before blending;
output tagging is unchanged and ffprobe-verified. A bundled **OFL-licensed font** (e.g. Noto Sans / Inter +
a monospaced timecode face) guarantees deterministic metrics and the failure-path glyphs with no host-font
dependency. The default and `wgpu` builds stay **LGPL-clean with no native C dependency**; `libass` is the
only native overlay dependency and is opt-in.

## Rationale

- **One linear blend path, in our compositor.** `glyphon`'s `prepare()/render()` targets a wgpu render
  pass blending in the framebuffer color space (`Rgba8UnormSrgb`/`Rgba8Unorm`); Multiview composites tiles in
  **compute** passes into an `Rgba16Float` **linear** canvas, then encodes NV12. Verified against the code:
  [`blend::over`](../../crates/multiview-compositor/src/blend.rs) is premultiplied source-over in linear light
  and [`resolve.rs`](../../crates/multiview-overlay/src/resolve.rs) already defines the portable
  atlas+quad contract. Using `cosmic-text`/`swash`/`etagere` for shaping/raster/atlas only — and drawing the
  quads ourselves — keeps **one** blend path for tiles and overlays and honors invariants #5/#8.
- **Raster atlas beats SDF/MSDF here.** Per [Red Blob Games](https://www.redblobgames.com/articles/sdf-fonts/),
  a raster atlas wins on fast preprocessing, hinting, color/emoji and "all characters"; SDF/MSDF only pay
  off for arbitrary scaling/rotation and same-glyph effects, at the cost of offline generation, SDF
  rounded-corner artifacts, and ~3× MSDF memory. Multiview renders at a fixed known pixel size (invariant #6),
  so scale-invariance is unused.
- **Caching is the lever, and it is automatic + per-glyph.** `cosmic-text`'s `SwashCache` keys rasterized
  bitmaps on `(font, glyph, size, subpixel-bin)`; `etagere` inserts each unique glyph once with LRU
  eviction. A clock `12:34:56 → 12:34:57` rasterizes the 10 digits + `:` once total; static labels upload
  nothing. Shaping is likewise cached per `Buffer`/run (`shape-run-cache`).
- **Change-detection is mandatory.** [mpv #7615](https://github.com/mpv-player/mpv/issues/7615) is the
  cautionary tale: updating an OSD overlay every frame with unchanged content drove the GPU to 60–100%.
  Re-work must be gated on a content revision/hash.
- **Bandwidth is the wall.** A full 1080p RGBA layer is ~249 MB/s @30 and competes with decode/encode
  ([efficiency §2.2](../research/efficiency.md)); dirty-region sub-texture uploads + a persistent atlas +
  uniform-driven primitives are load-bearing, mirroring ADR-E006's tile dirty-region logic.
- **Alpha correctness is load-bearing.** `swash`/`cosmic-text` output **straight coverage** → must be
  premultiplied (`rgb*coverage, coverage`) before upload (as `TextStyle.color` already documents); `libass`
  output is **already premultiplied** → must not be premultiplied again. A mismatch halos every antialiased
  edge — a silent, pervasive bug.
- **Version-compatible today (verified 2026-06).** `glyphon` 0.11.0 (2026-04-13) requires `wgpu ^29` and
  `cosmic-text ^0.18`; the workspace pins wgpu 29.0.3 and the compositor uses `wgpu = "29"`. The stack is
  all permissive (MIT OR Apache-2.0 + Zlib), so the default build stays LGPL-clean with no native dep.

## Alternatives considered

- **CPU-only software rasterizer for everything** (e.g. `fontdue`/`ab_glyph` + CPU blit) — rejected as the
  *primary* path: lacks complex-script shaping (no HarfBuzz) so it cannot do subtitles/i18n, and burns CPU
  on a continuous-output engine. Kept only as a possible lighter CPU-only build option for Latin-only
  labels; the CPU **reference** path uses the *same* `cosmic-text` engine as the GPU path to avoid divergent
  metrics.
- **Re-rasterize the whole overlay layer / re-upload the whole canvas every frame** — rejected: ~249 MB/s
  @1080p30 starves decode/encode bandwidth (tight on Apple unified memory); the mpv #7615 failure mode.
- **`glyphon`'s built-in `TextRenderer::render()` directly** — rejected: blends text in the framebuffer's
  sRGB/8-bit color space via a render pass, bypassing linear-light blend (invariant #8) and NV12-throughout
  (invariant #5). We take its shaping/raster/atlas pieces and draw the quads ourselves.
- **SDF/MSDF glyph atlas** — rejected for glyphs (fixed render size makes scale-invariance moot; offline
  generation + artifacts + memory cost); retained only for rounded vector chrome.
- **Pango/Cairo (or HarfBuzz + FreeType directly)** — rejected: pulls a native C/GLib stack into the default
  build, defeating the LGPL-clean, no-native-dep posture; `cosmic-text` gives equivalent shaping in pure
  Rust. (`libass` already brings HarfBuzz/FriBidi, but only behind the opt-in subtitle feature.)
- **Meters/scopes as rasterized images** — rejected: they are cheap analytic geometry; rasterizing them
  would add per-frame bitmaps and uploads for no benefit. They are uniform-driven primitives.
- **FFmpeg `drawtext`/`drawbox`/`overlay` filtergraph** — rejected for the live path: disruptive to
  reconfigure live, scales poorly with N tiles, requires libfreetype, and the `ass`/`subtitles` filter is
  synchronous and can stall. Retained only as a degraded software fallback.
- **Pixel-diff change detection on the hot path** — rejected: the readback cost can exceed the savings
  (consistent with ADR-E006); gate on source/state change signals instead.

## Consequences

- **Near-constant per-frame overlay cost**, web-UI/template-drivable, with correct antialiased edges and
  correct HDR color — at the price of disciplined premultiply handling (straight `swash` vs already-premul
  `libass`), atlas growth/eviction management, and a content-revision gate on every dynamic element.
- **One new feature-gated overlay module in `multiview-compositor`** (no new crate, conventions §3): the
  rasterizer + atlas + primitive shaders + the overlay blend sub-pass + the CPU-reference blit. `multiview-overlay`
  stays pure model/layout and keeps emitting the `DrawList`; its `DrawQuad` contract is extended with
  primitive (line/quad/SDF) descriptors and an explicit per-quad "already premultiplied" flag so libass and
  swash bitmaps never get double-premultiplied. The GPU draw rides the existing off-by-default `wgpu`
  feature; SRT/VTT need no native dep; `libass` remains the opt-in native dependency (ADR-R007).
- **A bundled OFL font** is added to the repo/build (licensed separately under OFL 1.1, attributed in
  `LICENSE`/`NOTICE`); `cargo deny` continues to gate the dependency graph and must be re-run after any
  `cosmic-text`/`glyphon` bump (the `rustybuzz → harfrust` shaper swap and the 0.15→0.18 line change deps).
- **Efficiency is regression-gated.** The brief's targets T1–T10 (overlay upload ≤ ~1 MB/s steady-state;
  zero atlas re-rasterization on unchanged frames; zero per-frame allocation; bounded LRU atlas; cost ≈
  constant in N tiles; meter/scope ≤ 30 Hz conflation; CPU==GPU text geometry within SSIM ≥ 0.98 / PSNR
  ≥ 40 dB; atlas-resident critical assets; rasterization off the hot path; no extra full-canvas readback)
  feed the perf-regression CI gate (ADR-E009) and the GPU SSIM/PSNR gate (ADR-I002).
- **Open risks** (full list in the brief §9): compute-vs-render choice for the overlay sub-pass (ping-pong
  copy vs render pipeline — measure on iGPU); subpixel-bin atlas growth under diverse label sizes; HDR
  overlay white anchoring (reuse the 203-nit BT.2408 anchor, ADR-C005); and re-blessing golden frames on any
  `cosmic-text`/`glyphon` version bump.
