# ADR-0096: Licensing live wire (heartbeat client) is gated on Conspect API finalization

- **Status:** Accepted
- **Area:** Management
- **Date:** 2026-06-16
- **Source:** operator (heads-up that the conspect.studio `/v0` licensing reference is about to be finalized, "closing the loop" on end-to-end licensing)

## Context

The runtime licensing/entitlement subsystem ([brief](../research/licensing-runtime.md),
[ADR-0050](ADR-0050.md)/[0051](ADR-0051.md)/[0052](ADR-0052.md)/[0053](ADR-0053.md)) is
**~90% shipped**. The `multiview-licence` crate holds the always-compiled state model
(lease/ladder/fingerprint/verify/challenge/store), the three never-off-air engine seams
**S1/S2/S3** are wired and proven by the CONSPECT-2 chaos gate (PR #94), and the control
routes, mesh plane, and web screens are delivered (CONSPECT-4..11). The engine reads
licence posture as **two wait-free atomics per tick** (`config_locked`, `watermark`) —
never licence logic or a network call on the hot path (invariants #1, #10).

**The only gap is the device→server network client.** `crates/multiview-licence/src/heartbeat.rs`
does not exist and there is no `[features] heartbeat`. The blocker is genuinely external:
the conspect.studio `/v0` device API (current baseline **v0.6.0**, captured) already
specifies the lifecycle — `POST /organisations/{orgId}/{activate,heartbeat,rebind,deactivate}`,
the offline-lease flow, signed `ActivationLease`/`HeartbeatLease` (`serial`/`notAfter`/
`signature`/`signerKeyId`/`leaseBytes`), and the 9-rung `enforcementState` ladder returned
**as data inside a 200, never a control verb** — but it does **not** yet publish:

- **D1 — key-trust:** the pinned-root trust anchor shipped with the build, the
  signer-key **rotation** cadence/overlap, and the chain format by which `signerKeyId`
  resolves to a root (there is no device-facing key endpoint).
- **D2 — PoP request-signing:** the proof-of-possession scheme authenticating a request
  with the registered `devicePublicKey` (what bytes are signed, header/encoding,
  nonce/timestamp/replay binding). v0.6.0 declares **no `securitySchemes` and no
  top-level `security`** at all.
- **D3 — canonical-CBOR lease pre-image + golden vector:** the deterministic encoding the
  device re-serialises to verify the bare-Ed25519 lease signature (field order, map-key
  sort, integer/string encoding), and a known-answer vector to pin a test against.

These map to operator confirmations **O1 (wire protocol) · O2 (Ed25519 key-pinning/rotation)
· O3 (claim-code glyph alphabet) · O4 (default orgId for free self-host)**. Per
[ADR-0050 §De](ADR-0050.md) and [ADR-0051 §6](ADR-0051.md), the confirmations gate **only
the live wire** — everything upstream is buildable and testable against an in-process fake
licence server.

## Decision

The licensing remainder (CONSPECT-3 heartbeat client + CONSPECT-12 live-wire/hardware
validation) lands as **one end-to-end LANE-CONSPECT wave the moment the Conspect API
publishes D1/D2/D3 and the operator confirms O1–O4.** It is **explicitly gated, not stubbed
and not parked** (rule 6): no `todo!()`, no guessed protocol shipped as if done. The wave:

- **Feature flags (default-off):** add `[features] heartbeat` to `crates/multiview-licence/Cargo.toml`
  and `heartbeat = ["multiview-licence/heartbeat"]` to `crates/multiview-cli/Cargo.toml`,
  appended to the `nvidia`/`apple`/`linux-vaapi`/`full` presets **only** (mirrors `mesh-mdns`).
  Default `cargo check --workspace` + `cargo deny` stay a pure, LGPL-clean, network-free shell.
- **New file** `crates/multiview-licence/src/heartbeat.rs` (cfg `heartbeat`): an object-safe
  `trait LicenceServer { async fn heartbeat(&self, req) -> Result<SignedLease, _>; … }`
  implemented by **both** the real HTTP client and an in-process `FakeLicenceServer`; a
  `HeartbeatClient` loop that sleeps to `nextDue`, signs the request (D2), verifies the
  returned signed lease against the **pinned root** (D1/D3), and drives the existing
  `store.install_binding` convergence — the same path the offline-lease install and the
  mesh relay already use.
- **Spawn point:** `spawn_heartbeat` in `multiview-cli/src/licence.rs` (modelled on
  `spawn_mesh_discovery`), called beside `spawn_mesh_discovery` in `main.rs`. The task holds
  **only** `Arc<LeaseStore>` + pinned key + `Arc<dyn LicenceServer>` — never an engine handle,
  channel, or lock the data plane takes (invariant #10). Its sole output is `install_binding`;
  renew/lapse propagates to S1/S2/S3 via the existing `refresh_signal` poll with zero
  engine-side wiring.
- **Tests (TDD):** heartbeat-OFF reports `unlicensed-build` honestly (never fake `active`);
  heartbeat-ON against the fake resets the lease to Active(35d) wait-free; a tampered/forged
  response is rejected (Ed25519 + pinned-root); no raw identifier appears in the payload
  (data-minimisation); and the CONSPECT-2 never-off-air chaos gate is extended to
  SIGKILL/stall/partition the heartbeat task and assert bit-unchanged output on every rung.

When the operator signals the docs are updated, the first step is to **diff the new spec
against the captured v0.6.0 baseline** and run the checklist below; only confirmed answers
are implemented (no guessing).

### Verification checklist — what the updated Conspect `/v0` docs must answer

**Wire (O1):** `POST …/heartbeat` request is exactly `{ bindingId, leaseSerial?, fingerprintDigest(salted), appVersion, transport∈{direct|relay|file} }` and response `{ lease, enforcementState, nextDue }`, field names verbatim · the lease object `{ serial(UUIDv7), licenceId, instanceBindingId, notAfter(epoch-ms), signature(hex), signerKeyId }` · RFC 9457 errors (seat-limit 409, fingerprint<threshold 422), `Idempotency-Key` on mutations, and **exactly how `nextDue` is computed** so client cadence and server expiry cannot drift.
**Auth today (O1):** which bearer/token type+scope the device uses under `/organisations/{orgId}/…` today, since v0.6.0 declares no global security scheme.
**Key-trust (D1/O2):** the shipped pinned-root set + how operators obtain it out-of-band · rotation cadence + the **dual-pin overlap window** so rotation never strands a fielded build · the intermediate-chain format `signerKeyId`→root.
**PoP signing (D2):** exact bytes signed, header name+encoding, nonce/timestamp/replay binding for `devicePublicKey` (Ed25519) auth.
**Canonical-CBOR (D3):** deterministic lease-body encoding (field order, map-key sort, int/string rules), **bare Ed25519 (no COSE)**, plus a **golden vector** (lease body + signed bytes + signature) to pin a test; same canonicalisation for the `resolveDeviceBuild` signed-manifest pre-image.
**Freshness/clock:** `serverNonce`/per-instance chain (monotonic counter + `prevLeaseHash`) semantics; revocation by **non-reissue only** (no push kill verb); on Untrusted clock, use "time since last verified contact", never wall-clock expiry to tighten.
**Claim-code (O3):** the 6-char glyph alphabet **verbatim**, TTL=15m, single-use + reject window=24h, free-auto-issue rate-limit, and how a free `licenceId`/`instanceBindingId` is minted (free tier = first-class signed lease).
**Rungs/timings:** the 9 `enforcementState` strings + each server trigger + the additive-only policy (client fail-opens unknown → most permissive); offline-grace timings equal the in-tree constants **verbatim**.
**orgId (O4):** which `{orgId}` a free self-host activates against (well-known default vs operator-set).
**Seat/rebind/offline/telemetry/build:** seat consumption, rebind ≥engineer / `fpScore`≥70 vs re-claim, the offline `.challenge`→90-day-lease→install signing, the telemetry closed allowlist + consent doc, and `resolveDeviceBuild` manifest+platform enum (`linux-{x86_64,aarch64}`/`macos-{aarch64,x86_64}`, no Windows).

### Constants that must match the portal verbatim (never round/re-derive)

`LEASE_FULL=35d · LEASE_GRACE=14d · LEASE_HARD=90d · ACTIVATION_WINDOW=31d · CLAIM_CODE_TTL=15m · CLAIM_REJECT_WINDOW=24h · TRANSFER_WINDOW=72h · FINGERPRINT_MATCH_STRONG=100 · FINGERPRINT_MATCH_THRESHOLD=70 · CLAIM_CODE_LEN=6 · signing = Ed25519`. "A portal showing 35 days and a machine enforcing 30 is a support incident."

## Rationale

The wire shapes and the enforcement ladder are stable across v0.4.0→v0.6.0, so the trait,
types, fake server, and the OFF-honesty/tamper/data-min/never-off-air tests are
shape-independent and could be scaffolded now — but the **trust and signing** details
(D1/D2/D3) are precisely what makes a heartbeat client *correct and secure*, and shipping a
guessed trust model is worse than waiting. Documenting the gate as a named service
dependency (this ADR) is the rule-42/rule-6 alternative to silent debt or a stub.

## Alternatives considered

| Alternative | Rejected because |
| ----------- | ---------------- |
| **Stub the heartbeat now against the guessed wire** | Ships a guessed trust/signing model as if done (rule 6); a wrong pinned-root/canonicalisation is a security defect, and `todo!()` on a security boundary is the worst place for it. |
| **Scaffold the feature-flag + trait + fake now, defer only the HTTP client** | Defensible and low-rework (the operator may opt into this), but the operator asked to review the *updated* docs first; the shapes may shift, and the net win is small versus landing it all in the one gated wave. Held, not rejected. |
| **Leave it undocumented until the docs land** | Loses the captured v0.6.0 baseline, the precise gap (D1/D2/D3), and the verification checklist — re-derived cost every session. |

## Consequences

- **Ready to fire:** when the operator signals, implementation is a diff-vs-baseline + this
  checklist + the one wave — fast, no re-discovery.
- **Honest gate:** the remainder is tracked as a gated task, not parked work, and the
  default build stays a pure LGPL-clean, network-free shell with `unlicensed-build` honesty
  ([ADR-0050 §7](ADR-0050.md)).
- **Invariant guard:** the heartbeat task is, by construction, physically unable to
  back-pressure the engine (#10), and revocation-by-non-reissue means no licence code path
  can take a running program off air at any rung (#1) — both must be re-proven by the
  extended chaos gate in the implementing wave.
- **Operator dependency:** progress on CONSPECT-3/12 is bounded by the API publication of
  D1/D2/D3 + the O1–O4 confirmations; this is the one licensing item with a real external
  blocker.
