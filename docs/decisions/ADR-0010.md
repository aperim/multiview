# ADR-0010: Declarative layered config: TOML/JSON + serde adjacently-tagged enums + JSON Schema

- **Status:** Proposed
- **Area:** Core Engine
- **Date:** 2026-06-02
- **Source brief:** [core-engine.md](../research/core-engine.md)

## Decision

Model Canvas->Layout->Cells->Overlays as serde structs with adjacently-tagged enums for unions, authored in TOML and canonicalized as JSON, validated by schemars-generated JSON Schema + garde/validator, loaded via figment, with a schema_version and migration policy. The resolver flattens layouts (presets / CSS-grid / absolute rects) into an ordered DrawQuad list.

## Rationale

Adjacent tagging is robust across TOML (non-self-describing) and JSON, unlike untagged. One type set yields (de)serialization + a published JSON Schema (the multiviewer industry pattern). DrawQuads map 1:1 onto any compositor backend, enabling backend-agnostic layouts and cheap atomic relayout.

## Alternatives considered

Untagged enums (rejected: fragile, poor errors, breaks forward-compat). YAML-only (rejected: weaker schema tooling). Pure absolute-rect schema (rejected: verbose for grids). Pure-grid schema (rejected: cannot express overlapping PiP).

## Consequences

Runtime relayout = diff resolved DrawQuads (atomic swap on custom compositor; probe-blocked pad add/remove on GStreamer). NDI bindings are name-based with offline-card fallback. TOML can't represent null / multiline area maps cleanly — JSON is canonical for the API.
