# ADR-0033 — AES67 / SMPTE ST 2110-30 audio I/O: send + receive architecture

**Status:** Accepted (2026-07-11) — the software tier (receive `Aes67AudioProducer`, the `Aes67Packetizer`/RTP send framing, marker=0 continuous egress, the `channel_bridge` drop-oldest fix) is implemented behind the off-by-default `st2110`/`aes67` features and loopback-validated; the PTP-anchored absolute send timestamp (§4), the `rubato` ppm boundary resampler (§5, identity when locked), and real Dante/AES67 hardware interop remain a flagged operator/hardware follow-on (needs a PTP-capable NIC with a PHC).
**Area:** streaming/timing (T) — audio-over-IP transport (implementation)
**Drivers:** the open-interop-first audio-over-IP requirement and the
LGPL-clean-default + open-standards-only mandate (CLAUDE.md,
[conventions §7](../architecture/conventions.md)).
**Research:** [aes67-delivery](../research/aes67-delivery.md).
**Related:** [ADR-T010](ADR-T010.md) (AES67/ST 2110-30 is the open Dante interop path; native
Dante is NOT supported — this ADR is the *implementation* under it), [ADR-T003](ADR-T003.md) (unified timing),
[ADR-T006](ADR-T006.md) (dynamic resample, never drop/dup), [ADR-R005](ADR-R005.md)
(audio mix/route + capability matrix), [ADR-0008](ADR-0008.md)/[ADR-0028](ADR-0028.md)
(NDI runtime-load — the contrast: AES67 needs **no** SDK).

## Context

[ADR-T010](ADR-T010.md) decided that Dante/broadcast audio-over-IP interop is delivered
via the **open** AES67 / SMPTE ST 2110-30 standard (RTP + L16/L24 PCM + SDP + SAP +
PTPv2), not native Dante. That ADR established *direction*; it did not specify the
send/receive implementation, the clock reconciliation, the discovery tiers, or the
config surface, and it framed discovery purely as SAP/SDP without reference to the
NMOS node that has since been built.

Multiview already owns the load-bearing receive primitives (the L16/L24 depacketizer
`v30.rs`, the RTP parser `rtp.rs`, the essence-agnostic ST 2022-7 merge, the pure
`PtpServo`/`ReferenceTracker`, the lock-free `AudioStore`) and a served NMOS IS-04/IS-05
node — all property-/integration-tested. Because AES67 is open, this ships in the
off-by-default but **pure-Rust, `unsafe_code = forbid`, LGPL-clean** `st2110` feature with
**no SDK and no runtime-load gate** — unlike NDI ([ADR-0008](ADR-0008.md)). The genuinely
new work is the sender, the f32↔Lk codec, an audio `FrameProducer`, AES67 SDP build/parse,
SAP, absolute placement on the store, the clock-domain resampler, and the config/control
surface.

## Decision

1. **Wire format.** Reuse `V30Payload` + `RtpPacket::parse` for receive **verbatim**.
   Build a new `Aes67Packetizer` (`multiview-output`) for send: 12-byte RTP header
   (v2, no pad/ext/CSRC, dynamic PT 96–127, random constant SSRC, `seq.wrapping_add(1)`,
   **`marker = 0` on every packet** — RFC 3551 "no silence suppression MUST set M=0";
   Multiview is continuous, inv #1); encode f32→big-endian Lk as the exact inverse of
   `v30.rs` — **L16** `clamp·32767`, **L24** `clamp·8388607` then high 3 bytes of `(v<<8)`
   (scale by `8388607`, never `8388608`, to prevent full-scale positive→most-negative
   wrap). RTP `timestamp` advances by **sample-groups** (+48 per packet at 1 ms/48 kHz),
   never by bytes or channel count.
2. **Conformance.** Ship **ST 2110-30 Class A** first (48 kHz, L16/L24, 1–8 ch, 1 ms
   ptime = 48 groups/packet; stereo L24 = 288-byte payload) — the mandatory *receiver*
   baseline and therefore the interop-maximizing *sender* target, and the Dante-AES67
   clamp. ptime 250 µs/125 µs (Class B/C) is additive config; AX/BX/CX are out of scope.
3. **Receive.** New `Aes67AudioProducer` (`multiview-input/src/st2110/`, parallel to the
   video `St2110Producer`, not a reuse of it): RTP → `V30Payload` → i32→f32 (L16 `/32768`,
   L24 `/8388608`) → `AudioBlock` → new `AudioStore::publish_at(frame)` for absolute,
   gap/reorder-tolerant placement (silence-fill, never a click). Interpret the media-clock
   timestamp at **48 kHz**, not the video 90 kHz. Honour the signalled `a=mediaclk:direct`
   offset (subtract before mapping; default 0 only under ST 2110-30-strict).
4. **Clocking — PTP media-clock reference, never a pacer** (inv #1; [ADR-T010](ADR-T010.md)
   §3 verbatim). Build on the existing `PtpServo`/`ReferenceTracker`; **do not** write a
   new servo or author/transmit PTP messages — rely on **external linuxptp** (`ptp4l` +
   `phc2sys`), reading the PHC via the `ptp` feature (safe `rustix`) or the disciplined
   system clock via `ntp`/`sysref.rs`. **No `multiview-ptp-sys`.** SEND timestamp is a
   **PTP-anchored absolute** value computed in `u128` or the bit-identical
   seconds/nanoseconds split (`secs*48000 + nanos*48000/1e9`) mod 2³² — never float
   (inv #3), never the tick-keyed `SampleClock` instance (that is `CLOCK_MONOTONIC`, not
   TAI). Re-anchor per packet from PTP; per-packet `wrapping_add(+48)`.
5. **Boundary resampler (net-new, soak-gated).** A continuous soft fractional resampler
   driven by the **measured PTP ppm offset** reconciles the monotonic tick vs the PTP
   media clock in both directions (the [ADR-T006](ADR-T006.md) "resample, never drop/dup"
   mechanism). Pure-Rust via `rubato` (not the `ffmpeg` resampler). Identity when
   `LockState::Locked`; exercised only in `Freerun` / divergent-grandmaster. Inherits
   ADR-T006's 72 h-soak + EBU-R37 acceptance. On receive it sits **between depacketize and
   `publish_at`**. Option (b) free-run is the **graceful degradation** of (a), same path.
6. **Discovery tiers.** Tier 0 static SDP/SDP-URL; Tier 1 SAP (RFC 2974, UDP 9875) on a
   **set** of groups — **`239.255.255.255`** (de-facto AES67/Dante, default, required for
   interop) **and** `224.2.127.254` (RFC-registered); ≥30 s AES67 cadence, 10×/1 h purge,
   `T`-bit deletes; Tier 2 NMOS IS-04/IS-05 (already built) finished **after** SAP/SDP,
   never blocking plug-and-play. **One** AES67-audio SDP parser/generator in
   `st2110/sdp.rs`; IS-05 `transport_file` delegates to it.
7. **Isolation (inv #1/#10).** Send is a bounded drop-oldest SINK fed from the existing
   off-hot-path bake consumer (`ProgramBus::tick()`); the engine never `.await`s it and
   `send_to` never runs on the output-clock loop. Receive never back-pressures the wire —
   but the `channel_bridge` "drop-oldest" defect (it currently drops the **newest**) is
   **fixed to genuine drop-oldest** before reuse, TDD-first.
8. **Resilience.** Stream loss → `AudioStore` silence-fill + `SignalLoss`/`Silence` alarms.
   Reference (PTP) loss → output free-runs, **program never muted** (Multiview's own
   choice; AES67 mandates no mute-on-unsync) + new **`AlarmKind::ReferenceLoss`**;
   receiver falls from last-disciplined origin (Holdover) to wall-clock arrival placement
   with PTS marked estimated (Freerun). PLC v1 = silence + edge fade ramp. ST 2022-7
   **receive in scope** (reuse `HitlessReconstructor`, with SSRC/address validation before
   dual-path bind), **send out** (v2).
9. **Crate/feature/config.** No new crate; extend `multiview-input`/`-output`/`-audio`/
   `-engine`. Add internally-tagged `#[non_exhaustive]` AES67 variants to **both**
   `SourceKind` (`schema.rs:213`) and `Output` (`schema.rs:531`); AES67-out is the first
   output with **no encode/gpu_pin stage** (handle the exhaustive `gpu_pin()`/`audio()`
   matches). Expose as IS-04 `MediaFormat::Audio` / `rtp.mcast`; read-only discovery
   endpoint; tile binding is Class-1 (inv #11). Linux = facility-grade; macOS =
   receive-tolerant / free-run-send (no public PHC), ST 2110-30-strict gated to Linux.

## Consequences

- Open AES67/ST 2110-30 send **and** receive on the standard the IP-video plant already
  speaks, with **no SDK, no licence gate, LGPL-clean default**, reusing the tested receive
  + PTP + NMOS code.
- New open work, all behind `st2110`/egress features and `unsafe_code = forbid`:
  `Aes67Packetizer`, f32↔Lk codec, `Aes67AudioProducer`, `AudioStore::publish_at`,
  `st2110/sdp.rs`, SAP, the `rubato` ppm resampler, `AlarmKind::ReferenceLoss`, the
  `channel_bridge` drop-oldest fix, and the config/control surface.
- **linuxptp (`ptp4l` + `phc2sys`) is an external operational dependency** for full
  facility-grade interop; Multiview never disciplines the PHC itself. macOS runs at
  reduced clock accuracy (system-clock reference), so ST 2110-30-strict conformance is
  Linux-only.
- The resampler is the long-pole: net-new, `rubato`-based, 72 h-soak-gated; until it
  lands, locked operation (ppm≈0) works on the FIFO alone and free-run drift is bounded by
  the AES67 jitter buffers.
- Limits from the AES67 standard (per-device enablement, multicast-only, non-redundant for
  plain Dante, ≤8 ch/flow, SAP/SDP-only discovery, DDM proxy for legacy Dante) are surfaced
  honestly. DVS is not an AES67-capable interop target.

## Alternatives rejected

- **Reuse `SampleClock`'s tick-keyed accumulator for the SEND timestamp** — it is
  `CLOCK_MONOTONIC`-referenced, not PTP/TAI; timestamps would walk away from every
  receiver and other sender. Use a PTP-anchored absolute (u128 / seconds-nanoseconds
  split) instead. Rejected.
- **`marker` set on first/after-discontinuity** — a RFC 3551 MUST violation for a
  continuous, non-silence-suppressed sender; conformance analyzers flag it. M=0 always.
  Rejected.
- **Hardcode a single SAP group (either `224.2.127.254` *or* `239.255.255.255`)** — the
  registered group discovers nothing from shipping Dante/AES67 gear; a single de-facto
  group ignores spec-pure/RAVENNA peers. Announce/join the **set**. Rejected.
- **The `ffmpeg`-gated fixed-ratio resampler as the reconciler** — it has no ppm
  compensation knob and pulls FFI into the `unsafe_code = forbid` path; ~100 ms drift
  accumulates. Use `rubato` driven by the measured PTP offset. Rejected.
- **A `multiview-ptp-sys` FFI crate / a Multiview-owned IEEE-1588 stack** — unnecessary;
  the servo is pure and the PHC read is safe `rustix`; linuxptp speaks the wire. Rejected.
- **A new dedicated AES67 crate** — the seams already exist in
  `multiview-input`/`-output`/`-audio`/`-engine`. Rejected.
- **Mute the program on PTP loss** (broadcast-endpoint behaviour) — violates inv #1; the
  local output free-runs. Rejected (and AES67 mandates no such failsafe anyway).
