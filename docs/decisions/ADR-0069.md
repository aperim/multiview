# ADR-0069: Transcoded ISO variant + trigger-gated capture ŌĆö a separate-encode proxy tier and a unified cue/motion/object arm/disarm rule

- **Status:** Proposed
- **Area:** Output / Engine / Detection / Config / Control / Web
- **Date:** 2026-06-13
- **Source brief:** [recording-storage-offload.md](../research/recording-storage-offload.md) ┬¦1, ┬¦2
- **Extends:** [ADR-0037](ADR-0037.md) (raw copy-remux ISO + Program recorder, the rolling segmenter, dual-axis retention, the bounded drop-oldest write path, disk-pressure disarm ŌĆö **reused verbatim, not re-decided**)
- **Builds on:** [ADR-0032](ADR-0032.md) (rolling segmenter + atomic publish), [ADR-0035](ADR-0035.md) (`HealthWarning` model)
- **Relates to:** [ADR-0067](ADR-0067.md) (record-on-named-object ŌĆö this ADR generalises its arm rule across cue/motion/object), [ADR-R004](ADR-R004.md) (Class-1/Class-2, inv #11), [ADR-T001](ADR-T001.md) / [ADR-RT004](ADR-RT004.md) (invariants #1/#10), invariants #6 (decode-at-display-res), #7 (encode-once-mux-many), #9 (resource-adaptive degradation)

## Context

[ADR-0037](ADR-0037.md) ships the **raw** ISO (faithful copy-remux of every elementary stream, zero re-encode) and the Program recorder (the dirty composited program, zero extra encode). The operator's intake adds two adjacent asks: an **ISO recording (transcoded)** ŌĆö a re-encoded archival proxy of a single source ŌĆö and **trigger-gated capture** (start/stop recording on a cue, motion, or object event). Neither is a new recorder; both are extensions of the [ADR-0037](ADR-0037.md) recorder.

A transcoded ISO is, by definition, a **re-encode at different codec/res/bitrate**. Invariant #7 forbids a *redundant* second encode of the *same* pixels at the *same* settings; it explicitly permits a separate encode when codec/res/bitrate differ ŌĆö which is the entire point of a small uniform proxy ("a week of 30 cameras on a NAS"). So the transcoded ISO is a *legitimate distinct encode pipeline*, the exact shape of the deferred *clean* program feed ([iso-program-recording.md ┬¦B](../research/iso-program-recording.md)).

Trigger-gated capture already has a partial decision: [ADR-0067](ADR-0067.md) defines record-on-named-object as "a rule that arms the existing recorder." The intake adds **cue** ([broadcast-cues.md](../research/broadcast-cues.md)) and **motion** ([motion-scene-detection.md](../research/motion-scene-detection.md)) triggers. Rather than three bespoke paths, they should share one rule contract so the recorder sees a single arm/disarm semantics.

## Decision

1. **Transcoded ISO is an optional separate-encode tier on the existing ISO sink.** Extend `IsoRecording` with an optional `transcode: Option<TranscodeProfile{codec, width, height, bitrate}>` (absent = raw, the [ADR-0037](ADR-0037.md) default). When present, the recorder tees from the **per-source decoded-frame path** (reusing the *already-paid* decode ŌĆö **no second decode**, invariant #6 ŌĆö at full source cadence + normalized source PTS, with the source's audio on its own path), feeds a **dedicated encoder** under its own rendition id, and routes the resulting packets into the **same** rolling segmenter + bounded drop-oldest write path the raw tier uses ([ADR-0037 ┬¦E](../research/iso-program-recording.md)). Only the encoder is new; segmentation, retention, disk-pressure, offload, and catalog are shared. **Off by default.** *(It must tap the source decode path, not the compositor's last-good tile: the tile store is display-res, output-cadence-sampled, can be frozen on STALE/NO_SIGNAL, and carries no source audio/PTS ŌĆö tapping it would silently produce a sampled visual proxy, not a faithful per-source archive. A cheaper "tile proxy" capture is a separate, explicitly-named mode ŌĆö see the brief ┬¦1.1 design note / `Open question`.)*

2. **A transcoded ISO is admitted and shed like any GPU/CPU co-tenant.** It is budgeted at **below-program priority** ([self-aware-placement.md](../research/self-aware-placement.md), [efficiency.md](../research/efficiency.md)) and **shed first** under resource pressure (invariant #9) ŌĆö the engine drops the archive proxy before any program tile or frame. A proxy that cannot be admitted (no encoder, no budget) surfaces a typed capability `HealthWarning` (the `AudioCapability` precedent); a refused proxy is loud, not silent. Validation requires an available encoder for the chosen codec; the LGPL-clean default proxy is H.264 (HEVC/AV1 are HW-gated / `gpl-codecs`-aware options).

3. **Frame-cap and ISO-audio are first-class media kinds of the same recorder.** A periodic/triggered still (frame-cap) and an audio-only ISO (register only audio inventory rows in the `IsoMuxSink`) reuse the recorder + retention + offload + catalog lifecycle; a `media_kind Ōłł {video, frame_cap, iso_audio}` discriminator tags each artefact (consumed by the catalog, [ADR-0071](ADR-0071.md)).

4. **One unified `CaptureTrigger` rule for cue / motion / object.** A trigger is *a rule that arms the existing recorder* ŌĆö never a new recording path. Each rule carries: a **source predicate** (cue type / motion zone / object label+zone+confidence ŌĆö each owned by its own brief), a **target** (an ISO on a named input or the Program ŌĆö reuses `IsoRecording`/`ProgramRecording` verbatim), a **window** (`pre_roll` = pin + do-not-prune the last N seconds; `min_duration`; `hold`/`cooldown`), and **de-bounce** (require K matches before arming). [ADR-0067](ADR-0067.md)'s object trigger becomes one instance of this contract.

5. **Triggers apply off the data plane, Class-1 hot.** A fired rule submits the **same** `Arm/Disarm{Iso,Program}Recording` command [ADR-0037 ┬¦F](../research/iso-program-recording.md) defines (`202` + op-id, `Idempotency-Key`, non-blocking `try_submit` shed-to-`503`), from the cue/detection worker tier ŌĆö **never** the clock thread. Arm/disarm is **Class-1 hot** ([ADR-R004](ADR-R004.md), invariant #11): it flips a best-effort sink's state. A triggered clip gets a `pinned_until` flag that exempts it from time/size retention prune **but never from disk-pressure disarm** (a full disk still wins). A trigger that cannot arm (disk pressure, capability) raises a `HealthWarning`.

6. **Config/API/realtime/UI are additive.** `transcode` on `IsoRecording`; a `[capture_triggers]` rule list; a `/capture-triggers` CRUD; the existing `RecordingStatus` event carries the pinned/transcoded state. `#[serde(tag=...)]` unions, never `untagged`.

## Consequences

**Positive.**
- The operator's transcoded-ISO ask is met without violating invariant #7 ŌĆö it is the sanctioned separate-encode case, off by default, budgeted, and shed-first.
- One trigger contract serves cue, motion, and object identically; [ADR-0067](ADR-0067.md)'s clip lifecycle delegation ([object-detection-ai.md ┬¦5.2](../research/object-detection-ai.md)) lands cleanly.
- Frame-cap and ISO-audio become first-class, catalog-ready artefacts with zero new lifecycle code.

**Negative / costs.**
- A transcoded ISO is a real encode (GPU/CPU). It is the one piece here with non-trivial cost; mitigated by off-by-default + admission + shed-first, but operators enabling many proxies must size hardware.
- A new encoder wiring (dedicated rendition id, tap the decoded frame) ŌĆö a footgun-bearing path mirroring the deferred clean-feed work; it must reuse the recorder's bounded ring, never its own bespoke buffer.
- Trigger de-bounce/hold tuning is operator-facing; bad values flap arm/disarm (mitigated by required `hold`/`cooldown` + K-match de-bounce).

**Neutral.**
- Arm/disarm stays Class-1 hot; warnings surface at `GET /api/v1/health`, not `/livez`/`/readyz`.

## Alternatives considered

1. **Transcode every source by default.** *Rejected* ŌĆö unbounded GPU/CPU cost; violates the efficiency standing review. Default off, named subset, budgeted.
2. **Transcoded ISO as a "second decode + encode" standalone pipeline.** *Rejected* ŌĆö wastes a decode (invariant #6); reuse the already-decoded last-good frame.
3. **A separate trigger path per sensor (cue vs motion vs object).** *Rejected* ŌĆö three bespoke arm semantics to maintain; one `CaptureTrigger` contract is simpler and is what [ADR-0067](ADR-0067.md) already implies.
4. **Triggers mutate the recorder on the engine thread for "instant" capture.** *Rejected* ŌĆö violates invariants #1/#10; arm/disarm rides the command bus at a frame boundary, which is already sub-frame-latency and Class-1 hot.
5. **Pin triggered clips by exempting them from disk-pressure too.** *Rejected* ŌĆö a full disk must always win (invariant: a recorder never blocks to keep bytes). Pinning exempts only *retention prune*, not disk-pressure disarm.

## How this honors the invariants

- **#1 (output-clock untouchable).** No path here runs on the output-clock thread. The transcoded encode runs on a recorder/co-tenant task; triggers enqueue commands the existing drain applies at a frame boundary. `out_pts = f(tick)` is independent of transcode success, trigger storms, or recorder state.
- **#10 (isolation).** The transcoded ISO routes through the **same bounded drop-oldest ring + off-hot-path writer** as the raw recorder ([ADR-0037 ┬¦E](../research/iso-program-recording.md)); the engine never awaits it. Triggers are off-data-plane, non-blocking `try_submit` shed-to-`503`. A wedged transcode or trigger storm drops + warns + backs off ŌĆö never stalls, never grows RAM.
- **#6 / #7 / #9.** Decode is reused (no second decode); the transcode is the inv-#7-sanctioned separate encode (codec/res/bitrate differ); it is admitted below program priority and shed first (inv #9).
