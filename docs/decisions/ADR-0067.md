# ADR-0067: Detection event model + record-on-named-object trigger — advisory events on the realtime stream, arming the existing ISO/Program recorder

- **Status:** Proposed
- **Area:** Detection / Events / Control / Output / Config / Web
- **Date:** 2026-06-13
- **Source brief:** [object-detection-ai.md](../research/object-detection-ai.md)
- **Builds on:** [ADR-0066](ADR-0066.md) (the inference seam + `Detector` trait that produces the detections), [ADR-0037](ADR-0037.md) (the ISO + Program recorder the trigger arms — bounded drop-oldest ring, disk-pressure gating, arm/disarm ops), [realtime-api.md](../research/realtime-api.md) (event envelope, topics, snapshot-then-delta, conflation), [self-aware-placement.md](../research/self-aware-placement.md) (the `HealthWarning` model).
- **Relates to:** invariant #1 (output never gated on detection), invariant #10 (detection events conflated/drop-oldest, never back-pressure), invariant #11 ([ADR-R004](ADR-R004.md) — arm/disarm is Class-1 hot).

> **Scope note.** This ADR makes record-on-named-object a **rule that arms the recorder already designed in [ADR-0037](ADR-0037.md)** — it is **not** a new recording path. Until a real detector is connected (per [ADR-0066](ADR-0066.md)), the rule fires only against the deterministic stub detector; nothing here ships a model.

---

## Context

The operator asked to **record on named object detection** once AI integration exists. Two failure modes to avoid: (a) inventing a second recording path instead of reusing the bulletproof one ([ADR-0037](ADR-0037.md) already gives a bounded, disk-pressure-gated, isolated recorder with arm/disarm ops and a rolling segmenter), and (b) letting detection events or triggers reach the data plane in a way that could back-pressure or stall output.

In-tree facts that make this cheap:

- **The recorder exists in design with arm/disarm as a Class-1 hot op.** `POST …/recording/arm|disarm` (`202` + op-id, `Idempotency-Key`, shed-to-`503`), an `Event::RecordingStatus`, and four additive `WarningCode` variants — [ADR-0037 §F](ADR-0037.md). Arm/disarm "only flips a best-effort sink's state."
- **The event model is tagged + `#[non_exhaustive]` and has a conflatable high-rate lane.** `crates/multiview-events/src/event.rs` (every payload `#[non_exhaustive]`, tagged, never `untagged`); `WarningCode` is `#[non_exhaustive]` (`event.rs:319`); `Topic::AudioMeters` (`crates/multiview-events/src/topic.rs:37`) is the precedent for a high-rate conflated lane (`topic.rs:122` marks it conflatable).
- **The recorder's rolling segmenter + retention** ([ADR-0037 §C](ADR-0037.md)) already keeps recent footage — which is exactly what **pre-roll** needs ("don't prune the last N seconds").

The decision is the **shape of detection events** and **how a match arms the recorder without touching the hot path.**

---

## Decision

**1. Detections are advisory events on the existing realtime envelope.** Add (all tagged, `#[non_exhaustive]`, never `untagged`, mirroring `event.rs`):
   - `Event::DetectionFrame(DetectionFrame { scope, detected_at: MediaTime, detections: Vec<Detection>, model: ModelId, dropped: u32 })`, where `scope ∈ { Input{input_id}, Program, CuedSource{id} }` (mirroring `RecordingStatus`'s scope) and `Detection { label: Label, confidence: f32, bbox: NormRect, zone: Option<ZoneId>, track_id: Option<TrackId> }`. Coordinates are **normalized `[0,1]`** so they survive the display-res decode (invariant #6) and any downscale. `track_id` is reserved (no tracker is specified here).
   - `Event::DetectionTrigger(DetectionTrigger { rule_id, action, label, confidence, scope })` — the auditable "why we armed a recorder."

**2. Detection events ride a conflated, drop-oldest, re-snapshotable lane (invariant #10).** They share the high-rate conflatable topic discipline of `AudioMeters`: latest-per-scope wins under load, a late subscriber gets the current state, and the engine publishes through the existing drop-oldest publisher and **never awaits** a subscriber. They are **never** correctness-critical and **never** polled. A busy scene / low threshold cannot back-pressure — conflation collapses the flood.

**3. Zones & labels are operator config in normalized geometry.** Zones are polygons/rectangles in normalized source/canvas coordinates (shared geometry with the motion gate, [motion-scene-detection.md](../research/motion-scene-detection.md), and the react-konva layout editor). A detection is tagged with the zone(s) it falls in. Label filtering is an allow/deny list + per-label min-confidence over the **open vocabulary** the model advertises (`Detector::labels()`, [ADR-0066](ADR-0066.md)); the engine hard-codes no class taxonomy.

**4. Record-on-named-object is a rule that arms the existing recorder.** A `DetectRecordRule` (additive, serde-default config) declares **match** (`labels`, `min_confidence`, optional `zones`/`scope`), **target** (an **ISO** recorder on the matched input or the **Program** recorder — reusing `IsoRecording`/`ProgramRecording` from [ADR-0037](ADR-0037.md) verbatim), and **window** (`pre_roll`, `min_duration`, `cooldown`/`hold`), with **de-bounce** (require K matches in a sliding window). On a sustained match the rule submits an **`Arm{Iso,Program}Recording`** command on the existing command bus; on no-match-for-`hold` it submits **disarm**. Pre-roll is satisfied by the recorder's rolling retention ("pin the last N seconds"); triggered clips are pinned segments whose lifecycle (retention/export/offload) is owned by [recording-storage-offload.md](../research/recording-storage-offload.md), not duplicated here.

**5. The trigger never touches the data plane (invariants #1, #11).** The rule runs in the detection worker tier (off the clock thread) and only **enqueues a command** that the existing drain applies at a frame boundary. Arm/disarm is **Class-1 hot** ([ADR-R004](ADR-R004.md)). The recorder's own bounded drop-oldest ring + disk-pressure gating ([ADR-0037 §E](ADR-0037.md)) means even a storm of triggers cannot stall output or grow RAM — a full/slow disk drops + warns + backs off, exactly as already designed. The trigger worker additionally **coalesces idempotently per (rule_id, target)**: it tracks the last-submitted arm/disarm intent and emits a command only on a *state transition* (disarmed→armed or armed→disarmed), with the existing `Idempotency-Key` on the arm/disarm op ([ADR-0037 §F](ADR-0037.md)) deduplicating retries. Combined with the K-of-N de-bounce and `hold`/cooldown, a noisy scene cannot flood the command bus — a stable detection yields at most one arm then one disarm per match episode.

**6. Auditability + fail-loud.** Every fire emits `Event::DetectionTrigger`; the recorder's `Event::RecordingStatus` shows bytes/segments/state; a `HealthWarning` (`WarningCode` is `#[non_exhaustive]`, `event.rs:319`) surfaces a down detector, a model that failed to load, or a rule that could not arm (e.g. disk pressure) — so a silently-not-recording detector is loud, not invisible.

---

## Consequences

**Positive.**
- Record-on-detection reuses the entire bulletproof recorder ([ADR-0037](ADR-0037.md)) — bounded, isolated, disk-pressure-gated — instead of a parallel path. The trigger is a thin rule.
- Pre-roll is nearly free (the segmenter already retains recent footage). De-bounce + hold kill false-positive flapping.
- Detection events are first-class on the existing realtime stream with the proven conflation/snapshot semantics; the UI gets boxes/labels/zones for free off the envelope.

**Negative / costs.**
- New event variants + a rule type + the zone geometry, plus the program→tile coordinate mapping for program-scope rules (open question, brief §7 Q3).
- Tuning surface (confidence, K-of-N, hold, pre-roll) is real operator-facing complexity; sensible defaults are required.
- A misconfigured rule (too-low confidence) could arm recorders often — bounded by the recorder's own disk-pressure gating + retention, and surfaced via warnings, but still an operator footgun to document.

**Neutral.**
- `track_id` and a possible `Event::SceneAssessment` (multimodal, slow cadence) are reserved/forward-compatible, not v1.
- Detection→switcher (auto-director) is explicitly out of scope; if pursued it must go through the command bus + frame-boundary seam, operator-opt-in.

## Alternatives considered

1. **A bespoke detection-recording path.** *Rejected* — duplicates the recorder's bounded-ring / disk-pressure / retention machinery and risks a non-isolated write path. Reuse [ADR-0037](ADR-0037.md); the trigger only arms/disarms with a window.
2. **Push detection results into the engine to drive recording synchronously.** *Rejected* — couples a best-effort consumer to the data plane (invariants #1/#10). The trigger enqueues a Class-1 command applied at a frame boundary, off the clock thread.
3. **A dedicated correctness-critical detection socket (no conflation).** *Rejected* — detections are advisory and high-rate; they must share the conflated drop-oldest discipline (`AudioMeters`), not a reliable lane that could grow under a busy scene.
4. **Untagged or non-`#[non_exhaustive]` event enums.** *Rejected* — violates the conventions (`#[serde(tag=…)]`, never `untagged`) and the in-tree `event.rs` posture; the catalog must be able to grow additively.
5. **Arm on a single-frame match.** *Rejected* — single-frame false positives would flap recorders. Require K-of-N de-bounce + a motion gate fronting the detector + a `hold`/cooldown before disarm.
6. **Make arm-on-detection a Class-2 (make-before-break) change.** *Rejected* — arm/disarm only flips a best-effort sink's state (safer than `Source.enabled`); it is Class-1 hot per [ADR-0037 §F](ADR-0037.md) / [ADR-R004](ADR-R004.md).

---

## How this honors the invariants

- **#1 (output clock never falters).** Detection events flow outward only; the trigger enqueues a command applied at a frame boundary by the existing drain — never read on the output-clock thread. Recording itself is the [ADR-0037](ADR-0037.md) best-effort sink that cannot stall output. A trigger storm changes nothing about `out_pts = f(tick)`.
- **#10 (isolation — no back-pressure).** Detection events are conflated, drop-oldest, re-snapshotable, published through the existing non-awaiting publisher (the `AudioMeters` lane discipline). The recorder's bounded drop-oldest ring absorbs trigger storms; a slow/full disk drops + warns + backs off. The engine never awaits the detector, the event subscriber, or the recorder.
- **#11 (live-apply classification).** Arm/disarm-on-detection is **Class-1 hot** ([ADR-R004](ADR-R004.md)) — it flips a best-effort sink's state at a frame boundary, strictly safer than `Source.enabled`.
