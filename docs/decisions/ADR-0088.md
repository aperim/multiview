# ADR-0088: Output metadata model — per-output intent projected onto per-transport carriers, capability-gated and verified

- **Status:** Proposed
- **Area:** Output / Config / Control / Web
- **Date:** 2026-06-13
- **Source brief:** [output-metadata-and-orientation.md](../research/output-metadata-and-orientation.md) §1
- **Extends:** [ADR-M002](../decisions/ADR-M002.md) (`EncodeProfile` + per-output transcode + `CapabilityReport` gating), [ADR-C006](../decisions/ADR-C006.md) (always tag output explicitly, then verify with ffprobe)
- **Relates to:** [ADR-M005](../decisions/ADR-M005.md) (live-apply classification + dry-run plan), [ADR-0037](../decisions/ADR-0037.md) / [ADR-0059](../decisions/ADR-0059.md) (bounded, drop-capable best-effort producers off the hot path — the timed-metadata pattern), [ADR-T001](../decisions/ADR-T001.md) / [ADR-RT004](../decisions/ADR-RT004.md) (invariants #1 / #10), [ADR-T003](../decisions/ADR-T003.md) (re-stamp from the tick counter)

## Context

The operator requires **full metadata control for outputs, where transport and codec support it** — a service/program name and provider, a language, a title, and (for HLS/TS) live "now-playing"/cue timed metadata, so downstream players and aggregators identify the stream correctly.

Today there is no per-output metadata surface. The `Output` enum (`crates/multiview-config/src/schema.rs:683`, `#[serde(tag = "kind")]`, `#[non_exhaustive]`) carries addressing + codec + optional `audio`/`gpu_pin`, and `Output::label()` (`schema.rs:1053`) derives a diagnostic label — but nothing maps operator-supplied metadata onto a muxer. A repo grep finds no `service_name`/`service_provider`/SDT/`onMetaData` writer in `multiview-output` — all net-new. Each transport expresses metadata **differently**, and verified facts pin what is real:

- **MPEG-TS:** FFmpeg `mpegtsenc` writes an **SDT** from `-metadata service_name=`/`service_provider=`, plus PMT per-stream ISO-639 language descriptors (DVB SI). (verified — ffmpeg `mpegtsenc.c`)
- **HLS:** timed metadata is **in-band ID3** in MPEG-TS PES (Apple HLS Metadata spec) and/or **out-of-band `EXT-X-DATERANGE`** in the playlist (HLS / W3C media-timed-events) — `EXT-X-DATERANGE` is a date-interval tag with mandatory `ID` + `START-DATE` (not a generic per-sample channel). Per-track language plausibly rides the fMP4 init segment / TS PMT; there is **no** universal playlist "container title" — title/session semantics are per-field, not a blanket capability. (verified)
- **RTMP/FLV:** **`onMetaData`** is an AMF script-data object (width/height/framerate/codec ids/duration + endpoint-read extras). (verified — FLV/Enhanced-RTMP)
- **File containers:** MP4/MOV `udta`/`ilst` and Matroska tags carry title/comment/language. (verified — ISO-BMFF / Matroska)
- **RTSP/NDI:** only thin/vendor surfaces (SDP `s=`/`i=`; NDI sender name via the runtime-loaded SDK).

There is **no universal metadata bag**; a model must declare intent and project it onto whatever the chosen transport+codec can carry, dropping the rest **visibly**.

## Decision

Add a per-output **`OutputMetadata`** declarative-intent struct and a **projection + verification** pipeline.

1. **`OutputMetadata` (config-layer, `multiview-config`).** Additive `metadata: Option<OutputMetadata>` on every `Output` variant (like `audio`/`gpu_pin`), `#[non_exhaustive]`, serde-tagged, round-tripping TOML+JSON. Named fields: `title`, `provider`, `language` (ISO-639-2), `service_id` (DVB/TS program number, `1..=65535`), `description`, and a `timed { id3, daterange }` opt-in. It is **operator intent**, not a per-transport union; it lives in config (not `multiview-core` — no FFI/encode there; not the canvas — it is per-output, not per-program). The runtime carries it on the per-output sink/`EncodeProfile` descriptor so the **mux** stage applies it.

2. **Projection to a capability plan.** For each `(Output, OutputMetadata)` the output layer computes an **`OutputMetadataPlan`**: per requested field, `Applied(target)` / `Dropped(reason)` / `Defaulted`, using a typed per-transport capability matrix (TS SDT/PMT, HLS playlist/ID3, RTMP `onMetaData`, container tags, RTSP SDP, NDI sender name) intersected with the [ADR-M002](../decisions/ADR-M002.md) `CapabilityReport` for codec-dependent carriers. The plan is returned from the dry-run plan endpoint ([ADR-M005](../decisions/ADR-M005.md)) **before** apply. Validation rejects only contradictions (bad `service_id` range), never an unsupported-but-harmless field — that degrades to a **visible `Dropped`**, never a silent no-op.

3. **Mux-time apply.** Each muxer writes its subset: TS → SDT service_name/provider + PMT lang; HLS → init-segment/SDT title+lang + (opt-in) ID3 PES / `EXT-X-DATERANGE`; RTMP → `onMetaData` AMF keys; file → container tags. Fields fan to **all sinks of a shared rendition** at zero extra encode (#7).

4. **Timed metadata is sampled, off the hot path.** ID3 PES / `EXT-X-DATERANGE` events come from the control/cue stream via a **bounded drop-oldest** queue and are injected at the **mux/segment** stage on the bake thread, stamped from the output tick counter (inv #3). A late/absent event ⇒ no metadata this segment, never a stalled segment. SCTE-35 splice carriage is out of scope (it is the [broadcast-cues](../research/broadcast-cues.md) brief's domain).

5. **One verification gate, more assertions.** The [ADR-C006](../decisions/ADR-C006.md) post-encode/remux ffprobe gate is **extended** to assert the requested metadata landed (SDT service_name present, `onMetaData` title present, container `language` set), alongside the existing color assertions. "Requested ≠ delivered" renders amber, never assumed.

## Consequences

- **Positive.** Operators get truthful, standards-based per-output identification on every transport that can carry it; the plan makes "what will and won't land" explicit before apply; the timed-metadata path gives live now-playing/cue carriage without ever touching the clock. Zero extra encode — metadata fans with the rendition (#7).
- **Negative / cost.** Net-new muxer code per transport (SDT/PMT writer wiring, `onMetaData` AMF, container tags, ID3 PES injection) and an extended ffprobe gate. The capability matrix must be kept honest as transports evolve (`#[non_exhaustive]`).
- **Neutral.** Most metadata applies at mux init / periodic SDT — no per-frame work. Timed metadata adds a small, bounded side stream. RTSP/NDI carry little; the plan says so rather than pretending.
- **Deferred (named).** An open `extra: map<string,string>` passthrough (file containers only) is deferred until a concrete need appears, to avoid an untyped escape hatch. NDI metadata depth beyond the sender name is deferred to the NDI integration owner (runtime-loaded, nominative only, [conventions §7](../architecture/conventions.md)).

## Alternatives considered

1. **A single universal metadata bag applied identically to every transport.** Rejected: transports differ fundamentally (DVB SDT vs AMF `onMetaData` vs `udta`); a uniform bag would either silently drop most fields or invent carriers that don't exist. Intent + per-transport projection + a visible `Dropped` is the honest model.
2. **Put metadata on the canvas/program.** Rejected: two outputs of one program legitimately carry different service names/providers; metadata is a per-*output* property. (Color, by contrast, *is* a program property and stays on the canvas, [ADR-C006](../decisions/ADR-C006.md).)
3. **Silently drop unsupported fields.** Rejected: violates the project's "warn, never silently drop" doctrine ([managed-devices](../research/managed-devices.md)). Unsupported fields are surfaced as `Dropped(reason)` in the plan.
4. **Trust the muxer to emit metadata, skip verification.** Rejected for the same reason C006 rejects trusting color defaults: requested ≠ bitstream. The ffprobe gate is extended, not bypassed.
5. **Carry timed metadata on a blocking in-band path tied to frames.** Rejected: it would let a cue producer pace a segment (inv #1/#10 violation). Timed metadata is a bounded, drop-capable, sampled side stream injected off the hot path.

## How this honors the invariants

- **#1 (output-clock).** All metadata is applied at **encode/mux**, never on the tick loop. Static fields are mux-init/periodic-SDT; timed metadata is **sampled** from the cue stream and injected on the bake thread, stamped from the tick counter — a missing event never stalls a frame. The output clock is untouched.
- **#10 (isolation).** The timed-metadata producer is a best-effort, **bounded drop-oldest** source (the [ADR-0059](../decisions/ADR-0059.md)/[ADR-0037](../decisions/ADR-0037.md) seam pattern); the engine never awaits it and it cannot back-pressure the mux or the clock. Under overload, events drop; the stream keeps flowing.
- **#7 (encode-once).** Metadata fans to all sinks of a rendition with no extra encode.
