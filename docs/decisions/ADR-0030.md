## Context

The engine runs exactly **one** program: one `OutputClock` drives one composited NV12
canvas per tick, encoded once, fanned to N transports (invariant #7). That program is
owned by `Pipeline` (`crates/multiview-cli/src/pipeline.rs`): one `cadence`
(`pipeline.rs:228`), one `Arc<Layout>` (`:954`), one source-store map + `ingest_plans`
(`:425`), one `OutputClock`+`CompositorDrive`+`StreamEgress` built inline in
`drive_streaming` (`:951â€“1027`). The config root models one
`canvas`+`layout`+`sources`+`cells`+`outputs` (`crates/multiview-config/src/lib.rs:89â€“134`).
The control plane has a **global** `Start`/`Stop` (`command.rs:82â€“88`); realtime events
carry no program identity (`topic.rs:16`, `event.rs`).

The operator requires **N concurrent, independently start/stoppable programs**, each one
of: (A) multiview composite (today), (B) sourceâ†’output passthrough (remux, no re-encode),
(C) sourceâ†’transcodeâ†’output. They run simultaneously and **must share decoded sources**:
efficiency is the product â€” a source used by k programs is decoded **once**; a passthrough
decodes/encodes **nothing**.

Verified enablers already in the code: `EngineRuntime` is per-instance (own
clock/seed/`ticks_emitted`/`StopSignal`, `runtime.rs:173â€“193,225`) and awaits only its own
pacer (`:405`) â€” no master clock exists. `TileStore` reads are lock-free/wait-free
multi-reader (`latest.rs:92`). `Supervisor`/`Actor`/`RestartPolicy` exist
(`supervisor.rs:165,181,61`). Remux primitives exist (`Muxer::add_stream_from_parameters`
= `avcodec_parameters_copy`, `mux.rs:136`; `Demuxer::read_packet`, `demux.rs:196`) but no
packet-copy consumer is wired. `multiview-hal::select_device` already gates VRAM + the
NVENC session ceiling and supports pinning (`select.rs:22â€“23,90,136,202`).

## Decision

**1. The `Program` abstraction.** A `Program` is one self-contained, independently
supervised output pipeline = one placement island (ADR-0017/0018). Config layer:
`ProgramSpec`, internally tagged (`#[serde(tag="kind")]`, `#[non_exhaustive]`, never
untagged):

```rust
pub enum ProgramSpec {
    Multiview  { canvas: Canvas, layout: Layout, cells: Vec<Cell>, overlays: Vec<Overlay>, outputs: Vec<Output> },
    Passthrough{ input_id: String, on_incompatible: PassthroughFallback, outputs: Vec<Output> },
    Transcode  { input_id: String, encode: TranscodeSpec, outputs: Vec<Output> },
}
pub enum PassthroughFallback { #[default] Reject, Transcode }
```

Engine layer: each program runs as a supervised actor; `MultiviewProgram` is today's
`Pipeline::drive_streaming` body refactored to own its pieces (a **move, not a rewrite**).
A/C share the existing `ProgramEncoder`â†’`StreamEgress`â†’`PacketMuxSink` tail; B gets a new
`PacketCopyFanout` feeding the **same** `PacketMuxSink`s from `Demuxer::read_packet`.

**2. `ProgramSet` supervisor + per-program independent output clocks + strong isolation.**
`ProgramSet` (new, `multiview-engine`) owns the `ProgramIdâ†’ProgramHandle` map, one shared
read-only `Arc<dyn TimeSource>`, the `SourceRegistry`, and the `PlacementController`. Each
clocked program (A/C) owns an **independent `OutputClock`+`EngineRuntime`** with its own
seed/cadence/`StopSignal`, on its own `tokio` task. **No master clock** â€” only a shared
monotonic *reference*. `Program` implements `Actor` so the existing `Supervisor` restarts
a crashed program with capped backoff while siblings keep emitting. `start(spec)` admits +
spawns without touching other programs; `stop(id)` raises only that program's
`StopSignal`, drains its egress within `SINK_WEDGE_GRACE`, joins. **New CI chaos gate:**
wedge one program's egress, assert another's `ticks_emitted` keeps advancing on cadence
(invariants #1 + #10, per program).

**3. Decode-once-use-many via a shared ref-counted `SourceRegistry`.** Hoist the
per-source `TileStore` + `IngestPlan` out of `Pipeline` (`pipeline.rs:425`) into a
process-global `SourceRegistry` owned by `ProgramSet`, keyed by canonical source identity.
One ingest/decode actor per **physical** source publishes into one `TileStore<Nv12Image>`;
every A/C consumer holds an `Arc<TileStore>` clone and *samples* it (zero-contention,
`latest.rs:92`). Ref-counted: ingest starts on first reference, dies on last release.
Decode **once at the supremum requested resolution**; each consumer scales in-shader (#5/#6).
The registry exposes a **decoded** facet (A/C) and a lazy **coded-packet** facet (B) so a
passthrough-only source never spins a decoder. Sources stay **top-level** in config.

**4. Passthrough = remux-no-reencode, transcode fallback.** B does zero
decode/composite/encode; it copies coded packets, paced by the input PTS (#4), with the
monotonic clamp + `avoid_negative_ts=make_zero` + small `max_interleave_delta`, re-anchoring
on discontinuity. **Passthrough is the explicitly-sanctioned exception to invariant #1**:
it does not promise frame-perpetuity (no encoder to fabricate frames); on loss it goes
`NO_SIGNAL` + supervised-reconnect (default) or, opt-in, promotes to a degenerate transcode
(slate). Codec/container incompatibility is a **build-time** gate: incompatible â†’
auto-downgrade to transcode (logged) per `PassthroughFallback`, or reject. Pathological
timestamps â†’ prefer transcode over copy.

**5. Per-program + global resource-adaptive degradation with admission control, tied to
ADR-0017/0018 placement.** Admission runs over the **union** of programs, crediting shared
decode **once** (the `SourceRegistry` emits one de-duplicated decode `TileLoad`/source at
its supremum resolution; per-program composite+encode add on top), fed to `Planner::admit`
(`planner.rs:181`) + `select_device` (VRAM + NVENC session gates already in
`select.rs:90,202`). Extend placement with a **same-source co-location** affinity term.
Degradation becomes cross-program, cheapest-impact-first with operator-assignable per-program
`priority` (default: shed transcode/passthrough before the primary multiview); per-program
queues stay bounded `DropOnOverload`. All programs run in **one process** (CUDA context
paid 1Ã—).

**6. `[[programs]]` config (adjacently/internally-tagged, backward-compatible).** Add
`#[serde(default)] programs: Vec<Program>` to `MultiviewConfig`. The legacy top-level
`canvas`/`layout`/`cells`/`overlays`/`outputs` desugars to one implicit `Multiview`
program (`id="main"`); `into_programs()` synthesizes it; `validate()` rejects populating
both. `schema_version`â†’2 (v1 via the legacy path). New checks: unique program ids; unique
output labels across all programs; every `input_id` resolves to a `sources` entry.

**7. Programs management API + per-program realtime state.** New `PROGRAM_KIND` resource +
`routes/programs.rs` (`GET/POST/PUT/DELETE /api/v1/programs[/{id}]`, ETag/If-Matchâ†’412,
Idempotency-Key, RFC 9457). Lifecycle: `POST /api/v1/programs/{id}/{start|stop|apply}`
(202 + op-id via `submit_accepted`) and `POST /api/v1/programs/{id}/plan` (returns the
Class-1/Class-2 classification, invariant #11). New `Command` variants
`StartProgram{op,program}`/`StopProgram{op,program}`/`ApplyProgram{op,program,..}`;
`SwapSource`/`ApplyLayout` gain `program: String`; legacy global `Start`/`Stop` become
aliases for `"main"`. Realtime: new `Topic::Programs` + `program.state` event
(`ProgramState{state,kind,outputs,fps}`, `ProgramRunState` reusing `OutputRunState`
incl. `Migrating`); scope `OutputStatus`/`TileState` by `Envelope.id="{program}/{output}"`;
snapshot + `CorrKey` extend.

## Consequences

**Positive.** Decode/demux/CUDA-context/VRAM cost for a source used by k programs drops
from O(k) to O(1); passthrough is ~free; the heavy per-program primitives
(`EngineRuntime`, `OutputClock`, `CompositorDrive`, `StreamEgress`, `Supervisor`,
`select_device`) are **reused unchanged** â€” the engine was authored per-program. Per-program
clocks make #1/#10 structural. WebUI manages each program independently.

**Negative / cost.** New shared mutable lifecycle (the `SourceRegistry` refcount) is new
surface that **must stay off every program's hot path** (an `Arc`-drop returning a pooled
buffer must not run in a program's async destructor â€” safety rule #4; design-note
required). Passthrough is a documented #1 *exception* â€” it must be guarded so nobody
"fixes" it into a hidden transcode. The supremum-resolution decode rule needs cost-model
confirmation. Admission must credit shared decode once or it over-rejects. `PipelineError`
must grow `ProgramId` context so one program's failure never aborts the set.

## Alternatives rejected

- **One master clock for all programs.** Rejected: cannot serve 25 + 60 fps without
  resampling (violates #3 exact-rational), and creates a shared stall surface (violates
  #1/#10). Chosen: shared *time source*, independent *clocks*.
- **Process-per-program.** Rejected: multiplies the ~84â€“115 MB CUDA context and forbids
  zero-copy surface sharing; cross-program decode-once becomes impossible. Chosen:
  one process, in-engine `ProgramSet`.
- **Sources nested per-program.** Rejected: forces per-program decode, defeating the
  operator directive. Chosen: top-level shared sources referenced by `input_id`.
- **Passthrough that holds-last-good / slates by default.** Rejected: it has no encoder;
  a default slate forces the decode+encode the operator opted out of. Chosen: honest
  `NO_SIGNAL`+reconnect by default, transcode-slate opt-in.
- **Outputs-as-programs (keep one canvas, add passthrough/transcode as new sink types).**
  Rejected: B/C are different *pipeline shapes* (no composite/encode for B), not new sinks;
  the single-canvas/clock assumption can't host a second cadence.
- **Per-(source,resolution) decode instances.** Rejected as default in favour of
  decode-at-supremum + in-shader downscale (cheaper; #5/#6); revisit if the cost model
  shows the largest consumer doesn't dominate.
- **`dyn Any`-style heterogeneous program list.** Rejected (guardrail bans `dyn Any`);
  use the tagged `ProgramSpec` enum + an `Actor`/enum-dispatch runner.
