# ADR-0030: Multiple active programs (multi-program engine)

- Status: **Proposed**
- Date: 2026-06-07
- Deciders: engine + control-plane maintainers
- Related: ADR-0017/0018 (GPU placement), ADR-E001/E002/E003/E004 (efficiency),
  ADR-0025/0026 (streaming/encode fan-out), ADR-W008 (command bus), ADR-RT002/RT003
  (realtime envelope/snapshot), ADR-R004 (make-before-break), ADR-0010 (config schema),
  ADR-T002/T003/T004 (last-good / timing / ingest pacing)
- Brief: [docs/research/multi-program.md](../research/multi-program.md)

## Context

The engine runs exactly **one** program: one `OutputClock` drives one composited NV12
canvas per tick, encoded once, fanned to N transports (invariant #7). That program is
owned by `Pipeline` (`crates/multiview-cli/src/pipeline.rs`): one `cadence`
(`pipeline.rs:228`), one `Arc<Layout>` (`:954`), one source-store map + `ingest_plans`
(`:425`), one `OutputClock`+`CompositorDrive`+`StreamEgress` built inline in
`drive_streaming` (`:951–1027`). The config root models one
`canvas`+`layout`+`sources`+`cells`+`outputs` (`crates/multiview-config/src/lib.rs:89–134`).
The control plane has a **global** `Start`/`Stop` (`command.rs:82–88`); realtime events
carry no program identity (`topic.rs:16`, `event.rs`).

The operator requires **N concurrent, independently start/stoppable programs**, each one
of: (A) multiview composite (today), (B) **guarded passthrough** — source→output that is
**packet-copy while the input is healthy** but **fails to BLACK or SMPTE-bars + 1 kHz tone**
on any disruption, (C) source→transcode→output. They run simultaneously and **must share
decoded sources**: efficiency is the product — a source used by k programs is decoded
**once**; a healthy passthrough decodes/encodes **nothing**.

**Bulletproof is non-negotiable (invariant #1).** A naive remux/packet-copy has no encoder
to fabricate frames, so it cannot meet the operator's fail-to-slate rule — it is therefore
**rejected as a floor**. The decision below makes passthrough invariant-#1-*preserving*
(not an exception) via a per-program degenerate clock + a pre-baked param-matched slate
spliced in on loss, with a capability ladder down to full transcode where a clean splice is
not expressible.

Verified enablers already in the code: `EngineRuntime` is per-instance (own
clock/seed/`ticks_emitted`/`StopSignal`, `runtime.rs:173–193,225`) and awaits only its own
pacer (`:405`) — no master clock exists. `TileStore` reads are lock-free/wait-free
multi-reader (`latest.rs:92`); its producer-liveness pattern (`tile.rs:281/293/331`) is the
watchdog primitive. `Supervisor`/`Actor`/`RestartPolicy` exist
(`supervisor.rs:165,181,61`). Remux primitives exist (`Muxer::add_stream_from_parameters`
= `avcodec_parameters_copy`, `mux.rs:136`; `Demuxer::read_packet`, `demux.rs:196`) and the
encoder-less `PacketMuxSink` (`sink.rs:1173`) + `PacketSource` trait (`sink.rs:125`) are the
slate-replay seam; but `Demuxer::open` opens bare (no `AVIOInterruptCB`/`rw_timeout`,
`demux.rs:111`) and `is_key`/`AV_PKT_FLAG_KEY` is **not** an IDR test — both are prerequisite
gaps (below). `multiview-hal::select_device` already gates VRAM + the NVENC session ceiling
and supports pinning (`select.rs:22–23,90,136,202`).

## Decision

**1. The `Program` abstraction.** A `Program` is one self-contained, independently
supervised output pipeline = one placement island (ADR-0017/0018). Config layer:
`ProgramSpec`, internally tagged (`#[serde(tag="kind")]`, `#[non_exhaustive]`, never
untagged). Passthrough carries a **`robustness_floor`** (the minimum failover the engine
must guarantee), **not** an `on_incompatible: Reject`:

```rust
pub enum ProgramSpec {
    Multiview  { canvas: Canvas, layout: Layout, cells: Vec<Cell>, overlays: Vec<Overlay>, outputs: Vec<Output> },
    Passthrough{ input_id: String, robustness_floor: RobustnessFloor, outputs: Vec<Output> },
    Transcode  { input_id: String, encode: TranscodeSpec, outputs: Vec<Output> },
}

/// The minimum failover behaviour the engine MUST guarantee for a passthrough.
/// The engine picks the CHEAPEST ladder rung whose splice is provably clean for
/// (transport, codec, param-stability, clean-IDR, framing); marginal cases fall
/// through to FullTranscode automatically.
pub enum RobustnessFloor {
    /// Operator's hard rule: any disruption fails to BLACK or SMPTE-bars + 1 kHz
    /// tone within `splice_threshold`. The DEFAULT — naive non-bulletproof relay
    /// is NOT selectable. Slate kind (Black | SmpteBars) + audio (Tone1k | Silence).
    #[default]
    SlateOnLoss { slate: SlateKind, audio: SlateAudio },
    /// Guaranteed sample-accurate click-free + always-bulletproof: forces
    /// FullTranscode regardless of remux availability.
    Transcode,
}
```

Engine layer: each program runs as a supervised actor; `MultiviewProgram` is today's
`Pipeline::drive_streaming` body refactored to own its pieces (a **move, not a rewrite**).
A/C share the existing `ProgramEncoder`→`StreamEgress`→`PacketMuxSink` tail; **B (guarded
passthrough) feeds the SAME encoder-less `PacketMuxSink` from a new `GuardedPacketSource`**
that emits either the copied input packets (healthy) or pre-baked slate packets (on loss) —
see §4. `PassthroughFallback { Reject | Transcode }` is **removed**.

**2. `ProgramSet` supervisor + per-program independent output clocks + strong isolation.**
`ProgramSet` (new, `multiview-engine`) owns the `ProgramId→ProgramHandle` map, one shared
read-only `Arc<dyn TimeSource>`, the `SourceRegistry`, and the `PlacementController`. Each
clocked program (A/C **and** B — a passthrough's clock is a degenerate, compositor-less
`EngineRuntime`, §4) owns an **independent `OutputClock`+`EngineRuntime`** with its own
seed/cadence/`StopSignal`, on its own `tokio` task. **No master clock** — only a shared
monotonic *reference*. `Program` implements `Actor` so the existing `Supervisor` restarts
a crashed program with capped backoff while siblings keep emitting. `start(spec)` admits +
spawns without touching other programs; `stop(id)` raises only that program's
`StopSignal`, drains its egress within `SINK_WEDGE_GRACE`, joins. **New CI chaos gate:**
wedge one program's egress, assert another's `ticks_emitted` keeps advancing on cadence
(invariants #1 + #10, per program).

**3. Decode-once-use-many via a shared ref-counted `SourceRegistry`.** Hoist the
per-source `TileStore` + `IngestPlan` out of `Pipeline` (`pipeline.rs:425`) into a
process-global `SourceRegistry` owned by `ProgramSet`, keyed by canonical source identity.
One ingest/decode actor per **physical** source publishes into one `TileStore<Nv12Image>`;
every A/C consumer holds an `Arc<TileStore>` clone and *samples* it (zero-contention,
`latest.rs:92`). Ref-counted: ingest starts on first reference, dies on last release.
Decode **once at the supremum requested resolution**; each consumer scales in-shader (#5/#6).

> **Implementation status (MP-2) — scope (PF4).** SLICE-1 ships the registry **per-`Pipeline`**
> (`pipeline.rs` `Pipeline::build` mints one `SourceRegistry::new()`), keyed by the **configured
> source id** (`SourceKey::from_canonical(source.id)`), *not* the physical/URL-canonical identity.
> So decode-once holds **within** a program at the configured-source granularity — every
> tile/consumer of one configured source shares its single decode — while two *distinct* configured
> ids pointing at the **same** URL still decode independently. The two dedup axes in the paragraph
> above are **future slices**, not yet shipped: (a) the **process-global `SourceRegistry` owned by
> `ProgramSet`** for *cross-program* sharing, and (b) **physical/URL-canonical keying** (a
> `StableStreamId`) so two configured ids at one location fold to one decode. Both are additive over
> this seam and change no public API; "one ingest/decode actor per **physical** source" describes
> that target, whereas SLICE-1's key is the configured source id.

> **Implementation status (MP-2).** The shipped `SourceRegistry` fixes the store/decode
> size at the **first** `acquire`: a later, larger `acquire` grows the recorded per-axis
> supremum **metadata** only (`requested_supremum`) and does **not** resize a live store.
> This is correct in practice because callers acquire the supremum up front —
> `Pipeline::build` computes the per-axis supremum across **all** binding cells before the
> first `acquire`, so the fixed size already equals the supremum. Sizing the decode at the
> supremum by *rebuilding* a live store on a later larger request applies only when the
> decode lifecycle itself is hoisted into the registry (a later milestone); until then the
> supremum metadata is the forward-looking record, not a live-resize trigger. The **winner** of
> a first-acquire is the sole inserter and fixes the size; a concurrent same-key **loser** shares
> the winner's store and folds only its requested size into the supremum **metadata** — never
> resizing the live store — exactly like a sequential later-larger `acquire`.
>
> **Decode-once (steady-state, self-correcting under a same-key race — PF6).** Both `acquire`
> **and** `acquire_store` run the first-reference `factory` **outside** the entries lock (inv #10),
> then re-check under the lock. So a rare concurrent **same-key first-acquire race** can transiently
> build **two** products — one wins insertion; the loser shares the winner's store and **discards**
> its own just-built product. Decode-once therefore holds in **steady state** and is
> **self-correcting** under the race, never a leaked decode — but it is **not** *exact*, not even on
> `acquire_store`:
> - **`acquire_store`** (MP-2 production, `actor: None`): the discarded product is only a **cheap
>   `TileStore` allocation** (an empty, never-published store — the loser drops its redundant
>   `Arc`). So `acquire_store` can transiently build **two stores** under a same-key race
>   (steady-state one), but **never** two decode actors.
> - **`acquire`** (the actor path, used once the decode lifecycle is hoisted — a later milestone):
>   the loser's redundant **decode actor** is stopped at once via `signal_and_detach` (PF1) —
>   `request_stop` then a `catch_unwind`-contained drop — so only the winner's decode stays
>   registered and the loser's detached decode thread exits.
>
> The load-bearing guarantee — **never two concurrent decode *actors* for one key** — holds on both
> paths; the transient duplicate is a discarded store allocation on the production path, a
> promptly-stopped actor on the actor path. (MP-2 production uses only `acquire_store`, so the
> worst-case transient is one discarded empty-store allocation, never a duplicate decode.)
>
> **Implementation status (MP-2) — bounded teardown off the hot path.** Last-release
> teardown of a source's decode actor runs off every hot path via a **fixed** worker pool
> (`TEARDOWN_WORKERS`) draining a **bounded** queue (`TEARDOWN_QUEUE_DEPTH`): a releasing
> `Drop` only *offers* the actor with a non-blocking `try_send`, so it never blocks, never
> spawns a thread, and never runs a decode-thread join in an async destructor (safety rule
> #4). The observable `pending_teardowns()` — reserved (queued **plus** in-flight) — is
> **hard-bounded at `TEARDOWN_CAPACITY` (queue + workers = D+K)** by a bounded-CAS
> **reservation** taken *before* the offer: at the ceiling a last-release **sheds without
> counting**, so no number of concurrent last-releases can push the observable past the bound
> (it never even transiently overshoots — no increment-then-undo). **Panic containment covers
> every actor the registry drops (RP1 + PF1):** the *worker* runs the graceful `shutdown()` under
> `catch_unwind`; a *shed* — on the **releasing engine thread**, not a worker — contains its
> `request_stop`+detach under its own `catch_unwind` (in `signal_and_detach`); and a **lost
> first-acquire race** discards its redundant just-built actor through that SAME `signal_and_detach`
> on the **acquiring** thread (PF1), never a bare `Drop`. So a panicking
> `request_stop`/`shutdown`/actor-`Drop` on ANY of these paths can neither shrink the fixed pool
> **nor unwind into the engine**, and no discarded actor leaks its decode (each is
> `request_stop`-signalled before it is dropped). A dedicated RAII `SlotGuard` owns the
> reservation's decrement as drop glue, so the observable is corrected **exactly once** whether a
> teardown completes, is shed, or unwinds. (A *double* panic — an actor panicking in `Drop` while
> `shutdown`/`request_stop` already unwinds — still aborts, a Rust fundamental the trait contract
> forbids.) The fixed pool is built
> **fail-closed**: each worker spawn is retried to absorb a transient EAGAIN so construction
> guarantees `TEARDOWN_WORKERS` in practice, and a persistent spawn failure is surfaced loudly
> (`tracing::error!`) — never a silent short pool (RP2). Every teardown — graceful **and** shed
> alike —
> **structurally signals decode termination first**: `SourceActor::request_stop(&self)` is a
> **required**, non-blocking trait method the pool calls before the graceful `shutdown()` join
> and before every shed's detach, so the wake-decode signal is compile-enforced on every
> implementor, not a `Drop` convention.
>
> The honest bound (PF4): the D+K `TEARDOWN_CAPACITY` ceiling bounds reserved teardown **slots**
> (queued **plus** in-flight offers), **not** the count of detached decode threads still winding
> down. A shed decode-thread's *exit* is as prompt as the implementor makes its decode I/O
> interruptible — the **required** `SourceActor::request_stop` MUST wake/interrupt that I/O
> **promptly** (a *forward* bounded-I/O contract; the pool cannot force-kill a wedged OS thread —
> Rust has no such primitive). A non-prompt `request_stop` can never push the slot observable past
> D+K (the reservation still caps it), but it *can* let detached decode threads linger — which the
> contract forbids. In MP-2 no real decode actor is wired yet: production
> (`Pipeline::build`) uses `acquire_store` (`actor: None` — decode stop/join still lives in the
> run's external `StopRegistry`), so only tests inject actors (incl. a thread-owning
> `ThreadedProbe` that proves a shed terminates its real thread via `request_stop`).
> **On the `actor: None` path (RP4, verified not a bug):** last-release removes the entry and
> drops the store `Arc`, but it does **not** stop the producer — the decode producer is stopped
> by the run's `StopRegistry`, by design. Because the `TileStore` is `Arc`-owned, a producer
> that publishes after the consumer's handle drops writes into a still-valid (merely orphaned)
> store — never a use-after-free — until `StopRegistry` stops it. This
> resolves the "design-note required" flag in *Consequences → Negative* below.
The registry exposes a **decoded** facet (A/C) and a lazy **coded-packet** facet (B) so a
passthrough-only source never spins a decoder; a source both passthroughed **and** tiled is
demuxed once, packets forked to the passthrough muxers **and** the decoder. **Added for B:**
at program start the passthrough's `GuardedPacketSource` (i) runs the **param probe** over
the facet's `Demuxer` to **pre-bake the param-matched slate once**, (ii) installs the
per-stream `PacketLiveness` watchdog atoms fed off the same packet tap (the fork point
Release-stamps `last_packet_at_ns`/`last_advancing_dts_at_ns` from the shared `TimeSource`),
and (iii) requires that `Demuxer` be opened with the new `AVIOInterruptCB`+`rw_timeout`.
Sources stay **top-level** in config.

**4. Passthrough = guarded passthrough: packet-copy on the healthy path, pre-baked
slate-splice on loss (invariant-#1-PRESERVING).** A healthy passthrough does **zero**
decode/composite/encode (steady-state cost == naive packet-copy). It is **no longer an
exception to #1.** Each passthrough is a **degenerate `EngineRuntime`** (the existing tick
loop, **no compositor**) ticking at the program's nominal cadence, gated solely on the one
wall-clock `pacer.wait_until` `.await` — **never on the demuxer**. The byte copy runs on a
**separate** source-paced egress thread (`Demuxer::read_packet`). A passthrough is thus
**source-paced for DATA, clock-paced for its LIVENESS DECISION**.

Each tick the watchdog runs a pure `classify(elapsed, thresholds)` (reuse
`multiview-framestore::state::classify`, `state.rs:126`) over an `AtomicI64 last_packet_at_ns`
the copy thread Release-stamps **from the shared monotonic `TimeSource` (arrival instant,
never the packet PTS)** — producer-liveness, not picture-age (nothing is decoded). The clock
thread decides **copy-vs-splice** and flips an `AtomicU8` mode; the egress thread Acquire-reads
it to choose source-vs-slate bytes. The single cross-thread coupling is wait-free
Release/Acquire atomics; a stale Acquire read **fails safe** (older stamp → larger elapsed →
biases toward splice, never false-LIVE). **#1 holds per program:** the degenerate clock emits
a valid coded access unit — a source packet **or** a pre-baked IDR-led slate AU — every tick
interval, forever, and never `.await`s an input.

- **Pre-bake-once slate.** At program start, probe the input's coded params via
  `Demuxer::streams()`/`stream_parameters()`: codec, W/H, profile/level, chroma/bit-depth, the
  **actual SPS/PPS/VPS** (H.264/HEVC) or `sequence_header` OBU (AV1), the stream time-base, the
  **extradata framing** (avcC/hvcC vs Annex-B), and audio config (AAC AOT etc.). Encode **once**
  (a sub-second static transcode, encoder then **released — no held session**) a short
  **IDR-led, closed-GOP, B-free** (DTS==PTS) loop of black/SMPTE-bars video + an integer number
  of AAC AUs of 1 kHz tone / silence, in **bit-identical** params and the same encoder time-base
  `add_stream_from_parameters` registers; the slate's active SPS/PPS/VPS bytes are **copied from
  the probed input's extradata** and re-emitted **in-band** before the slate IDR. Cache as a
  shared `Arc<[EncodedPacket]>`. **Memory is O(1) in outage length** (looped by integer offset);
  bake with **CRF/CQ** (a few hundred KB even at 4K, < 5 MB — a CBR-padded bake of a 25–80 Mbps
  source would breach that, so CRF/CQ is mandatory). The slate is **inert bytes** replayed
  through the **encoder-less `PacketMuxSink`**, so failover costs **zero** of the NVENC
  concurrent-session ceiling `select.rs:202` gates.
- **The watchdog — three signals, all on the clock tick.** Two `PacketLiveness` instances
  (**video + audio tracked independently** — per-PID streams fail independently): (1) **hard
  death** (`read_packet` Eof/error, or `elapsed ≥ splice_threshold`); (2) **slow-loris/stutter**
  (packets arriving but gaps exceed `hold`: rides LIVE with widened tolerance, never splices
  pre-threshold — no slate-flapping); (3) **stalled PTS** (bytes flowing but max-seen DTS not
  advancing: `last_advancing_dts_at_ns` fires at `pts_stall = video splice + 2·T`). Splice =
  `video_loss OR audio_loss`. Thresholds tied to `T=1/fps` and segment duration `Sd`:
  **STALE@2·T, SPLICE@max(4·T,150 ms), NO_SIGNAL@max(2·Sd,3 s)**; audio splice ~max(4
  audio-frames, 100 ms).
- **Blocking-read prerequisite (real gap).** `Demuxer::open` opens bare. The watchdog DECISION
  is independent of the read returning (it fires on the clock, so failover is never blocked),
  but RECOVERY needs the wedged demuxer torn down → `Demuxer` must grow an injected
  `AVIOInterruptCB` (nonzero past STALE) + `rw_timeout`. **Prerequisite, not optional.**
- **The splice seam — `GuardedPacketSource: PacketSource`.** Sole producer feeding
  `PacketMuxSink::run_av`; wraps the demuxer (live) + slate ring (failover); runs the pull +
  `muxer.write_packet` on a **dedicated egress thread, NEVER the watchdog/clock thread**, wrapped
  in the existing **drop-oldest + `SINK_WEDGE_GRACE` detach** posture (a wedged push peer is shed,
  never back-pressures the producer).
  - **Re-stamp rule (#3 for the copy path).** NEVER `out_pts=f(tick)` (destroys B-frame DTS
    reorder). One persistent monotonic-clamp accumulator **per stream** across BOTH boundaries:
    `dts' = max(raw_dts+offset, last_dts+1)`; `pts' = max(raw_pts+offset, dts')`. **The clamp
    (`last_dts+1`) — not any FFmpeg flag — is what prevents `av_interleaved_write_frame`'s
    non-monotonic-DTS abort** (mp4/mov abort on *equal* DTS; mpegts is non-strict). Only `offset`
    changes at a boundary; raw deltas pass through (durations/reorder preserved); exact rationals.
    `avoid_negative_ts=make_zero` is a **one-shot leading shift** (not a mid-stream fix);
    `max_interleave_delta` is an **interleave-flush** knob, **not** an abort guard (don't set it
    "small"). Pathological raw timestamps → drop down the ladder, don't clamp-corrupt.
  - **Boundary 1 (input→slate).** `offset=(last_dts+1)−slate_first_dts`; slate's first VIDEO
    packet is its closed-GOP IDR. Slate loop **paced to the output clock**, advancing `offset` by
    one loop-duration per wrap (PCR/baseMediaDecodeTime stay honest, never flush-to-catch-up).
  - **Boundary 2 (slate→input recovery).** Supervised reconnect (capped-exp + jitter); discard
    every input packet until a **TRUE input VIDEO IDR** — **NOT** `is_key`/`AV_PKT_FLAG_KEY`
    (FFmpeg sets it for HEVC **CRA/open-GOP** and H.264 **recovery-point-SEI** frames, whose
    leading pictures reference now-absent frames → garbage seam). Re-anchor only on a strict RAP
    via a cheap header classifier: H.264 `nal==5`; HEVC `IDR_W_RADL(19)|IDR_N_LP(20)` (reject
    `CRA(21)/BLA`); AV1 `KEY_FRAME && show_frame==1 && !show_existing_frame && temporal_id==0 &&
    seq-hdr OBU present`. Finish the in-flight slate GOP first, then re-anchor `offset`.
    **Audio is video-clean / soft-step, not pop-free** (coded-domain AAC IMDCT/TDAC seam transient
    is uncancellable → ≥2 leading silence AUs + AU-granular alignment; sample-accurate click-free
    ⇒ the transcode floor).
- **Framing prerequisite (real gap).** The slate side and the input-copied side **must** reach
  the muxer in **identical framing**; for TS/SRT/RTP/raw run `dump_extra(freq=keyframes)` /
  `extract_extradata` so the active PS repeat **in-band before BOTH** the slate IDR and the
  recovery IDR (`h264_mp4toannexb` alone inserts PS only at stream-start). No such BSF stage
  exists in the output path today.
- **No clean IDR / mid-stream param change.** If the input has no detectable IDR (intra-refresh
  /GDR/open-GOP-CRA) or a mid-stream param change invalidates the cached slate (per-AU in-band PS
  diff; the container snapshot doesn't update mid-stream), the matched-slate-splice rung is
  unavailable → **drop to rung 2 (container-discontinuity)** and **background async re-bake** off
  the failover hot path.

### The robustness ladder (cheapest valid rung first)

Each program declares a `robustness_floor`; the engine picks the **cheapest rung whose splice
is provably clean** for `(transport, codec, param-stability, input-provides-clean-IDR,
extradata-framing)`. **Rung 0 (naive packet-copy) is the steady-state cost reference, NOT a
floor** — on loss it has no frames → black/`NO_SIGNAL` → violates the operator rule.

1. **`matched-slate-splice` (cheapest valid floor).** Pre-baked, bit-identical, IDR-led
   closed-GOP slate spliced in-place into the copied ES at a true input IDR, re-stamped via the
   monotonic clamp+offset. Steady-state cost == naive packet-copy (0 decode/encode, 0 NVENC
   session). Available where slate params match **and** PS framing is in-band: **MPEG-TS / SRT /
   RTSP-RTP / RTMP-FLV (matched)** and **HLS/LL-HLS fMP4 reusing the same `EXT-X-MAP` (matched)**.
   Splices **silently** (no markup) on those.
2. **`container-discontinuity`.** Segment/container-boundary splice carrying a possibly-fresh
   param set via the transport's primitive — **HLS/LL-HLS:** new `EXT-X-MAP` + `EXT-X-DISCONTINUITY`
   (bump `DISCONTINUITY-SEQUENCE`, already auto-bumped `hls/media.rs:327`); **TS/SRT:**
   `discontinuity_indicator=1` + PMT `version_number++` + per-PID CC + PCR re-base. Still **zero
   live encode** (replays the pre-baked slate); pays a correctly-signalled decoder re-init.
   **HLS fresh-param needs a per-segment `EXT-X-MAP` override — `map_uri` is one playlist-level
   `String` today (`hls/media.rs:134/234`), an unwired gap; until landed, fresh-param HLS drops to
   rung 3.**
3. **`full-transcode` (always-bulletproof floor).** Decode last-good + live-re-encode the slate at
   negotiated params, holding an encoder session (the **only** rung that pays continuous
   decode+encode and session-accounts). Auto-selected when input params are unstable, the transport
   can't express the required discontinuity (RTMP/FLV codec change; legacy RTSP SDP-`sprop`),
   codec/container is incompatible at the build-time gate (HEVC→baseline FLV), or the floor demands
   guaranteed click-free audio. Class-2 / make-before-break.

**Selector rule:** marginal cases (segmented transport + frequent flaps, unstable params, no clean
IDR, framing mismatch, must-transcode-anyway) **fall through to rung 3 automatically**. The UI
surfaces "will splice (remux)" vs "will splice (discontinuity)" vs "will transcode" — plus
"video-clean / audio soft-step" — as the per-program efficiency-and-robustness contract badge.

**5. Per-program + global resource-adaptive degradation with admission control, tied to
ADR-0017/0018 placement.** Admission runs over the **union** of programs, crediting shared
decode **once** (the `SourceRegistry` emits one de-duplicated decode `TileLoad`/source at
its supremum resolution; per-program composite+encode add on top), fed to `Planner::admit`
(`planner.rs:181`) + `select_device` (VRAM + NVENC session gates, `select.rs:90,202`). Extend
placement with a **same-source co-location** affinity term. A guarded passthrough admits at
**0 Mpix/s and consumes 0 NVENC sessions at rungs 1–2** (slate replay is encoder-less); the
engine **never holds a warm encode session for failover** (the slate is pre-baked inert bytes —
a held session would increment the very `select.rs:202` ceiling for zero benefit). It credits a
**fixed slate-memory term** (`Arc<[EncodedPacket]>`, < 5 MB, O(1) in outage length) + the host-IO
copy term; only a program whose floor resolves to **rung 3** counts an NVENC session +
decode/encode Mpix/s (like kind C). Degradation is cross-program, cheapest-impact-first with
operator-assignable per-program `priority` (default: shed transcode/rung-3 passthrough before the
primary multiview); per-program queues stay bounded `DropOnOverload`. All programs run in **one
process** (CUDA context paid 1×).

**6. `[[programs]]` config (adjacently/internally-tagged, backward-compatible).** Add
`#[serde(default)] programs: Vec<Program>` to `MultiviewConfig`. The legacy top-level
`canvas`/`layout`/`cells`/`overlays`/`outputs` desugars to one implicit `Multiview`
program (`id="main"`); `into_programs()` synthesizes it; `validate()` rejects populating
both. `schema_version`→2 (v1 via the legacy path). New checks: unique program ids; unique
output labels across all programs; every `input_id` resolves to a `sources` entry.

**7. Programs management API + per-program realtime state.** New `PROGRAM_KIND` resource +
`routes/programs.rs` (`GET/POST/PUT/DELETE /api/v1/programs[/{id}]`, ETag/If-Match→412,
Idempotency-Key, RFC 9457). Lifecycle: `POST /api/v1/programs/{id}/{start|stop|apply}`
(202 + op-id via `submit_accepted`) and `POST /api/v1/programs/{id}/plan` (returns the
Class-1/Class-2 classification **and the selected robustness rung**, invariant #11). New
`Command` variants `StartProgram`/`StopProgram`/`ApplyProgram`; `SwapSource`/`ApplyLayout`
gain `program: String`; legacy global `Start`/`Stop` become aliases for `"main"`. Realtime:
new `Topic::Programs` + `program.state` event (`ProgramState{state,kind,outputs,fps,rung}`,
`ProgramRunState` reusing `OutputRunState` incl. `Migrating`); scope `OutputStatus`/`TileState`
by `Envelope.id="{program}/{output}"`; snapshot + `CorrKey` extend.

## Consequences

**Positive.** Decode/demux/CUDA-context/VRAM cost for a source used by k programs drops
from O(k) to O(1); the heavy per-program primitives (`EngineRuntime`, `OutputClock`,
`CompositorDrive`, `StreamEgress`, `Supervisor`, `select_device`, `PacketMuxSink`,
`PacketSource`) are **reused unchanged** — the engine was authored per-program. Per-program
clocks make #1/#10 structural for **all three kinds**. **Guarded passthrough now satisfies the
operator's BLACK/SMPTE+1 kHz rule and #1 while preserving naive-packet-copy steady-state cost —
it is no longer a documented #1 exception**, so nobody can "fix" it into a hidden transcode (that
anti-pattern is replaced by the explicit ladder); the slate is encoded **once** (#7) and failover
holds **zero** scarce NVENC sessions at rungs 1–2. WebUI manages each program independently.

**Negative / cost.** New shared mutable lifecycle (the `SourceRegistry` refcount) must stay off
every program's hot path (an `Arc`-drop returning a pooled buffer must not run in an async
destructor — safety rule #4; design-note required). **Three prerequisite code gaps must land
before guarded passthrough is safe, not optional:** (a) `Demuxer` `AVIOInterruptCB`+`rw_timeout`
(recovery teardown); (b) a strict-IDR header classifier (`is_idr`, distinct from `is_key`, which
mis-splices CRA/recovery-point); (c) a `dump_extra(freq=keyframes)` / Annex-B framing BSF stage on
the output path so PS repeat in-band at both seams. Two further: per-AU in-band PS diffing (the
container snapshot doesn't update mid-stream) feeding background re-bake; and a per-segment
`EXT-X-MAP` override in `hls/media.rs` before fresh-param HLS is selectable. The egress
pull+`write_packet` **must** run on a dedicated thread wrapped in drop-oldest + `SINK_WEDGE_GRACE`
detach, or a wedged push peer blocks `write_packet` and stalls the watchdog — so the CI chaos gate
tests **two** vectors (ingress stall AND egress-peer wedge). Audio remux failover is **soft-step,
not click-free** (documented contract); the `< 5 MB` slate figure holds **only** with CRF/CQ bake.
The supremum-resolution decode rule needs cost-model confirmation; admission must credit shared
decode once or it over-rejects; `PipelineError` must grow `ProgramId` context.

## Alternatives rejected

- **One master clock for all programs.** Rejected: cannot serve 25 + 60 fps without
  resampling (violates #3 exact-rational), and creates a shared stall surface (violates
  #1/#10). Chosen: shared *time source*, independent *clocks*.
- **Process-per-program.** Rejected: multiplies the ~84–115 MB CUDA context and forbids
  zero-copy surface sharing; cross-program decode-once becomes impossible. Chosen:
  one process, in-engine `ProgramSet`.
- **Sources nested per-program.** Rejected: forces per-program decode, defeating the
  operator directive. Chosen: top-level shared sources referenced by `input_id`.
- **Naive non-bulletproof relay (rung 0 as a floor).** Rejected: on loss it has no frames →
  black/`NO_SIGNAL` → **violates the operator's hard rule**. Kept only as the steady-state cost
  reference; the default `RobustnessFloor` is `SlateOnLoss`, not selectable-away.
- **"Always transcode" (make every passthrough kind C).** Rejected: pays continuous decode+encode
  24/7 and **burns a scarce per-system NVENC session** (≤8 on consumer GeForce) for a feed that is
  byte-clean whenever healthy. Transcode is the **floor of last resort**, auto-selected only when a
  clean remux splice is unexpressible.
- **Hold a warm encode session for instant failover.** Rejected: the slate is pre-baked inert bytes
  replayed encoder-lessly through `PacketMuxSink`; a held session would consume a gated
  `select.rs:202` slot + encoder VRAM for **zero** benefit (no cold-start to mask).
- **`out_pts = f(tick)` restamp for the copy path.** Rejected: collapses B-frame DTS reorder. The
  copy path's #3 obligation is the **monotonic clamp + per-boundary offset** (exact rationals),
  distinct from the encoder-path tick-restamp.
- **Gate recovery on `AV_PKT_FLAG_KEY` / `is_key`.** Rejected: FFmpeg flags HEVC CRA and H.264
  recovery-point I-frames as "key"; re-anchoring there decodes garbage at the seam. Gate on the
  strict-IDR classifier (`is_idr`); on no-IDR inputs descend the ladder.
- **Set `max_interleave_delta` "small" / rely on `avoid_negative_ts` for the abort.** Rejected: the
  abort guard is the DTS clamp; `make_zero` is a one-shot leading shift; a small interleave delta
  forces premature flushing and degrades A/V interleave at the seam.
- **Old §4 "honest `NO_SIGNAL`+reconnect by default, slate opt-in".** **Superseded:** the operator
  requires fail-to-slate; the degenerate-clock + pre-baked-slate design delivers it at packet-copy
  steady-state cost, so slate-on-loss is now the default floor.
- **Outputs-as-programs / `dyn Any` heterogeneous list.** Rejected (B/C are different pipeline
  shapes, not new sinks; the guardrail bans `dyn Any`); use the tagged `ProgramSpec` enum + an
  `Actor`/enum-dispatch runner.
