# ADR-0029 — ACME/TLS for the control plane: DNS-01 only, rustls, pluggable DNS providers

- **Status:** Proposed
- **Date:** 2026-06-07
- **Area:** W (web/API stack) — `multiview-control`
- **Relates to:** invariant #10 (isolation), safety rule #2 (no back-pressure on the engine),
  [`conventions.md` §7 (licensing)](../architecture/conventions.md), brief [acme-tls](../research/acme-tls.md)

## Context

The control plane (`multiview-control`) serves axum REST + WebSocket + SSE + the embedded SPA + OpenAPI
over **plain HTTP** (`serve()` over a pre-bound `tokio::net::TcpListener`, bound by `multiview run`). We
need automatic TLS with certificate automation. Constraints:

1. The host may have `:80`/`:443` unreachable (NAT, firewall, ports owned by program output), and we
   need **wildcard** certs → the ACME challenge must be **DNS-01**; HTTP-01 and TLS-ALPN-01 are unusable.
2. The default build must stay **LGPL-clean** and every dep must satisfy the cargo-deny allowlist
   (**MIT / Apache-2.0 class only**, `deny.toml:40-52`) → **rustls, never openssl**.
3. **Invariant #10:** TLS handshakes, ACME ordering, and renewal must run **off** the engine/output path
   and must **never** block or back-pressure the output clock. A renewal failure must keep serving the
   existing cert, never stall the engine.
4. tokio async throughout; opt-in via config + an off-by-default Cargo feature; secrets (the Cloudflare
   API token) via 1Password/env, least-privilege, never committed.

## Decision

Implement **ACME (RFC 8555) with the DNS-01 challenge ONLY** (HTTP-01 and TLS-ALPN-01 are forbidden),
on a **rustls** stack with **no openssl**, behind an off-by-default feature, with a **pluggable async
`DnsProvider` trait** whose **first (currently only) implementation is Cloudflare**.

**Exact crate choices + licenses (all on the allowlist, all rustls/no-openssl):**

| Role | Crate | Version | License |
|---|---|---|---|
| ACME client (RFC 8555, DNS-01) | **`instant-acme`** | **0.8.5** | **Apache-2.0** |
| TLS termination + hot cert reload | **`axum-server`** (`tls-rustls` feature) | latest 0.x | **MIT** (note: MIT-only, not dual) |
| TLS implementation (transitive) | **`rustls`** | **0.23** | Apache-2.0 / MIT / ISC |
| Cloudflare DNS HTTP client | **`reqwest`** (`default-features=false`, `["json","rustls-tls","http2"]`) | latest 0.x | MIT / Apache-2.0 |
| CSR / cert keypair | **`rcgen`** | latest 0.x | MIT / Apache-2.0 |
| Async trait objects | **`async-trait`** | latest 0.x | MIT / Apache-2.0 |

Concrete elements of the decision:

- **DNS-01 only.** `ChallengeType::Dns01`; publish `_acme-challenge.<id>` TXT =
  `KeyAuthorization::dns_value()` (instant-acme computes the digest). Wildcards supported.
- **Reject `rustls-acme`/`tokio-rustls-acme`** — TLS-ALPN-01/HTTP-01 only, no DNS-01. Reject
  `acme2`/`acme-lib` — openssl and/or sync and/or stale. Reject the **`cloudflare` crate
  (BSD-3-Clause + native-tls→openssl)** — hand-write the 3-endpoint client over `reqwest`+rustls so the
  allowlist is untouched.
- **Pluggable `DnsProvider` trait** (`async-trait`, object-safe, `Box<dyn DnsProvider>` selected from
  config): `create_txt_record` → opaque `TxtRecordHandle` newtype (**not `dyn Any`**); idempotent
  `delete_txt_record`; `wait_for_propagation(deadline)` default no-op (Cloudflare overrides it to poll
  **authoritative** NS). Route53 / Google Cloud DNS / RFC2136 drop in as one `impl` + one config variant.
- **Cloudflare first provider:** Bearer token, `POST/DELETE /zones/{zone_id}/dns_records`, `ttl:60`,
  **least-privilege token = Zone.DNS:Edit on a single zone**, `zone_id` pinned in config to avoid
  needing `Zone:Read`. Token via env/1Password, never inline.
- **Hot-reloadable certs:** serve via `axum_server::from_tcp_rustls` over the existing bound listener;
  install renewed certs with `RustlsConfig::reload_from_pem` (atomic arc-swap — new handshakes get the
  new cert, in-flight connections finish on the old, **no listener restart**).
- **Renewal = detached `tokio::spawn` background task** owned by the control plane, renewing at
  **⅓-remaining-lifetime + jitter**, **fail-soft** (log + telemetry + bounded backoff, keep serving the
  existing cert), structurally a sibling of the serve task — **cannot violate invariant #10**. Persist
  the ACME `AccountCredentials` (`0600`) and reuse across restarts; never re-register.
- **Tiered delivery:** Tier-0 `mode="static"` (operator-supplied cert, no ACME, no network) ships first
  to de-risk the rustls serving glue; Tier-1 `mode="acme"` adds issuance/renewal + provider trait,
  reusing the same hot-swap path.
- **Opt-in:** config `[control.tls]` (absent ⇒ plain HTTP, today's behaviour) **and** off-by-default
  Cargo features (`tls`, `acme`) on `multiview-control`, wired into `multiview-cli` aggregate flags.
  Directory URL config-driven, **default staging** in non-prod.
- **Operator prerequisites** (documented, not code): CAA pinning LE + RFC 8657 `accounturi`/
  `validationmethods=dns-01`; CT-log monitoring; writable uid-10001 state volume; staging-first promotion.

## Consequences

**Positive.** TLS with no inbound reachability and wildcard support; default build unchanged
(plain-HTTP, openssl-free, LGPL-clean, allowlist green); the engine/output path is untouched and
provably isolated; new DNS providers are additive (one `impl` + one `#[non_exhaustive]` config variant);
cert rotation is hot (no dropped connections / no restart); minimal new attack surface (single-zone
token, CAA-pinned issuance).

**Negative / costs.** Hand-written Cloudflare client (3 endpoints + a response envelope) to maintain
instead of a vendored SDK — accepted to keep the allowlist MIT/Apache-only. `axum-server` is **MIT-only**
(narrows the relicensing surface of that one dep) — accepted, it's on the allowlist. DNS-01 introduces a
zone-write credential whose compromise enables mis-issuance — mitigated by least-privilege token + CAA +
CT-log monitoring. Operators must publish CAA and provision a writable state volume. Let's Encrypt **5
duplicate-certs/week** limit means the renewal scheduler must be idempotent and threshold-based (renew on
remaining lifetime, never on every boot). `instant-acme` 0.8.x method signatures have churned — pin
0.8.5 and re-verify at wiring time.

**Neutral.** ARI (RFC 9773) and EAB (for non-LE CAs) are designed-for (optional config) but not shipped
in the first cut.

## Alternatives rejected

- **HTTP-01 / TLS-ALPN-01 challenges** — require inbound `:80`/`:443` (fail behind NAT / closed ports)
  and cannot issue wildcards. Forbidden by requirements.
- **`rustls-acme` / `tokio-rustls-acme`** — rustls-native and recent, but implement **only**
  TLS-ALPN-01/HTTP-01; no DNS-01 path. Disqualified.
- **`acme2` (openssl, 2022) / `acme-lib` (sync + openssl)** — violate no-openssl and/or async-throughout;
  stale.
- **openssl / native-tls backends** — banned (LGPL-clean default + allowlist).
- **Official `cloudflare` crate** — BSD-3-Clause (would widen the allowlist) and defaults to
  native-tls→openssl; oversized surface for 3 endpoints. Hand-write over `reqwest`+rustls instead.
- **DNS provider as `dyn Any` / stringly-typed handle** — banned by repo guardrail; use a concrete
  `TxtRecordHandle` newtype + `#[non_exhaustive]` enum config.
- **Wildcard-by-default certs** — broader blast radius if the host is compromised; prefer SAN/explicit-host
  certs, wildcard only when hostnames are dynamic and justified.
- **Renewal inline / sharing engine locks** — would risk invariant #10; renewal is a detached, fail-soft
  background task.
