# ADR-0093: Typed track/stream selection replacing free-text

- **Status:** Proposed
- **Area:** Config / Control / Web (operability; also a live defect)
- **Date:** 2026-06-13
- **Source brief:** [webui-operability-gaps.md §5](../research/webui-operability-gaps.md)
- **Extends:** [ADR-M004](ADR-M004.md) (Source owns track attributes; the routing-matrix UI greys
  impossible cells), [ADR-0036](ADR-0036.md) (the precedent: a typed catalog + capability-gated
  dropdown replacing a free-text field), [decoupled-routing.md §3/§4](../research/decoupled-routing.md)
  (`StreamInventory` + `StreamRef`)
- **Relates to:** [ADR-R005](ADR-R005.md) (per-input audio routing), [ADR-0019](ADR-0019.md)
  (`CaptionSelector`), [current-defects-2026-06-13.md §1.5](../development/current-defects-2026-06-13.md)

## Context

Tracks appear in the SPA as a **free-text list**, not a typed selectable control: `captionsTrack` and
`audioTracks` are raw strings (`web/src/resources/forms.ts:423`, `:1032`, `parseTrackList`). The typed
*destinations* exist in config — `CaptionSelector` (`crates/multiview-config/src/schema.rs:476`),
`SubtitleCrosspoint` and the independently-keyed audio/subtitle routing maps
(`crates/multiview-config/src/routing.rs`) — and the demuxer enumerates every elementary stream
(`crates/multiview-ffmpeg/src/demux.rs:301`, language at `:318`). But the input path **discards** the
non-video rows ([decoupled-routing.md §2](../research/decoupled-routing.md)), so **no discovered
inventory reaches the UI**, which therefore degrades to free text. This is both a feature gap and a live
defect ([current-defects §1.5](../development/current-defects-2026-06-13.md)). It is the exact situation
[ADR-0036](ADR-0036.md) fixed for the free-text codec field.

## Decision

Surface [decoupled-routing.md](../research/decoupled-routing.md)'s `StreamInventory` read-only over the
API and feed a typed selector — the ADR-0036 pattern applied to tracks. This ADR **consumes** the
inventory; generalising the libav/TS/HLS discovery to stop discarding rows is decoupled-routing's work.

1. **Expose `GET /api/v1/inputs/{id}/streams` → `StreamInventory`** — a pure read off the ingest
   actor's demuxer, off-engine (inv #10), with `input.streams` deltas on re-probe / PMT-version bump
   ([decoupled-routing.md §9](../research/decoupled-routing.md)). Each `InventoryStream` carries a
   `StableStreamId` (kind-scoped: TS=PID, HLS=`group_id+name`, soft-key fallback), `kind`, codec,
   `language: Option<Bcp47>`, `default`, and a **key-stability tier** (hard/soft) so a soft-keyed
   selection is an operator-visible reorder risk. The HLS `group_id+name` example is not unique on its
   own across renditions, duplicate names, or language variants and is exposed to playlist churn: the
   key must include enough fields (URI / group / type / name / language) plus the stability tier — or an
   explicit collision-disambiguation strategy — to be unique within an input. The exact key composition
   and tie-break are decoupled-routing's to own; this ADR requires uniqueness within an input and the
   visible stability tier.
2. **A typed `<TrackSelect>`** (built on the existing `KindSelect` generic + shadcn `Select`,
   [conventions §8](../architecture/conventions.md)) fed by a `useStreamInventory(inputId)` React-Query
   hook, rendering `CAM-A · A2 · spa · 5.1`-style options. **Impossible cells are greyed** by the
   [ADR-M004](ADR-M004.md) carrier capability matrix (HLS=select-one, NDI=channel-map,
   RTMP=single-program), each with a reason tooltip (the ADR-0036 discipline). It replaces the free-text
   `captionsTrack`/`audioTracks` fields.
3. **Selectors stay typed end-to-end.** The SPA emits the typed `CaptionSelector::Track { id }` /
   audio crosspoint (`StreamRef { input_id, kind, selector }`,
   [decoupled-routing.md §4](../research/decoupled-routing.md)), never a raw string the server must
   re-parse. A free-text entry survives **only** as an explicit "advanced / not-yet-probed" escape with
   a clear warning, mirroring ADR-0036's reason-tooltip escape.

## Consequences

- **Positive.** Operators pick tracks from the *discovered* set with language/codec/layout/label, and
  impossible carriers are greyed before a take — the silent-misroute and free-text-typo classes vanish.
  One inventory feeds config validator, API, and UI ([ADR-M007 single-source discipline, via ADR-0036]).
- **Negative / cost.** Depends on decoupled-routing surfacing `StreamInventory` (stop discarding rows);
  soft-keyed streams carry a reorder risk that must be shown, not hidden. Whether the free-text escape
  is ever removed per source-kind is open (brief Open question 5).
- **Neutral.** Inventory is computed once at open and on re-probe (it already is — the fix is to not
  throw it away); the API read is a cached snapshot; presence is structural (from demux enumeration),
  so no decode is added.

## Alternatives considered

- **Keep free text + a regex/string allowlist.** Rejected per [ADR-0036](ADR-0036.md): still no
  discovered set, no capability greying, stringly-typed — the exact bug class being removed.
- **A bespoke per-input track endpoint unrelated to `StreamInventory`.** Rejected: would create a second
  discovery surface that can drift from the routing model; reuse the one typed inventory.
- **Front-end-only widget over the existing string field.** Rejected: without a discovered set the
  widget has nothing typed to offer; the gap is the discovery surface, not only the widget.

## How it honors the invariants

- **#1 (output clock).** Inventory and the selector are control-side reads/writes; a selector change
  rides the live subtitle/audio re-point seams (Class-1, [ADR-0094](ADR-0094.md)/[ADR-0059](ADR-0059.md))
  at the frame boundary, never on the clock loop. Discovery adds no decode and no per-tick cost.
- **#10 (no back-pressure).** `GET /inputs/{id}/streams` is a wait-free read off the ingest actor's
  snapshot; `input.streams` rides the existing drop-oldest broadcast; no new engine-inward channel.
