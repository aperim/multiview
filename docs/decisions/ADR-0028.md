# ADR-0028 — Native NDI FFI binding data design (the safe layer over the v6 table)

**Status:** Accepted (2026-06-07)
**Area:** I/O / FFI — native NDI (NewTek/Vizrt) send + receive
**Supersedes/extends:** [ADR-0008](ADR-0008.md) (runtime-load, never-vendor). Research:
[ndi-integration](../research/ndi-integration.md). Design fan-out: workflow `wdqyo851f`
(4 facets: table-access, frame ownership/RAII, safe boundary, bindgen/build config).

## Context

`NDIlib_v6_load()` returns a pointer to a versioned, append-only function table
(`NDIlib_v6` is a `typedef` alias of `NDIlib_v6_3`). `bindgen` renders the first
~107 table slots as **anonymous unions** (`__bindgen_anon_N`), each carrying one
function pointer under **two** members — the short name and its `NDIlib_`-prefixed
deprecated alias (the header's `NDIlib_DEPRECATED` macro). So a call today reads
e.g. `table.__bindgen_anon_3.version` — fragile (the index is a bindgen artifact
that renumbers if the header inserts a deprecated wrapper) and `unsafe`.

This is **intrinsic to the NDI header**, not a Multiview quirk (confirmed identical
in `ndi-sys`). There is **no bindgen flag** that collapses a two-member same-type
anonymous union into one named field (blocklist/allowlist operate on whole items
only). The mature open-source bindings (`grafton-ndi`, `deno-plc/rust-ndi-sdk`)
sidestep it by **static-linking** `libndi` and calling global symbols — exactly
the path ADR-0008 forbids us (runtime `dlopen` only, never linked/vendored). The
NDI **Advanced** SDK v6.3.2.0 is licensed to the operator and the bindgen'd ABI is
already **proven on real hardware** (`multiview-ndi-sys` `tests/live_load.rs` read
`6.3.2.0` through the table on gpu-test-box). We design the safe layer **before**
building send/recv, so it is deliberate and stable across SDK versions.

Invariants in scope: #1 (output clock never stalls; NDI never paces us), #3
(timecode from the tick), #5 (NV12-throughout), #10 (ingest/egress never
back-pressures the engine). Guardrail: **all `unsafe` stays in `multiview-ndi-sys`**;
`multiview-output`/`-input` are `forbid(unsafe_code)`.

## Decision

### 1. Table access — resolve once into a flat `NdiV6`
At load, read each needed `__bindgen_anon_N` member **exactly once**, `NonNull`-check
it, and store the bare `unsafe extern "C" fn(...)` value in a **flat, hand-named,
stable struct** `NdiV6 { version, find_create_v2, send_create, send_send_video_async_v2,
recv_create_v3, recv_capture_v3, recv_free_video_v2, … }`. The `__bindgen_anon_N`
indices are confined to this one resolver function; **no other code in the workspace
ever names a bindgen anon field.** The ABI stays **bindgen-derived** from the licensed
header (no hand-fabricated layout → ADR-0008-clean), and because v6_3 is append-only,
SDK bumps add bindgen fields without renumbering what we read. *(Rejected: scattering
`__bindgen_anon_N` accessors — the fragility the lead flagged; a hand-written `#[repr(C)]`
mirror — reproduces the proprietary layout by hand, UB-prone, highest maintenance; a
`#undef NDIlib_DEPRECATED` wrapper header — reaches into proprietary-macro internals.)*

### 2. Frame ownership / lifetimes (RAII, type-enforced)
- **Send (async-buffer pinning, the #1 NDI mis-use):** `NDIlib_send_send_video_async_v2`
  requires the host buffer stay valid until the **next** async send or destroy.
  Model it as a **typestate**: a `SendVideoFrame<'pool>` borrows from the **per-device
  frame pool allocated at start** (safety rule #5 / the existing double-buffer), and
  the sender holds an `AsyncSendGuard` such that submitting frame N+1 (or `Drop`)
  is the **only** way to release frame N — the borrow checker makes the contract
  unviolatable. `Drop` sends a **NULL frame** (the flush idiom) before
  `NDIlib_send_destroy`. `clock_video/audio = false` (NDI is a pure sink; our tick is
  the sole clock — inv #1/#3).
- **Receive (free-exactly-once):** `NDIlib_recv_capture_v3` returns **SDK-owned**
  frames freed via `NDIlib_recv_free_video_v2`. A `RecvVideoFrame` **owns** the
  allocation and frees on `Drop` **exactly once**, keeping its receiver alive by
  construction (an `Arc` to the recv instance) so the free target is always valid.
  Recv yields UYVY → NV12 via the existing converter, written last-good into the
  per-source store (sampled-not-pacing, #1/#10).

### 3. Safe boundary + trait wiring
`multiview-ndi-sys` exposes safe RAII handles — **`NdiSender<'pool>`, `NdiFinder`,
`NdiReceiver`** (+ `NdiSourceName`, `RecvVideoFrame`) — whose methods are the *only*
way the rest of the workspace touches NDI; constructors return typed errors (never
panic/block); `Send + !Sync` markers with `// SAFETY` capture NDI's thread-affinity.
The live `SdkNdiApi` (in `multiview-output/src/ndi/`) and `SdkNdiReceiver` (in
`multiview-input/src/ndi/`) implement the **existing, verbatim** `NdiApi`
(`api.rs:162`) and `NdiReceiver` (`receiver.rs:77`) traits by **delegating** to these
handles, and stay `forbid(unsafe_code)`. `RecvVideoFrame` never escapes
`multiview-ndi-sys` into a consumer type. The boundary is stable across `v6`/`v6_3`
and future SDK versions because §1 absorbs the table shape.

### 4. bindgen / build config
Keep **regenerating the ABI at build time** from the licensed header behind the
`bindings` feature (do **not** commit the generated `.rs` — it is the SDK-derived
proprietary layout; keeping it build-time-only honours ADR-0008 and the crate's
"never reproduces the proprietary struct layout" stance). **Blocklist the direct
link-requiring `extern "C"` function declarations** (`NDIlib_version()` etc.) since we
runtime-load and never link — keep the *types* + the table struct. Use a small
`wrapper.h` (includes only the headers we use), newtype/rustified enums for the
FourCC + `frame_type_e`, `layout_tests(false)`, `generate_comments(false)`. Locate
the header via `NDI_SDK_INCLUDE_DIR` / `/opt/ndi/include` / `.ndi-sdk/include`, assert
the SDK version, `rerun-if-changed`, and emit a clear typed error when absent.

### Module layout
```
crates/multiview-ndi-sys/         (sole unsafe home; bindings feature)
  src/ffi.rs        bindgen include (generated, gated)
  src/table.rs      NdiV6 — resolve-once flat fn-ptr struct (§1)
  src/send.rs       NdiSender<'pool> + AsyncSendGuard + SendVideoFrame (§2)
  src/recv.rs       NdiReceiver + RecvVideoFrame (free-on-Drop) (§2)
  src/find.rs       NdiFinder + NdiSourceName
multiview-output/src/ndi/   SdkNdiApi: NdiApi  (forbid unsafe; delegates)
multiview-input/src/ndi/    SdkNdiReceiver: NdiReceiver (forbid unsafe; delegates)
```

## Consequences

- The implementation is **deliberate, not reactive**: the only place that knows the
  bindgen table shape is `table.rs`'s one resolver; everything above is flat, safe,
  panic-free, and stable across SDK bumps.
- Invariants are **type-enforced**, not merely documented: the async-send pin contract
  is a borrow-checked typestate; recv frees exactly once by RAII; NDI cannot pace the
  clock (`clock=false`) and cannot back-pressure the engine (bounded, off-hot-path).
- The default build stays NDI-free / LGPL-clean; the SDK-derived ABI never enters git.
- Cost: a one-time `NdiV6` resolver maps ~12 anon-union slots by index (verified once
  against the header; documented). If a future SDK renumbers them, only `table.rs`
  changes — the blast radius is one file.

## Alternatives rejected
- **Static-link `libndi`** (grafton-ndi/deno-plc) — forbidden by ADR-0008.
- **Hand-written `#[repr(C)]` table mirror** — reproduces the proprietary layout by
  hand; one misordered field silently misaligns every later pointer (UB).
- **Leak `__bindgen_anon_N` accessors workspace-wide** — the exact fragility this ADR exists to remove.
- **Commit the generated bindings** — would vendor the SDK-derived ABI into git.
