# ADR-0040 — IP multicast transport (UDP-TS + RTP) input + output, with a container-without-host-networking health warning

**Status:** Proposed (2026-06-08)
**Area:** streaming/timing (T) — contribution/distribution transport (implementation)
**Drivers:** multicast (UDP-TS / RTP) is the broadcast-plant contribution/distribution norm and
the media path the SAP discovery layer and AES67/ST 2110 require; plus the operator requirement
that the app **warn, bulletproofly and clearly, when multicast cannot work** (a container without
host networking).
**Research:** [multicast-transport](../research/multicast-transport.md).
**Related:** [ADR-0031](ADR-0031.md) (LGPL FFmpeg already enables `udp,rtp`),
[ADR-0030](ADR-0030.md) (`AVIOInterruptCB` fail-open), [ADR-0035](ADR-0035.md) (sense→detect→warn
`HealthWarning` catalog — the precedent this mirrors), [ADR-T003](ADR-T003.md) (unified timing),
[ADR-R001](ADR-R001.md)/[ADR-R003](ADR-R003.md) (continuous output + supervised reconnect),
[ADR-0039](ADR-0039.md) (SRT — the sibling typed-transport pattern). **The SAP discovery layer
that sits on top is [ADR-0041](ADR-0041.md).**

## Context

Multicast can be ingested today only as `SourceKind::Ts { url: "udp://@group:port" }` — an
**untyped** URL handed to libav (`pipeline.rs:3519`/`:4219`), with no address validation, no
interface/buffer/SSM control, and no IPv6 type guarantee. There is **no `Output::Udp`**, so the
already-implemented `PushProtocol::UdpTs` (`sink.rs:1028`, muxer `mpegts`) is **unreachable from
config** (`build_outputs` only builds `Rtmp`/`Srt`), and there is no RTP push at all. The
`st2110` module holds reusable RTP prior art (pure `RtpPacket::parse`, RFC-1982 seq math, the
bounded drop-oldest `channel_bridge`, the essence-agnostic ST 2022-7 `HitlessReconstructor`) but
its socket layer is audio/uncompressed-shaped and feature-gated.

Worse, multicast **fails silently in the default development and many deployment environments**:
on Docker's default **bridge** network the IGMP membership Report never leaves the container's
net namespace (moby/moby#23659), so a joined group receives zero LAN traffic — yet the libav open
and the IGMP join **succeed with no error**. The repo's own devcontainer is bridged
(`172.17.0.2/16`, no `--network=host`), so an operator testing multicast there sees a dead input
with no clue why. `--network=host` is not always available (it breaks VS Code extension install;
cloud Codespaces/VPCs carry no LAN multicast at all), so the engine must **degrade gracefully and
warn**, not fail.

The implemented typed-extension precedent is **`RtspOptions`** on `SourceKind::Rtsp`
(`schema.rs:175`/`:245`) — an `Option<Opts>` beside the `url` (a typed `SrtOptions` does **not**
exist; ADR-0039 is docs-only). The `HealthWarning` precedent is `GpuPresentNoVulkanAdapter` +
`emit_capability_warnings`/`CompositeMismatchView` (`warning_ingest.rs:38`/`:60`) +
`capability_warn::probe_and_emit` (`capability_warn.rs:111`).

## Decision

1. **Make IP multicast a first-class typed transport.** Add `SourceKind::Udp`/`SourceKind::Rtp`
   (ingest) and `Output::Udp`/`Output::Rtp` (egress) with **structured** fields
   (group/port/transport/`sources`/`block`/`interface`/`ttl`/`buffer_bytes`/`fifo_packets`/
   `overrun_nonfatal`/`pkt_size`; `Rtp` adds `rtcp_port`), modelled on the **`RtspOptions`**
   `Option<Opts>`-beside-`url` precedent (internally tagged, `#[non_exhaustive]`, **never
   `untagged`**, bare-`url` back-compat). The libav `udp://`/`rtp://` URL is **derived** from the
   typed config in `multiview-cli` — a single validated source of truth — so SSM/interface/buffer
   defaults are guaranteed, not left to operator URL hand-crafting. Respects inv #3 (typed config,
   never float) and gives address validation a home.

2. **Scope to compressed contribution/distribution multicast** (TS-over-UDP and TS-over-RTP
   MP2T PT 33) opened via the existing libav `ts` demux; **uncompressed ST 2110-20** (RFC 4175,
   multi-Gbps, PTP/ST 2059 hard-lock) is **out of scope**. **Reuse only** the `st2110` module's
   pure RTP-header parser, RFC-1982 seq math, ST 2022-7 `HitlessReconstructor`, and bounded
   drop-oldest channel — never the -20 raster assembly or PTP lock. Keeps the default build
   LGPL-clean; the multicast input free-runs on the output clock like every other source (inv #1).

3. **A multicast receive socket is an input jitter buffer, never a pacer** (inv #1). The kernel
   `SO_RCVBUF` + libav FIFO (or the `st2110` bounded channel) absorb burst/reorder; the output
   clock **samples** the latest reassembled frame each fixed tick and re-stamps PTS from the tick
   (`out_pts = f(tick)`, inv #1/#6). The TS 33-bit PTS is unwrapped by the existing `PtsNormalizer`
   (`WrapBits::Mpeg33`, inv #3); the RTP 90 kHz clock only **informs** per-input normalization,
   never slaves the master clock — `Rtp32` stays reserved for a future raw-essence path.

4. **Multicast loss/no-arrival rides the existing last-good-frame tile state machine**
   LIVE→STALE→RECONNECTING→NO_SIGNAL (inv #2) over the lock-free `TileStore` + `reconnect_backoff`.
   A freshly-joined multicast input starts with an **extended initial grace/RECONNECTING window**
   (the existing `PRIME_WAIT_BUDGET`, widened) so the normal first-packet-after-IGMP-join loss is
   **not** misread as NO_SIGNAL (RFC 2236 itself repeats the initial Report).

5. **Reach `PushProtocol::UdpTs` from `Output::Udp`** in `build_outputs`, and add a **new
   `PushProtocol::Rtp`** (muxer `rtp_mpegts`) reached from `Output::Rtp`. Multicast egress is
   **encode-once-mux-many** (inv #7): the canvas is encoded once and the **same** packets fan to
   the multicast sink alongside RTSP/HLS/NDI/SRT, re-stamped from the tick (inv #1/#6).

6. **Both the receive socket and the egress sink are bounded drop-oldest and the engine never
   awaits them** (inv #10): ingest reuses the `st2110` `channel_bridge` bounded mpsc + an explicit
   `SO_RCVBUF` cap (a flood degrades to drops, not memory growth); egress reuses the existing
   `SINK_WEDGE_GRACE` (2 s) to detach a stalled/dead multicast sink so it cannot back-pressure the
   output clock (inv #1).

7. **Warn — do not auto-fail or block start — when running in a container/devcontainer without
   host networking and a multicast source/output is configured.** A first-class `HealthWarning`
   modelled on [ADR-0035](ADR-0035.md) sense→detect→warn, **layered**: a cheap **advisory startup
   heuristic** (`WarningCode::ContainerNoHostNetworking`) **plus** an **authoritative runtime
   confirmation** (`WarningCode::MulticastJoinedNoTraffic` — join succeeded but zero bytes in a
   bounded window, which also catches a wrong `interface` / missing SSM source / no querier on a
   real host). Both ride the existing drop-oldest `Event::HealthWarningRaised` path (inv #10), are
   pure-Rust/default-build, latched (cannot flap), observability-only off the data plane (inv #1),
   and emit **nothing** unless **container AND non-host-networking AND a multicast config** all
   hold — no false positives. Detection layers many signals (`/.dockerenv`, cgroup v1+v2 substring
   scan, PID-1 `container=`, `KUBERNETES_SERVICE_HOST`, Codespaces/devcontainer vars, the
   net-namespace inode compare, and the strongest app-level signal: the configured `interface` is
   absent in-container). Detection lives in a new **pure** `multiview-cli/src/network_warn.rs`
   (the `NetworkMismatchView` analogue of `CompositeMismatchView`); emit mirrors
   `emit_capability_warnings`; the store + `GET /api/v1/health` are unchanged (keyed by `as_str`).

8. **Validate multicast group addresses at config-load** (pure, default build): accept
   `224.0.0.0/4` and `ff00::/8`; **reject** `224.0.0.0/24` (link-local control, never a media
   group) and the reserved `232.0.0.0`/`FF3x::4000:0000`; **require** a non-empty `sources` list
   for any `232/8` (SSM) group and flag the IGMPv3/MLDv2 requirement; prefer admin-scoped `239/8`
   for in-plant ASM. Security posture: **SSM source-filtering** is the inject/spoof defense, a
   **bounded receive path** is the flood/DoS defense (inv #10), and a multicast egress is a
   deliberate eavesdrop trade-off flagged for confidential content.

9. **Add multicast socket/buffer telemetry + health** (`multiview-telemetry`): packets
   received/dropped, the **granted-vs-requested `SO_RCVBUF`** (warn when the kernel silently clamps
   below request — document raising `net.core.rmem_max`), and the no-traffic state — surfaced on
   `GET /api/v1/health` and the realtime stream, never as a hard engine fault.

10. **Multicast is the media path; SAP is signalling.** This ADR (0040) is the transport
    foundation; the SAP announce/discover layer that sits on top is **[ADR-0041](ADR-0041.md)**,
    and AES67/ST 2110 also build on this. The implementation backlog (`MC-0..MC-12`) is in
    [docs/development/multicast-transport-backlog.md](../development/multicast-transport-backlog.md).

## Consequences

- IP multicast becomes a typed, validated, observable ingest **and** egress transport with the
  reliability defaults (interface binding, SO_RCVBUF/FIFO sizing, SSM source-filtering) baked in,
  reusing the `st2110` RTP/2022-7 prior art — **no new feature flag, no GPL escalation** (it rides
  the existing `ffmpeg` feature; `udp`/`rtp` are already LGPL-clean per ADR-0031).
- The silent-multicast-in-a-bridge trap becomes a **loud, actionable, false-positive-free**
  warning — the operator is told exactly why a multicast input is dead and how to fix it, without
  the engine ever failing or auto-disabling a source (inv #1).
- The already-built `PushProtocol::UdpTs` finally has a config path; RTP push (`rtp_mpegts`) is the
  one genuinely new egress muxer.
- Existing `Ts { url: "udp://@…" }` configs keep working (back-compat) and can migrate to the typed
  kinds for validation + defaults.
- Bounds the scope explicitly: uncompressed ST 2110-20 and a manual socket2 SSM path are deferred,
  named, not stubbed.

## Alternatives rejected

- **Keep `Ts { url: "udp://@…" }` as the only (untyped) multicast path.** Rejected: no address
  validation (a media group in `224.0.0.0/24` silently never routes), no interface/buffer/SSM
  control (the default-gateway-join trap and HD/UHD kernel-drop corruption both bite with no
  operator knob), no IPv6 type guarantee, and no home for the multicast reliability defaults. A raw
  URL cannot carry them.
- **Auto-fail / refuse to start (or auto-disable the source) when no host networking is detected.**
  Rejected: the startup heuristic cannot prove the LAN is silent and would false-positive on a
  real bare-metal host, a correct `--network=host`/macvlan-L2 container, or a deliberate bridge-only
  deployment; a misconfigured/limited input must never stall output (inv #1) or be silently
  removed. **Warn** (advisory startup + authoritative runtime), never fail.
- **Block start inside a devcontainer/Codespace.** Rejected: operators legitimately run
  non-multicast configs there, and `--network=host` is not always available. The conjunctive
  (container AND non-host AND multicast-configured) warning targets exactly the risky case without
  blocking anything.
- **Fold multicast into the `st2110` effort.** Rejected: ST 2110-20 is uncompressed, multi-Gbps,
  PTP-hard-locked, jumbo-frame, kernel-bypass — a separate heavy effort with a different payload
  and timing regime. Compressed TS-over-UDP/RTP is a tens-of-Mbps libav-demuxed transport that
  free-runs on the output clock. We **reuse** the RTP-header parser + seq math + 2022-7
  reconstructor + bounded channel, but ship multicast as its own LGPL-clean transport.
- **Derive the libav URL by string-templating operator fields.** Rejected: footgun-prone (`@`
  placement on send vs receive, emitting a nonexistent `multicast=` option, `fifo_size`
  byte-vs-packet confusion, missing `localaddr`). A deterministic typed derivation with one builder
  is testable and guarantees the defaults.
- **A manual `socket2`/`libc` SSM (`IP_ADD_SOURCE_MEMBERSHIP`) receive path for v1.** Rejected:
  libav `sources=` already performs the IGMPv3 INCLUDE join for the single-path case; a manual
  socket path adds FFI/feature-gated complexity for no current benefit. Deferred behind a feature
  (mirroring the `st2110` socket layer).
- **Detect host-vs-bridge networking from the libav open/join result.** Rejected: `bind` and
  `IP_ADD_MEMBERSHIP` **succeed** on a bridge — the failure is silent (zero packets), so an
  errno-based detector never fires. Detection must be out-of-band (the startup heuristic) plus the
  authoritative "joined-but-zero-bytes-in-window" runtime signal.
- **A single container-detection signal** (only `/.dockerenv`, or only a cgroup substring).
  Rejected: `/.dockerenv` is absent at buildx build time; the cgroup substring is empty under
  cgroup v2 (`0::/`); the `container=` var lives on PID 1, not the child. Layer multiple
  independent signals; trust none alone.
