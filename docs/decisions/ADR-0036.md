# ADR-0036: A typed codec catalog with transport×codec carriage and a per-host acceleration class — replacing the free-text output codec and killing the silent MPEG-2 fallback

- **Status:** Proposed
- **Area:** Config / FFmpeg / HAL / Output / Control / Web
- **Date:** 2026-06-08
- **Source brief:** [codec-capabilities.md](../research/codec-capabilities.md)
- **Builds on:**
  - [ADR-0035](ADR-0035.md) (self-aware placement / runtime capability detection — the
    `CapabilityReport` that supplies *"what the hardware can actually do"*).
  - [ADR-M007](ADR-M007.md) (`CapabilityReport` as the **single** machine-readable gate for
    UI **and** validator — never two sources of truth).
  - [ADR-0003](ADR-0003.md) (three-layer capability model; *"NVML cannot report codec
    capability — must use SDK caps APIs or probes."*).
  - [ADR-0011 / ADR-0012](README.md) + [conventions.md §4/§7](../architecture/conventions.md)
    (feature flags; LGPL-default licensing; `gpl-codecs` off by default; `ndi` proprietary
    runtime-loaded).
  - [ADR-0026](ADR-0026.md) (encode-once-mux-many — decodable ≠ encodable, two codec axes).
  - [ADR-R004](ADR-R004.md) (make-before-break Class-2 migration).
- **Relates to:** the audio transport-carriage precedent `multiview-audio/src/capability.rs`;
  [management-capability-matrix.md](../research/management-capability-matrix.md) (codec change
  = Class-2); invariant #10 (no back-pressure).

## Context

The output `codec` is **free text** with no real validation, and the run-time path
**silently substitutes `mpeg2video`** for anything it cannot encode. Verified in code
(2026-06-08):

1. `pub codec: String` on 5 of the 6 `Output` variants in
   `crates/multiview-config/src/schema.rs` (`RtspServer`, `LlHls`, `Hls`, `Rtmp`, `Srt`);
   `Output::Ndi` correctly has no codec field.
2. `validate_outputs()` in `crates/multiview-config/src/lib.rs` rejects **only**
   `codec.is_empty()` — no transport-carriage check, no host-encodability check, no
   real-codec allowlist anywhere in the config crate. Any non-empty string passes.
3. `crates/multiview-cli/src/pipeline.rs` performs **two silent substitutions**:
   `logical_codec()`'s `_ => VideoCodec::Mpeg2Video` maps every unknown token to MPEG-2, and
   `resolve_encoder()` `tracing::warn!`s + falls back to `select_encoder(Mpeg2Video)` when
   the requested codec is not encodable. On the LGPL-clean default build (no `cuda`/
   `gpl-codecs`), even the valid token `"h264"` silently becomes `mpeg2video` because H.264
   has **no** LGPL software encoder (`lgpl_software_encoder()==None`), confirmed by the
   in-tree test `default_build_cannot_encode_h264_or_h265`.

This is precisely how the demo silently became `mpeg2video`. The product goal — most robust,
bulletproof, **really easy to use** — requires the form to offer **only valid choices**, each
labelled with its acceleration class, and to **reject** an uncarryable/unencodable codec with
an actionable reason rather than silently downgrade it.

Two reusable precedents already exist: the FFI-side typed enums + honest feature-gated
encoder resolvers in `multiview-ffmpeg/src/codec.rs`, and the audio transport-carriage model
(`OutputTransport` + pure-const `for_transport()` + typed `validate_*`) in
`multiview-audio/src/capability.rs`. The missing half — *which (device, codec) pairs this
host can actually encode* — is exactly ADR-0035's (unbuilt) `CapabilityReport`; today
`multiview-hal`'s `Capability` has **no codec axis** at all.

## Decision

Build a **typed codec catalog** as the single key space, a **pure transport×codec carriage
matrix**, a **per-host acceleration class derived from the ADR-0035 `CapabilityReport`**, and
wire all of it into a **validated surface** (config → API → UI), removing the free-text field
and the silent fallback.

1. **Typed catalog in `multiview-core`** (new `multiview-core/src/codec.rs`). `core` is the
   only crate `multiview-config` may depend on (it must not pull FFI-owning
   `multiview-ffmpeg`), so the `VideoCodec`/`AudioCodec` enums **move to `core`** with serde
   (`snake_case` unit variants → bare-string tokens, conventions §9-compliant) plus
   `CodecEntry` metadata (`wire_id` RFC 6381 + MPEG-TS stream_type, profiles, `LicenseClass`)
   and `ALL` const arrays. `multiview-ffmpeg` re-exports them and keeps the libav encoder-name
   resolvers (`lgpl_software_encoder`/`gpl_software_encoder`/`nvenc_encoder`/`select_encoder`)
   as impls over the relocated enum — no duplicate definition, existing import paths intact.
   The encode `VideoCodec` **gains `Av1`, `Vp9`, `Vp8`** so the transport matrices are
   expressible; because the enum is `#[non_exhaustive]` with exhaustive resolver `match`es,
   the new variants **force** honest, feature-gated resolver arms. `AudioCodec` gains `Ac3`,
   `Eac3`, `PcmS16Le`.

2. **`LicenseClass { LgplClean, GplEscalating, Proprietary }` keyed on (codec, backend)**, not
   flatly per-codec: H.264/H.265 are `LgplClean` when HW-encoded (NVENC) and `GplEscalating`
   when SW-encoded (`libx264`/`libx265`, `gpl-codecs`). AV1/VP9/VP8 = `LgplClean`
   (royalty-free encoders). AC-3/E-AC-3 = `LgplClean` (FFmpeg native `ac3`/`eac3`) with a
   patent-pool obligation noted. Native `aac`/`libopus`/`mp2`/PCM/MPEG-2/FFV1/MJPEG =
   `LgplClean`. `libfdk_aac` is **never** modelled. NDI's wire codec is `Proprietary` and
   **not in this catalog**.

3. **Transport×codec matrix in a pure crate**, mirroring `multiview-audio/src/capability.rs`:
   an `OutputTransport` enum + pure-const `for_transport()` + `validate_codec(transport,
   codec) -> Result<(), CodecCarriageError>`. SRT collapses to `MpegTs` via a single shared
   `transport_class()` (so audio and video can never drift). RTMP splits into **classic-FLV**
   (video `{H264}`; audio `{Aac, PcmS16Le}`) vs **Enhanced-RTMP-v2** (video `{H264, H265,
   Av1, Vp9, Vp8}`; audio adds `{Opus, Ac3, Eac3}`), default conservative to classic. The
   RTSP allowed-set is **sourced from the in-process server's payloader table** (`RtspCodec`),
   so the gate and the catalog cannot disagree.

4. **Codec×acceleration class derived from ADR-0035's `CapabilityReport`.** Add the missing
   per-(device, codec) `{present, usable, in_use, reason}` to the `CapabilityReport` via a
   throwaway `ctx.open_as(encoder)` / `*_cuvid` open at target resolution, refined over a
   static per-(vendor, gen, codec) prior. `AccelerationClass { CpuOnly, GpuOnly, Both,
   DedicatedFuture }` is **derived** from the actual non-empty usable-backend set — never from
   `EnvProbe` presence. Present-but-unusable (AV1 NVENC on pre-Ada; NVENC linked but session
   fails to open) **downgrades** with a `reason`, never a silent fallback.

5. **The validated surface, two phases + API + UI.**
   - **Phase 1 (pure, `multiview-config`):** codec field becomes typed `VideoCodec`/
     `AudioCodec`; `validate_outputs()` additionally enforces `codec ∈ for_transport(t)`.
   - **Phase 2 (host, CLI/control):** new `validate_with_capabilities(&self,
     &CapabilityReport)` rejects un-encodable picks with the report's `reason`, and the
     `pipeline.rs` `_ => Mpeg2Video` and warn+fallback are **replaced** by a hard
     `PipelineError::Codec { codec, reason }`. This validation runs **pre-flight**.
   - **API:** read-only `GET /api/v1/system/capabilities/codecs?transport=<kind>` returns the
     intersection (catalog ∩ transport-allowlist ∩ host encode-support), each entry carrying
     `{id, label, kinds, acceleration, backends, license, license_escalating, supported,
     reason?, default}`, served from a cached `CapabilityReport` snapshot (invariant #10).
   - **UI:** a `<CodecSelect>` dropdown (built on `KindSelect`) fed by a
     `useCodecCapabilities(transport)` hook, each option badged CPU / GPU / CPU+GPU /
     dedicated with a ⚠ license-escalating marker, unsupported codecs greyed with the `reason`
     tooltip; default from `default:true`. Client/types generated from OpenAPI.
   - A codec change on a **live** output is a **Class-2** controlled reset, surfaced as such
     (keyed on whether the output is currently running).

## Consequences

**Positive**

- The operator can no longer type a codec a transport cannot carry or this host cannot
  encode; the silent `mpeg2video` substitution is eliminated end to end.
- One catalog + one `CapabilityReport` is the single source of truth for the config
  validator, the API, and the UI (ADR-M007) — no UI/validator drift.
- License correctness is explicit and per-backend; the LGPL-clean default is preserved and
  `gpl-codecs` picks are flagged ⚠. The default build never selects nonfree `libfdk_aac`.
- Transport carriage rules are pure (no hardware), testable, and mirror the audio precedent.

**Negative / costs**

- **Behaviour change:** a config naming a non-encodable codec now **fails the run** instead of
  degrading. Mitigated by running `validate_with_capabilities` pre-flight so the failure is a
  clear pre-start rejection, not a mid-pipeline crash.
- Adding `Av1`/`Vp9`/`Vp8` to the `#[non_exhaustive]` encode enum forces new resolver arms,
  which must be filled **honestly** (AV1 HW gated Ada+; VP9 has no NVENC; verify SW encoders
  are present in the linked FFmpeg) — an incorrect arm would re-create the silent-fallback bug.
- The acceleration class depends on ADR-0035's `CapabilityReport`, which is **unbuilt**; the
  per-(device, codec) probe extension is on this ADR's critical path (CODEC-3).

**Neutral**

- Moving the enums to `core` is non-breaking (re-export preserves `multiview_ffmpeg::codec::*`
  paths). Phase 1 (pure) ships and tests with zero hardware; Phase 2 needs a host snapshot.

## Alternatives considered

- **Keep free text, add a regex/string allowlist in config.** Rejected: still no
  transport/host validation, no license/acceleration metadata, and stringly-typed — the exact
  class of bug we are removing.
- **Define the catalog in `multiview-ffmpeg` (where the enums live today) and have config
  depend on it.** Rejected: `multiview-config` must stay FFI-free and feature-flag-free;
  depending on `multiview-ffmpeg` would pull libav into the config crate.
- **Keep the `mpeg2video` fallback but log louder.** Rejected: a `warn` on a daemon is not a
  user-facing rejection; "easy to use + bulletproof" means the form offers only valid choices
  and an invalid one is rejected, not silently degraded.
- **Two capability sources (a UI list + a separate server validator).** Rejected by ADR-M007:
  drift → the UI offers what the server rejects. One `CapabilityReport` gates both.
- **Store `AccelerationClass` as a fixed per-codec constant.** Rejected: the class is
  per-host (depends on compiled features + the actual usable GPU generation); it must be
  **derived** from the `CapabilityReport`, never assumed (e.g. H.264 is *Unsupported* on a
  pure default build, not GPU-only).
- **Hardcode the RTSP set as the abstract spec `{H264,H265}`.** Rejected: source it from the
  server's payloader table so adding a payloader widens the catalog in one edit and never
  advertises a codec the server cannot payload.
