# ADR-0050 — Conspect entitlement: the `multiview-licence` crate, enforcement-ladder-as-data, never-off-air

**Status:** Proposed (2026-06-10)
**Area:** Account / Licensing — the entitlement plane of the Conspect account-side subsystem
**Drivers:** Multiview ships source-available with a paid commercial licence ([ADR-0043](ADR-0043.md)).
The product needs a machine-side **entitlement** mechanism — claim a machine, hold a heartbeat-granted
lease, and *gently* enforce when the lease lapses — that **never** takes a broadcaster's program off
air. There is no lease/entitlement/fingerprint primitive in the tree (greenfield); the only precedent
is the NDI **codec**-licence gate (`multiview-input/src/ndi/license.rs`), which is unrelated to
*account* licensing.
**Research:** [conspect-account-architecture](../research/conspect-account-architecture.md)
(the subsystem brief — §2 constants, §5 seams, §6 ladder, §12 state machines).
**Related:** [ADR-0043](ADR-0043.md) (source-available non-commercial relicence — the legal frame this
machine obeys), [ADR-0051](ADR-0051.md) (mesh relay — how an offline machine still heartbeats),
[ADR-0052](ADR-0052.md) (two-pipe heartbeat/telemetry separation), [ADR-0053](ADR-0053.md)
(support/audit), [ADR-I001](ADR-I001.md) (the wait-free isolation primitives the seams reuse),
[ADR-T001](ADR-T001.md)/[ADR-R001](ADR-R001.md) (output-clock invariant #1 this ADR must not break),
[ADR-RT004](ADR-RT004.md)/[ADR-P001](ADR-P001.md) (isolation invariant #10).

## Context

The entitlement model must be (a) **honest** — a source build can compile the heartbeat out, and the
licence binds regardless (the official binary is the licensed *convenience*, not a DRM wall); (b)
**never off air** — the hardest enforcement rung still emits one valid frame per tick; (c) **data, not
control flow** — the enforcement state is a field on a resource that the engine, the API, the portals,
and the operator all read identically; and (d) **privacy-preserving** — machine identity is a *score*
over salted digests, never raw serials/MACs.

The external companion documents (Licensing Architecture spec, Pricing & Hardware Platform Strategy v4,
portal HTML prototypes) are **not in this repo** and are treated as **external governing artefacts**;
where they are silent on a machine-side mechanism this ADR proposes an approach and flags it for
operator confirmation (the brief §14 collects them; O1/O2/O3/O6 are this ADR's).

## Decision

### 1. One new leaf crate: `crates/multiview-licence`

Lib `multiview_licence`. Depends only on `multiview-core`, `multiview-events`, `thiserror`, `tracing`,
`ed25519-dalek` (`MIT OR Apache-2.0`, deny-clean), and `chrono` (exact lease arithmetic — never float).
**No** `multiview-ffmpeg`, **no** GPU, **no** dependency on `multiview-engine` (the cli feeds the
engine seams *from* this crate; the dependency direction stays acyclic: `licence → {core, events}`,
`cli → licence`). Default build is a **pure shell** so `cargo check --workspace` stays green and
deny-clean.

### 2. The enforcement ladder is DATA on the licence resource

`enforcement` is a field, not a code path: `{ level, reasons[], since, lease{…} }`. The **level** is
one of `active · warning · config-locked · watermark · block-new-instance · unlicensed-build`. It is
computed **only** in this crate, **off the hot loop**, from the lease arithmetic (§4) + fingerprint
continuity (§5) + claim/transfer state. The engine, `GET /api/v1/account/licence`, the chrome banner,
and the portals all read the **same** `level`. There is no second opinion and no hidden enforce path.

### 3. The crate publishes ONE wait-free `LicenceStatus`; the engine reads TWO derived atomics

This crate publishes `LicenceStatus` (level + reasons + lease bounds) via `arc_swap::ArcSwapOption`
(the [ADR-I001](ADR-I001.md) primitive). The cli derives exactly two cheap atomics the engine reads on
the hot path — `config_locked: AtomicBool` and `watermark: AtomicBool`. Everything richer is read only
off the loop (the control plane). **That is the entire engine surface: two atomic loads per tick,
derived from data.** No licence logic, no allocation, no network, no lock ever runs on the tick path.

### 4. Lease arithmetic — EXACT constants (the portals use them)

A successful heartbeat grants a **35-day** lease (`LEASE_FULL`). After expiry there is a **14-day**
grace (`LEASE_GRACE`). The absolute outer bound from last contact is **90 days** (`LEASE_HARD`). The
minimum contact cadence is **monthly** within a **31-day** activation window (`ACTIVATION_WINDOW`).
These are carried as named constants and asserted by property tests; **they are not rounded or
re-derived** — a portal showing "35 days" and a machine enforcing 30 is a support incident. Mapping to
levels:

| Level | Trigger | Start (S1) | Hot-reconfig (S2) | Canvas (S3) | Program |
|---|---|---|---|---|---|
| `active` | within `LEASE_FULL` | ✓ | ✓ | clean | **on air** |
| `warning` | nearing expiry / in `LEASE_GRACE` | ✓ | ✓ | clean | **on air** |
| `config-locked` | past grace, before hard | ✓ | **denied** | clean | **on air** |
| `watermark` | further lapse toward `LEASE_HARD` | ✓ | denied | **watermark** | **on air** |
| `block-new-instance` | past `LEASE_HARD` (~90d) | **denied** | denied | watermark | **on air (running stays up)** |
| `unlicensed-build` | heartbeat compiled out (§7) | ✓ | ✓ | watermark (honest) | **on air** |

### 5. Three engine seams (S1/S2/S3); the never-off-air proof

- **S1 — Startup gate:** at `SoftwareEngine::build()` in the cli, **before** `EngineRuntime::new()`,
  refuse to create a *new* engine when the ladder is at `block-new-instance`. A *running* engine never
  re-enters `build()`, so the hardest rung cannot touch it. Typed `LicenceError`, mirrors the NDI
  pre-flight gate.
- **S2 — Config-lock:** the existing frame-boundary control hook reads `config_locked` and **skips**
  `apply_layout`/`rebind_cell` when set — the running scene keeps playing; you simply cannot
  reconfigure it. O(1), allocation-free, wait-free, no `.await`.
- **S3 — Tile watermark:** post-composite, before publish, when `watermark` is set, blit a corner
  watermark into the NV12 canvas (region-limited, NV12-throughout, ADR-0023 dirty-rect bake). Reads
  status; mutates only the canvas.

**Never-off-air (inv #1 + #10):** no rung is on the tick path; S2/S3 read pre-derived atomics only;
`block-new-instance` is a *startup* gate; config-lock/watermark are *conveniences*; the entitlement
task publishes via `arc_swap` and the engine *samples* it (never awaits, never shares a lock). On
internal failure the crate publishes `warning`, **never** a harder rung — it fails toward leniency and
hardens only on positive evidence of a computed lapse.

### 6. Fingerprint scoring — salted digests, threshold 70/100

Machine identity is a **score** over salted hardware-component digests (§8 of the brief): each
component is salted with a per-deployment salt then hashed; only the **digest set** and the **score**
are exposed (never raw serials/MACs). `FINGERPRINT_MATCH_STRONG = 100` (perfect set match);
`FINGERPRINT_MATCH_THRESHOLD = 70` — at/above 70 it is the *same* machine (hardware drift tolerated),
below 70 a *new* machine requiring re-claim. The salt provenance (server-issued at claim vs
locally-generated) is an operator-confirm item (brief O6).

### 7. The honest source-build caveat (binding regardless)

The heartbeat **client** lives behind a `heartbeat` Cargo feature (off in default/CI, on in shipped
presets). The **state model** (ladder, lease, claim/transfer, fingerprint, resource shape) is **always
compiled** so the API/UI render consistently. A build with the client off reports
`enforcement.level = "unlicensed-build"` **honestly** — it does not fake `active`. We state plainly, in
copy and docs, that removing the heartbeat does not grant a commercial right: the
[licence](../../LICENSE) binds *use*; the official signed binary is the licensed *convenience*;
enforcement is a convenience-and-evidence mechanism, **not** a cryptographic lock (operator directive,
spec §4.1).

### 8. Claim / transfer state machines (data; constants exact)

Pure state machines in this crate, exposed as resource state, tested with `proptest-state-machine`:
**claim** `Unclaimed → CodeIssued(ttl=15m) → {Confirmed→Claimed | Rejected(24h window) | Expired}`;
**transfer** `Claimed → TransferPending(72h) → {Confirmed→Claimed(new) | Cancelled/Expired→Claimed(old)}`.
The claim code is **6 chars** from an ambiguity-free alphabet (the exact glyph set copied verbatim from
the portal — brief O3), case-insensitive on input, canonicalised on store, single-use. `Idempotency-Key`
on claim/transfer; RFC 9457 problem docs; `ETag`/`If-Match` on the licence resource.

### 9. Feature ladder + licensing

- Crate features: `heartbeat` (the network client, off by default), nothing else native.
- cli: an `account` aggregation enables the entitlement plane; shipped presets (`nvidia`, `apple`,
  `linux-vaapi`, `full`) turn `heartbeat` on. The **default** build stays a pure shell.
- `ed25519-dalek` + `chrono` are `MIT OR Apache-2.0`, already deny-friendly; nothing here touches the
  GPL or proprietary boundaries.

## Consequences

- **One auditable entitlement plane.** Lease, claim, transfer, fingerprint, and the ladder live in one
  leaf crate; the engine surface is two atomics. The state is data every surface renders identically.
- **Program is sacred on every rung** — the hardest enforcement still emits one valid frame per tick;
  this is provable by construction and is made an executable chaos test (CONSPECT-2).
- **Greenfield, collision-free** — the crate touches no file the parallel WebUI session edits; it is
  built and merged first.
- **Honest by design** — a source build that drops the heartbeat reports `unlicensed-build`, not a
  fake `active`; the licence binds regardless. We never market enforcement as DRM.
- **Operator confirmations gate only the live wire** (O1 wire protocol, O2 key-pinning/rotation, O3
  glyph alphabet, O6 salt provenance) — the crate is fully buildable and testable against an
  in-process fake licence server before any of them land.

## Alternatives rejected

- **Enforcement as control flow (an `if licensed { … }` scattered through the engine).** Unauditable,
  invites a #1-breaking path on the hot loop, and gives the engine, API, and portals divergent
  opinions. Rejected for the ladder-as-data model (one state, every surface reads it).
- **A hard DRM kill-switch (stop output when unlicensed).** Directly breaks invariant #1 and the
  product promise; also pointless against a source build. Rejected — enforcement degrades conveniences
  only and never the program.
- **Folding entitlement into `multiview-control`.** Bloats the control crate, couples licensing to the
  API surface (collision with the WebUI session), and loses the clean leaf-crate test boundary.
  Rejected for the greenfield leaf crate.
- **Raw hardware identifiers for machine identity.** A privacy and correlation hazard, and brittle to
  benign hardware drift. Rejected for the salted-digest **score** with a 70/100 tolerance threshold.
- **Always compiling the heartbeat client in.** Forces the network dependency into the default/CI
  build (deny-noise, non-pure default) and pretends a source build can't remove it. Rejected — gate
  the client, always compile the honest state model.
