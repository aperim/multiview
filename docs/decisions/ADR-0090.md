# ADR-0090: Ubiquitous non-disruptive preview — preview anywhere a stream is referenced

- **Status:** Proposed
- **Area:** Web/API stack ↔ control ↔ preview (operability)
- **Date:** 2026-06-13
- **Source brief:** [webui-operability-gaps.md §2](../research/webui-operability-gaps.md)
- **Extends:** [ADR-P002](ADR-P002.md) (per-scope transport: JPEG grids / WHEP focus / LL-HLS fallback),
  [ADR-P003](ADR-P003.md) (on-demand activation + auto-stop, cost ~0 when idle),
  [preview-subsystem.md](../research/preview-subsystem.md)
- **Relates to:** [ADR-P001](ADR-P001.md) / [ADR-RT004](ADR-RT004.md) (isolation, inv #10),
  [ADR-0059](ADR-0059.md) (audio meters — consumed, not built here)

## Context

The preview subsystem is a real, isolation-structural side-channel: built endpoints
(`GET /api/v1/preview/program.jpg`, `/preview/inputs/{id}.jpg`, `/preview/inputs`, WHEP focus —
`crates/multiview-control/src/routes/preview.rs`), a built provider
(`crates/multiview-cli/src/preview.rs` `CliPreviewProvider`, with the live-updatable `SharedStores`
map per [ADR-W018](ADR-W018.md)), refcounted lifecycle ([ADR-P003](ADR-P003.md)), and per-scope
transport selection ([ADR-P002](ADR-P002.md)). A slow/absent preview consumer cannot back-pressure
the engine ([preview-subsystem §2](../research/preview-subsystem.md)).

The operator's 2026-06-13 ask is **not** more preview machinery — it is **ubiquity + non-disruption**:
*anywhere in the WebUI that references a video or audio input/output there should be an easy way to
preview it, without interrupting whatever workflow the user is in.* Today preview is reachable only
from dedicated preview pages, and there is no reusable, page-overlay affordance attached to the many
places a stream is named (source list, output editor, layout-editor source palette, router matrix,
track inspector, tile context menu).

## Decision

Define a single reusable **`<PreviewAffordance>`** UI contract plus one thin read-only descriptor, both
layered over the existing (built) preview taps — **no new engine surface, no new transport**.

1. **`<PreviewAffordance>` keyed by a typed `PreviewTarget = { scope: input|program|output, id }`.**
   It renders a small hover/focus-revealed button (with a keyboard affordance, WCAG 2.1 AA) on any
   element naming a stream. Activating it opens a `<PreviewOverlay>` **layered above the current view**
   (Radix popover/dialog, [conventions §8](../architecture/conventions.md)); the page underneath keeps
   all state, including in-progress unsaved edits. Closing returns focus to the trigger. **The overlay
   never issues a state-changing request and never navigates** — non-disruption is the contract,
   enforced by an interaction test.
2. **Default mode = the cheapest tap for the scope** ([ADR-P002](ADR-P002.md)): polled snapshot / MJPEG
   thumbnail. A "Focus / Live" toggle upgrades the one open overlay to WHEP and reverts on close
   (refcount + linger, [ADR-P003](ADR-P003.md)); opening a second focus demotes the first
   (the existing one-focus ergonomic). Audio targets show the live numeric meter (existing realtime
   lane) and are **meter-only** until an audio audition path exists — stated honestly, never silent
   pretend-audio.
3. **One new read-only descriptor** `GET /api/v1/preview/targets/{scope}/{id}` (or a field on the
   existing list endpoints) returning `{ available_modes, fidelity_label, subscriber_count }`, served
   from a cached descriptor. The affordance greys unavailable modes and renders the mandatory fidelity
   label (`REAL ENCODED OUTPUT (tap: …)` vs `PRE-ENCODE CANVAS APPROX`,
   [preview-subsystem §6](../research/preview-subsystem.md)). `subscriber_count` exposes who-is-watching
   activity, so it is **role/scope-gated** (admin/operator only); non-admin callers see a coarse
   `in_use | capacity_limited` status instead of an exact count, so the descriptor never leaks
   operational activity across users/tenants.
4. **Shed honestly.** Under load the planner sheds preview first ([preview-subsystem §3](../research/preview-subsystem.md));
   the overlay shows "preview reduced — system busy", never a frozen frame pretending to be live.

## Consequences

- **Positive.** Preview becomes reachable wherever a stream is named, at the cheapest transport, with a
  consistent fidelity label, and never interrupts the operator's current task. Reuses all built taps,
  lifecycle, and isolation guarantees; the only new server code is one cached read.
- **Negative / cost.** A new shared SPA component + one descriptor endpoint + generated types. Risk of
  visual clutter (Open question 6 in the brief — default hover/focus-reveal, not a persistent per-row
  button). Audio preview is meter-only at first.
- **Neutral.** Idle cost stays ~0 ([ADR-P003](ADR-P003.md)); an open snapshot is one HW
  downsample + JPEG; a focus is one budgeted WHEP, shed first.

## Alternatives considered

- **A dedicated preview page only (status quo).** Rejected: forces a context switch / navigation away
  from the operator's workflow — exactly what the intake says is wrong.
- **Inline always-live thumbnails on every reference.** Rejected: violates the cost-~0-idle invariant
  ([ADR-P003](ADR-P003.md)) and clutters every list; the overlay-on-demand model preserves both.
- **A new preview transport for "everywhere".** Rejected: the three built transports already cover
  grid/focus/fallback ([ADR-P002](ADR-P002.md)); the gap is UI ubiquity, not transport.

## How it honors the invariants

- **#1 (output clock).** The affordance only reads existing taps; preview reads are wait-free
  latest-frame slots and a request-task JPEG/encode, never on the output-clock loop
  ([preview-subsystem §2](../research/preview-subsystem.md), `crates/multiview-cli/src/preview.rs`).
  No path here can pace or block the clock.
- **#10 (no back-pressure).** Every tap is capacity-1 drop-oldest; the descriptor is a cached read; the
  engine never awaits a preview consumer; preview is admission-shed-first. Opening N overlays cannot
  stall the engine — they share one tap per entity and degrade to "reduced" under pressure.
