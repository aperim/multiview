# ADR-0080: HTML/CSS/JS motion-graphics engine — offscreen GPU surface, engine isolation, efficiency budget

- **Status:** Proposed
- **Area:** Compositor / Overlays (motion graphics from web technologies)
- **Date:** 2026-06-13
- **Source brief:** [motion-graphics-html.md](../research/motion-graphics-html.md)
- **Extends:** [ADR-R008](ADR-R008.md) (overlay layer stack + portable premultiplied-RGBA atlas/quad contract — graphics ride this, input-decoupled), [ADR-0016](ADR-0016.md) (one linear blend in *our* compositor; never a third-party render pass), [ADR-0058](ADR-0058.md) (NV12+A — the production-tier graphics-fill payload), [ADR-0056](ADR-0056.md) (keyer fill + multi-box composition — the on-clock consumption point)
- **Relates to:** invariant #1 / [ADR-T001](ADR-T001.md)/[ADR-R001](ADR-R001.md) (output clock), invariant #10 / [ADR-RT004](ADR-RT004.md) (isolation), invariant #5 / [ADR-E002](ADR-E002.md) (NV12-throughout), invariant #8 / [ADR-C003](ADR-C003.md)/[ADR-C006](ADR-C006.md) (color order), [ADR-E007](ADR-E007.md) (admission/degradation), [ADR-R011](ADR-R011.md) (source-loss policy), [ADR-0059](ADR-0059.md) (the consumer-owned RCU control-seam shape), [conventions §3/§4/§7](../architecture/conventions.md) (feature-gating + LGPL-clean default), [ADR-0081](ADR-0081.md) (templating/data + N-source), [ADR-0082](ADR-0082.md) (render-to-VT)

## Context

Operators author motion graphics (lower-thirds, tickers, scoreboards, clocks, full-screen transitions) overwhelmingly as **HTML/CSS/JS**. The operator asked for HTML+CSS (and JS) motion graphics. The open-source CasparCG HTML producer is the conceptual demonstration that an embedded browser can render live broadcast graphics; we cite it conceptually only and build from this repo's own contracts.

A browser engine is the **heaviest plausible dependency** in Multiview: hundreds of MB of binary (CEF/Chromium) or a large Rust build (Servo), a multi-process sandbox or a large in-process surface, and per-instance CPU/GPU/RAM. Verified external facts (web-checked 2026-06): CEF offscreen rendering surfaces frames via `OnPaint` (CPU buffer) / `OnAcceleratedPaint` (GPU shared texture, most mature on Windows/D3D11), and `SendExternalBeginFrame` lets the host drive frame timing (disabling the browser's internal timer). Servo offers an embeddable Rust WebView with WebRender + `surfman` offscreen GPU surfaces (MPL-2.0), with embedding/offscreen an active 2024–2025 workstream.

The decisive constraint is isolation. This repo already pins that overlays/aux consumers are **input-decoupled and cannot back-pressure the engine** ([ADR-R008](ADR-R008.md)/[ADR-0016](ADR-0016.md)/[ADR-RT004](ADR-RT004.md)), and that a graphics fill enters the canvas through **our** fixed linear-light pipeline, never a foreign render pass ([ADR-0016](ADR-0016.md) rejected `glyphon`'s own framebuffer blend for exactly this). The NV12+A payload ([ADR-0058](ADR-0058.md)) rides the generic `TileStore` "with zero framestore changes," giving a ready producer→engine substrate. The question is how to embed a browser without ever letting it touch the output clock, and at what cost.

## Decision

1. **An HTML graphic is a feature-gated producer behind a `GraphicsEngine` trait.** Ship an off-by-default **`html-graphics`** feature (never in the default LGPL-clean, no-native-dep build — exactly the `ndi`/`gpl-codecs` posture, [conventions §7](../architecture/conventions.md)). Abstract the engine behind a `GraphicsEngine` trait so the rest of Multiview sees only "a producer that publishes a premultiplied RGBA surface." **Recommend CEF for the first shippable tier** (proven broadcast path, mature external-BeginFrame + offscreen), **track Servo as the Rust-native second backend** (MPL-2.0, no C++ multi-process bridge). No new top-level crate is mandated; the producer lives feature-gated alongside the compositor/overlay area ([ADR-0016](ADR-0016.md)'s "one new feature-gated module, no new crate" precedent).

2. **Producer/sampler split — the browser never touches the output clock (inv #1/#10).** The engine runs on its own thread(s)/process, renders, and **publishes** the finished surface into a **lock-free latest-slot store** (the generic `TileStore<T>` substrate, [ADR-0058](ADR-0058.md)). The engine `read_at`s per tick (`f(tick.pts)`); it **never awaits the browser**. Command/data into the graphic ride a **bounded drop-oldest RCU seam** (the `AudioControlHandle`/subtitle-seam shape, [ADR-0059](ADR-0059.md) §1). A graphic that GC-pauses, hangs JS, crashes its renderer, or renders slowly **holds last-good** (inv #2) and rides LIVE→…→NO_SIGNAL; persistent loss drops the layer/keyer to off-air-safe ([ADR-R011](ADR-R011.md)).

3. **Cadence via external BeginFrame — plus a deterministic authoring subset for frame-exactness.** Multiview issues BeginFrame at the **output cadence as exact rationals** (inv #3 — never float fps), replacing the browser's internal frame timer (`SendExternalBeginFrame` semantics), so a graphic's animation advances by tick, not by an independent timer that would beat against the program. A paint that misses its tick is **not awaited** — the engine samples last-good and the graphic catches up. **External BeginFrame alone does not make a page deterministic** (JS timers, async/font/network loads, layout, `Date.now`/`Math.random`/`rAF` are not synchronised by it); frame-exact bakes and production-tier use additionally require the **deterministic authoring subset** (brief §2.5: offline bundle, fonts pre-loaded, no network, BeginFrame-derived clock/seeded random, rAF-only animation, a paint-deadline policy), **conformance-validated at import**.

4. **Two consumption tiers, one data contract.** **Monitoring tier** (clocks/bugs/lower-thirds, no program-video sampling, drop-oldest tolerable): the overlay portable RGBA atlas+quad, off-thread egress bake ([ADR-R008](ADR-R008.md)/[ADR-0016](ADR-0016.md)). **Production tier** (must land on the take tick, or samples program video — wipe / PIP+scorecard): the surface is converted **off-thread** to **NV12+A** ([ADR-0058](ADR-0058.md)) and consumed as a keyer/multi-box fill at the verified insertion point ([ADR-0056](ADR-0056.md)). The author selects the tier; the injection/N-source contracts ([ADR-0081](ADR-0081.md)) are identical across tiers. **Frame-exactness bound:** what is on-clock is the transition **progress** (driven by the transition engine, [ADR-0055](ADR-0055.md)) and the **bounded per-frame manifest** the producer publishes (placeholder-rect transforms, a static/baked alpha mask, scalar progress) that Multiview evaluates as `f(tick)` — the engine never reads back arbitrary browser layout per tick, and a late surface/manifest samples last-good (so a per-frame web-authored *look* can be one tick stale). Genuinely frame-critical web-authored mask/geometry animation belongs in render-to-VT ([ADR-0082](ADR-0082.md)), where every frame is exact ([ADR-0081](ADR-0081.md) §4 pins the extraction contract).

5. **Color: take the surface, not the render pass (inv #8).** The browser emits **sRGB premultiplied RGBA**. Multiview declares it premultiplied once, linearizes (sRGB EOTF), blends in linear light, maps to canvas transfer/primaries for HDR ([ADR-C005](ADR-C005.md) 203-nit anchor), then OETF/encode/tag — the fixed pipeline order is untouched ([ADR-C006](ADR-C006.md)). The browser's own 8-bit sRGB framebuffer blend is **bypassed** (the [ADR-0016](ADR-0016.md) rule). Premultiplied-vs-straight is declared once, never double-applied. HDR-native graphic authoring is out of scope for v1 (the NV12+A mezzanine is 8-bit).

6. **Surface capture: CPU-readback MVP, accelerated path as a per-platform optimization.** Accelerated shared-texture OSR (`OnAcceleratedPaint` → a GPU texture, no GPU→CPU→GPU round trip) is most mature on Windows/D3D11, which Multiview does not target. The **MVP path on Linux/macOS is the CPU surface** (`OnPaint` → a premultiplied RGBA buffer we upload), accepting and budgeting the copy exactly as [ADR-0016](ADR-0016.md)/[efficiency §2.2](../research/efficiency.md) budget the overlay upload (~249 MB/s for a full 1080p RGBA layer; reduced by OSR damage-rect / partial paint). Accelerated capture into wgpu is gated behind capability detection (consistent with "no stable wgpu external-texture import," [efficiency §3](../research/efficiency.md)), not the MVP contract.

7. **Crash containment + supervision.** A renderer-process crash is contained and the supervisor restarts the graphic in the background while the store holds last-good, reporting `recovering` (the supervised-reconnect-bracket posture, [media-playout §14](../research/media-playout.md)).

8. **Efficiency budget (standing review).** The browser binary loads only when a graphic exists; each live graphic is **admission-accounted as a composite-class load** ([ADR-E007](ADR-E007.md)) with a hard cap on concurrent live graphics; a single shared engine process multiplexes graphics where the backend supports it. **Render-to-VT ([ADR-0082](ADR-0082.md)) is the recommended default for deterministic graphics** — it removes the browser from the hot path entirely.

## Consequences

**Positive.**
- Operators author motion graphics in the most widely-known surface (HTML/CSS/JS) without recompiling the engine; designers ship bundles.
- Zero new invariant risk: the graphic is a producer on the proven last-good substrate, consumed through the already-specified overlay/keyer/NV12+A design contracts ([ADR-R008](ADR-R008.md)/[ADR-0016](ADR-0016.md)/[ADR-0056](ADR-0056.md)/[ADR-0058](ADR-0058.md), all Proposed); the engine never awaits it.
- Default build stays LGPL-clean and GPU-free; the heavy engine is strictly opt-in.

**Negative / cost.**
- The browser is the heaviest dependency in the product (binary size, build, process model, per-instance RAM/CPU/GPU); a `cargo deny` + NOTICE pass is required at integration, and concurrent-graphic caps are load-bearing.
- The CPU-surface readback costs real upload bandwidth (the [ADR-0016](ADR-0016.md) figure); accelerated OSR on our platforms is unproven and deferred.
- A second graphics vocabulary (web graphics vs the native overlay/keyer model) that docs and the UI must keep distinct.

**Neutral / deferred.**
- Engine pin (CEF-first vs Servo-first) re-validated at integration against 2026 Servo embedding/offscreen maturity; in-process vs multi-process default; accelerated-OSR feasibility on Linux/macOS — all open questions in the brief §7.
- Author-bundle security default is fixed **now** (it is an architectural constraint, not implementation polish): **offline packaged bundles only; no filesystem access outside the bundle; no outbound network unless per-graphic allowlisted; no ambient/host credentials reachable from the page; a CSP enforced; arbitrary Handlebars custom helpers banned ([ADR-0081](ADR-0081.md) §5).** **Remote-loaded graphics are out of scope** until a separate security ADR pins origin model, asset packaging, font loading, credential isolation, and the allowlist grammar (brief §7.8). The exact CSP/allowlist/air-gapped profile remains an open question.

## Alternatives considered

- **Let the browser blend into its own framebuffer and composite that.** Rejected: blends in 8-bit sRGB, bypassing linear-light blend (inv #8) and NV12-throughout (inv #5) — the exact mistake [ADR-0016](ADR-0016.md) rejected for `glyphon`. We take the surface, not the render pass.
- **Run the browser on (or synchronously with) the engine thread.** Rejected: a browser is unbounded-latency (JS, GC, layout, network); any synchronous coupling can stall the output clock (inv #1/#10 violation by construction). Producer/sampler split is mandatory.
- **Drive animation by the browser's internal frame timer.** Rejected: an independent timer beats against the output cadence (judder, drift — float-fps class error, inv #3). External BeginFrame at exact-rational cadence is required.
- **System WebView (Wry/Tao) for the render path.** Rejected: depends on the host OS WebView, no external-BeginFrame, no broadcast-grade offscreen capture. Fine for the *control UI*, not graphics rendering.
- **Default-on / no feature gate.** Rejected: the binary footprint and license set are incompatible with the LGPL-clean, no-native-dep default ([conventions §7](../architecture/conventions.md)). Opt-in only.
- **A pure-Rust non-browser motion-graphics DSL instead of HTML.** Rejected as the primary: it abandons the operator's ask and the enormous HTML/CSS/JS authoring ecosystem; the existing native overlay paths ([ADR-0016](ADR-0016.md)) already cover the must-never-fail elements without a browser.

## How this honors the invariants

- **#1 (output clock):** the browser is sampled (`read_at`), never pacing; external BeginFrame advances animation by tick; a late/slow/crashed graphic holds last-good — the clock never de-paces or stalls.
- **#10 (isolation):** browser on its own thread/process; bounded drop-oldest data + command RCU seams ([ADR-0059](ADR-0059.md) shape); the engine never awaits the page; a CI chaos gate (wedge / kill / starve the graphic; the clock keeps ticking, RSS bounded — the MP-1 pattern, [ADR-0037](ADR-0037.md)/[ADR-0059](ADR-0059.md)) is required for the integration PR.
