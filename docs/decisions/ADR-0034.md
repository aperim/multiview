# ADR-0034: Per-stream decoupled routing + instant crosspoint switching

- Status: **Proposed**
- Date: 2026-06-07
- Deciders: engine + control-plane maintainers
- Related: **ADR-0030** (multi-program / `ProgramSet` + `SourceRegistry`),
  **ADR-R004** (pin output session params / seamless scene-graph swap /
  make-before-break), **ADR-R005/R006** (program bus + discrete tracks + R128),
  **ADR-0019/0024** (unified `CaptionCue` + cue store + HLS SUBTITLES discovery),
  ADR-0010 (config schema / never-untagged enums), ADR-W008 (command bus),
  ADR-RT002/RT003 (realtime envelope/snapshot), ADR-T002/T003 (last-good / timing),
  ADR-0025/0026 (encode fan-out)
- Brief: [docs/research/decoupled-routing.md](../research/decoupled-routing.md)

## Context

Inputs, layouts and outputs are not independent resources today: a layout cell
binds a whole **input** (`cells[].source.input_id`), audio routes key an input
(`AudioRoute`, assuming one audio stream per input), and outputs are nested in a
program with no operator id. An input, however, is a **bundle of elementary
streams** — 1+ video, multiple audio tracks (each with a BCP-47 language + channel
layout), subtitle/caption tracks, SCTE-35/data, timecode. The operator needs the
broadcast **router + multiviewer crosspoint** model: every elementary stream
discovered, visible in model/API/UI, and **independently routable** — *audio /
subtitle breakaway* (video from input A, audio track 2 from input B, subtitles
from input C, simultaneously) — with **instant** (sub-frame-seamless where the
output session is untouched) switching of input→cell, audio-track→bus, subtitle→
layer, and layout/input→output.

The machinery to build on already exists and is verified in code:

- **Demux enumeration** (`crates/multiview-ffmpeg/src/demux.rs:301`) already emits
  **all** container streams with `language` (`:318`) — but the result is
  **discarded** (`multiview-input/src/libav.rs` consumes only `best_stream(Video)`).
- **`MediaKind`** is exactly `Video|Audio|Subtitle|Other`
  (`convert.rs:29-41`); `best_stream` returns `None` for `Other` — so SCTE-35 (already
  discovered as PIDs in `mpegts/selection.rs:42`), timecode and KLV (zero matches
  repo-wide) are **not routing kinds**.
- The **frame-boundary apply seam** exists: `run_inner` drains
  `control(&mut self.drive)` between `clock.tick()` and `drive.compose()`
  (`multiview-engine/src/runtime.rs:411/420/424`) over a bounded
  non-back-pressuring bus (`command.rs:242/264`).
- The **video swap** path re-solves the whole layout (`solve_layout()` allocates,
  `control.rs:152/170`) rather than re-pointing O(1).
- The **program bus** pulls `samples_per_tick` on the output `SampleClock` and
  silence-fills (`multiview-audio/src/program.rs:81`); `route_to_program`/
  `mix_program` sum gains (`mixer.rs:90/148`) — but `bus.tick()` is driven by
  **surviving** video frames (`pipeline.rs:1915`, policy `DropOnOverload`), not the
  tick index, and there is **no method to swap a channel's `Arc<AudioStore>`**.
- **GP-6 `RestampAccumulator`** (`restamp.rs:40/82/106`) and **GP-1 `is_idr`**
  (`idr.rs:77`, true-RAP) are real, pure, default-build, reusable verbatim — but
  have **zero non-test callers**, and there is no `GuardedPacketSource`, no runtime
  sink-mover, no `ProgramSet` in the engine.

Full as-built-vs-missing table and the adversarial review results are in the brief.

## Decision

Lift the per-program binding key from `input_id` to a stable
**`StreamEndpoint = (input_id, StreamKind, selector)`** and make the three
already-existing crosspoint maps independently keyed, so **breakaway falls out
structurally**. Specifically:

1. **`StreamInventory` discovery.** Generalise the libav path to stop discarding
   non-video streams; fold in TS PMT `ElementaryStream`/`scte35_pids` + typed
   ES-descriptors and HLS `MediaRendition`. Define `StreamKind` SUPERSETTING
   `MediaKind` with `Data{Scte35|Klv}` and `Timecode{TcSource}` (KLV is new). Key
   each stream by a **kind-scoped `StableStreamId`** (TS=PID, HLS=group_id+name,
   general=kind+ordinal+codec+lang+title hash) that survives re-probe /
   PMT-version / rendition reorder; tag each with a **key-stability tier**
   (hard/soft) and surface it. Validate `language` to BCP-47.

2. **Per-stream routing model.** Add a top-level `routing: Option<RoutingTable>`
   (a `RouteMatrix` per program) with `StreamRef{input_id,kind,selector}` and an
   internally-tagged `StreamSelector` (`Index|Language|Best|StreamId`, never
   untagged). Key audio routing per `(input_id, selector)`. Bump
   `schema_version` to **3** and **desugar** an absent `routing` so v1/v2 docs
   route identically (sugar-superset). Key `SourceRegistry` per elementary stream;
   ref-count per-stream so a spare spins a decoder only on first reference.

3. **Instant in-program re-point (Class-1).** Build `CompositorDrive::rebind_cell`
   (O(1) cell→source map, skip `solve_layout`/`validate` on a pure source
   re-point), pre-resolve layouts at ARM time, coalesce a drained batch
   (≤1 re-solve/tick), and cap commands per tick. Decode routable sources at a
   canonical size + **scale-at-composite** (so a cross-geometry swap is correct),
   and gate every take behind **WARM-ON-ARM** (ref-count up + bounded control-side
   prime-wait) so the take frame is never a slate.

4. **Audio breakaway + pop-avoidance.** Build `ProgramBus::repoint(RoutePoint,
   Arc<AudioStore>)` with an absolute-tick-indexed cursor (seek to the live edge on
   re-point). Drive `bus.tick()` from the **output tick index** (catch up across
   `DropOnOverload` gaps) so lip-sync (inv #3) is structural. Add an **intra-block
   (per-sample) equal-power gain envelope** to the mixer for a ~10 ms cross-fade.
   Surface two tiers: **SOFT-STEP** (coded passthrough, video-clean, Class-1) vs
   **CLICK-FREE** (decode→mix→re-encode, transcode floor, Class-2).

5. **Subtitle breakaway.** Re-point the layer's `Arc<CueStore>`; CLEAR-on-switch
   for the boundary frame; hard cut. Class-1.

6. **Output ← program splice.** Add a stable output id + `OutputRef{output->
   program}`. **Seamless** = move the sink registration to the target's
   `PacketRouter` and rebase per stream (GP-6 per egress coded stream, video at a
   true IDR via GP-1, audio at an AU coincident with the video IDR) — zero new
   encode if the target is already encoding (inv #7). **Class-2 / MBB** = ADR-R004
   make-before-break with `OutputRunState::Migrating` + container discontinuity,
   admission-gated. **Converge** the two `EncodedPacket` types and build the
   runtime sink-mover + the egress GP-7 splicer (all unbuilt today).

7. **SCTE-35 continuity.** Build the data-PID restamp sibling: add `Data{Scte35}`
   to the egress packet model and a `SpliceInfoSection::reserialize_with_
   pts_adjustment(new_adj)` (recompute CRC-32/MPEG-2 via the existing
   `mpegts/crc.rs`). At a seam, shift `pts_adjustment` by the **same** GP-6 offset
   (90 kHz, mod 2^33); the data PID rides the video seam. On HLS, re-derive
   `EXT-X-DATERANGE` from the re-stamped tick timeline.

8. **Atomic mixed salvo.** Extend `Salvo` (config), `SalvoChange`/`SalvoBatch`
   (engine) and the OpenAPI `SalvoDoc` mirror in lockstep with `audio`/`subtitle`/
   `output` recalls; lift `SourceRecall` to a `StreamRef`. Add additive
   `Command::RouteVideo/RouteAudio/RouteSubtitle/RouteOutput` (`#[non_exhaustive]`);
   `SwapSource` becomes the desugared alias for `RouteVideo{...,Video,Best}`.

9. **API / realtime / UI.** `GET /inputs/{id}/streams` → `StreamInventory`
   (off-engine). `POST /routing/{kind}/take` returns the #11 class first
   (`200{class1,applied}` vs `202{operation_id}`); `POST /routing/plan` classifies
   without applying and inspects the **destination's pinned params**, returning
   `{class1, reset_lite, class2}`. Add `Topic::Routing` + `routing.*` deltas +
   `input.streams`, published through the existing wait-free `LatestState` +
   drop-oldest broadcast (folded into the conflated snapshot blob; read-only).
   A Router crosspoint MATRIX UI (tabs per stream-type, breakaway badge, Cut+Take).

10. **#11 classification is honest at the edges.** In-program re-point onto an
    **existing** destination is Class-1; a re-point requiring a single IDR is
    **Reset-lite**; an audio/subtitle breakaway onto a **discrete output track**
    whose pinned layout/codec/track-set would change is **Class-2** (or an
    operator-confirmed coerced Class-1-with-degradation); output←program with
    mismatched pinned params is **Class-2** (MBB). The classifier compares the
    source's `StreamKind` payload against the destination's pinned
    `StreamCodecParameters` (the `sink.rs` snapshot).

## Consequences

**Positive.**
- Inputs, layouts and outputs become independent resources wired by per-stream
  crosspoints; audio/subtitle breakaway is structural, not a special case.
- In-program switches are Class-1 frame-boundary re-points: no encoder/session
  reset, decode-once and encode-once preserved (inv #5/#7) — zero redundant
  decode/encode when the target is already running.
- Fully backward-compatible: an absent `routing` desugars to today's behaviour;
  `SwapSource` and existing salvos keep working via desugar targets.
- Reuses ADR-0030/R004/R005/0019 and GP-6/GP-1 verbatim; the new surface is the
  inventory, the route matrix, the O(1) re-point primitives, the audio cross-fade,
  the egress splicer, and the SCTE-35 restamp target.

**Costs / risks (all surfaced by adversarial review; none silently assumed away).**
- **The O(1) re-point is not as-built** — the current swap re-solves the layout.
  Until `rebind_cell` + coalesce + cap-per-tick land, a salvo storm of K commands
  is K re-solves/tick. Mitigation is mandatory, not optional.
- **Cross-geometry video swap is visually broken** without scale-at-composite (a
  source decoded at one cell's size painted into a different cell clips/smears).
  This is the normal router case — it must ship with the scale fix.
- **Lip-sync is not structural** until `bus.tick()` is tick-index-driven; under
  `DropOnOverload` audio drifts. A held-out test must assert no drift across an
  overload + breakaway take.
- **No audio re-point primitive, no egress splicer, no runtime sink-mover, no
  `ProgramSet`, no SCTE restamp target** exist today — these are net-new builds,
  not wiring of existing code.
- **`output ← program` routing depends on ADR-0030 landing.** Until then it
  degenerates to the single `"main"` program (the desugar default); gate the
  program-target affordance behind a capability flag.
- Soft-keyed `StableStreamId` (same-codec/same-language tracks with no other
  discriminator) is a known reorder risk — surfaced as a tier badge, not hidden.

## Alternatives considered

1. **Per-input crosspoints only (no breakaway).** Rejected — the operator's
   explicit requirement is per-stream; a per-input matrix cannot express
   audio/subtitle breakaway and would duplicate, not extend, the existing
   per-stream audio/cue machinery.
2. **Bind by container `index`.** Rejected — the demux snapshot is open-time-only
   and `index` is volatile across re-probe/PMT-version/rendition reorder
   (`param_probe.rs:7-11`); a crosspoint bound by index silently mis-routes after a
   re-probe. The kind-scoped `StableStreamId` exists precisely to survive this.
3. **Untagged serde unions for `StreamSelector`/`RouteMatrix`.** Rejected per
   ADR-0010 — untagged enums are fragile across TOML/JSON; internally-tagged only.
4. **Hard-cut audio (no cross-fade).** Rejected as the default — a sample-accurate
   hard cut is waveform-discontinuous and clicks. Hard cut is reserved for
   silence↔source and explicit operator cut; the equal-power envelope is the
   default for breakaway.
5. **Per-tick scalar gain for the cross-fade** (via the existing
   `route_to_program` gain). Rejected — at a 20-40 ms tick block a 10 ms fade steps
   in 1-2 coarse jumps and still clicks; an intra-block per-sample envelope is
   required.
6. **A new engine-awaited channel for routing state to the control plane.**
   Rejected per inv #10 — routing/inventory ride the existing wait-free
   `LatestState` slot + drop-oldest broadcast, read-only from the engine.
7. **Leave SCTE-35 `pts_adjustment` unshifted across a splice** (rely on the
   monitoring decode only). Rejected — passed-through ad cues would fire at the
   wrong instant relative to the re-stamped video (SCTE-35 2023r1 mandates shifting
   `pts_adjustment` for a re-stamping passthrough).
8. **Universal "all in-program re-points are Class-1".** Rejected — the capability
   matrix classifies discrete-track layout/track-set changes as Class-2 and defines
   a Reset-lite tier; the classifier must inspect the destination's pinned params,
   not just whether the op stays in-program.
