## Account / Conspect (account-side subsystem)

The **account-side** subsystem вЂ” entitlement/licensing, local mesh, two-pipe heartbeat/telemetry, and support вЂ” that **never touches the protected output core** (inv #1). Backed by [conspect-account-architecture](../research/conspect-account-architecture.md); `CONSPECT-*` backlog in [work-schedule](../development/work-schedule.md).

- [ADR-0050](ADR-0050.md) вЂ” Conspect entitlement: NEW leaf crate `multiview-licence` (depends only on `core`+`events`; `heartbeat` feature-gated, state model always compiled) owning the entitlement/lease/claim/transfer state machines + the **enforcement-ladder-AS-DATA** (`enforcement.level` on the licence resource: `active`/`warning`/`config-locked`/`watermark`/`block-new-instance`/`unlicensed-build`) + salted-digest fingerprint **score** (`FINGERPRINT_MATCH_THRESHOLD = 70`/`STRONG = 100`). EXACT lease constants (`LEASE_FULL`=35d, `LEASE_GRACE`=14d, `LEASE_HARD`=90d, `ACTIVATION_WINDOW`=31d; claim `CLAIM_CODE_TTL`=15m/`CLAIM_REJECT_WINDOW`=24h/6-char ambiguity-free code; `TRANSFER_WINDOW`=72h). **Never-off-air** (inv #1/#10): the engine reads exactly TWO pre-derived atomics on the hot loop (config-lock S2 + watermark S3) + a startup gate S1 вЂ” `block-new-instance` blocks a NEW instance, the **running program stays on air on every rung**; enforcement fails toward leniency. **Honest source-build caveat**: heartbeat can be compiled out в†’ `unlicensed-build` reported honestly, licence binds regardless, official binaries are the licensed convenience. Ed25519 signing *(Proposed)*
- [ADR-0051](ADR-0051.md) вЂ” Conspect mesh: NEW leaf crate `multiview-mesh` (depends `licence`+`core`+`events`; `mdns` feature-gated) вЂ” **always-on** IPv6-first mDNS (`ff02::fb`, IPv4 `224.0.0.251` legacy) announce/browse of **Ed25519-signed** salted-digest summaries (never raw identity); discovered peers enter an **untrusted inventory** with explicit operator **confirm-adopt** (the ADR-0041 doctrine); **end-to-end-signed relay** so an offline machine heartbeats via an online LAN neighbour that is a **dumb carrier** (cannot read/forge/alter the assertion вЂ” relay integrity never depends on trusting the relayer); opt-in per machine, bounded drop-oldest, off-hot-loop control-plane actor (inv #10) *(Proposed)*
- [ADR-0052](ADR-0052.md) вЂ” Conspect two-pipe model: **mandatory monthly licensing heartbeat** vs **opt-in daily product telemetry** вЂ” NEVER co-mingled (separate transport/consent/copy/API/retention; the UI must never present them as one switch). Telemetry consent is a versioned single-value doc resolved **last-writer-wins** by `updated_at`; the heartbeat consent is implicit in running the official build (not in the doc). **Consent-independent** local metrics retention (engine seam S5): turning telemetry OFF stops the outbound daily pipe but NOT the bounded auto-pruned local buffer that feeds the operator's own support context-pack. Data-minimisation: salted digests + aggregated/anonymised counters only, never raw serials/MACs/media/URLs. Extends `multiview-telemetry` (additive, off-hot-loop, inv #10) *(Proposed)*
- [ADR-0053](ADR-0053.md) вЂ” Conspect support: ticketing + **redacted** context-pack (local metrics buffer + licence/enforcement state + config-as-code with **reference-only secrets** per ADR-M006 вЂ” diagnostics only, never media/raw-identifiers/secret-values) + **data-request local approval** (no data egress without an explicit operator approval вЂ” surfaced as a pending action) + **append-only** account audit (claim/transfer/lease/enforcement/consent/context-pack/data-request вЂ” no update/delete route, immutable timestamped entries; in-memory default, sqlite under the `sqlite` feature per ADR-I003). Extends `multiview-control` (additive routes/stores, off-hot-loop, inv #10) *(Proposed)*

## Production Switcher

The **production-switcher layer** вЂ” M/E (mix/effects) stages with program/preview (PGM/PVW) buses, transitions, keyers/FTB, media playout, and switcher audio вЂ” evaluated **per tick inside one Program**, so no transition, T-bar move, keyer, player, or macro can ever pace or back-pressure the engine (inv #1/#10 by construction). Backed by the [production-switcher](../research/production-switcher.md) + [media-playout](../research/media-playout.md) briefs; `SW-*` backlog in [production-switcher-backlog](../development/production-switcher-backlog.md). The companion area ADRs live in their own sections below: [ADR-M012](ADR-M012.md) (Management), [ADR-RT008](ADR-RT008.md) (Realtime API), [ADR-W021](ADR-W021.md) (Web/API Stack), [ADR-P007](ADR-P007.md) (Preview), [ADR-T015](ADR-T015.md) (Streaming/Timing), [ADR-C007](ADR-C007.md) (Color), [ADR-MV006](ADR-MV006.md) (Broadcast Multiviewer), [ADR-R011](ADR-R011.md) (Resilience & A/V).

- [ADR-0054](ADR-0054.md) вЂ” Production switcher layer: an M/E (mix/effects) stage lives **inside ONE [ADR-0030](ADR-0030.md) Program** вЂ” PGM and PVW are two scene states composed against the **same `Tick`** (the PVW bus is a second compose, never a sibling program: independent clocks break frame-aligned take; mix/wipe math is impossible across encoded streams); the switcher state machine is a **pure value machine** at the frame-boundary control seam (the per-tick hook **widened to receive the `Tick`**, salvo/tally/alarm house style) and the per-tick placement derivation is promoted to an injectable **render-plan resolver** `fn(scene_states, switcher_state, tick) в†’ placements` (allocation-pooled, synchronous, inv #1 by construction); **build-vs-route split**: anything that must blend rides the in-program render plan, feeds that genuinely need their own encode (aux buses) are ADR-0030 programs via the RT-12 outputв†ђprogram crosspoint (inv #7); **DSK/FTB composite ON-CLOCK** inside the render plan (the off-thread egress overlay bake is not frame-coordinated and stays for monitoring overlays only); clean feed = the **pre-DSK tap** on the already-published per-tick pre-overlay canvas `Arc` (ADR-0037 CLEAN/DIRTY vocabulary); bus-tap taxonomy `{program (post-FTB), clean (pre-DSK), preview, me[n], aux[j]}`; **no new crate** (ADR-I004 precedent вЂ” engine/compositor/core/config/control/web modules only) *(Proposed)*
- [ADR-0055](ADR-0055.md) вЂ” Transition engine: progress is a **pure function of the tick index** вЂ” `clamp((tick.index в€’ start_tick)/d, 0, 1)` as an exact rational with durations in **integer frames** ([ADR-T015](ADR-T015.md)), deterministic/replayable under `ManualTimeSource`, *sampled never pacing* (inv #1); taxonomy cut В· mix{rate} В· dip{rate, dip_source, switch_point} В· wipe{SDF-mask pattern, softness, border, position, reverse} В· DVE push/squeeze В· stinger media transition{pre_roll, trigger_point, mix_rate; validated trigger+mix в‰¤ duration}; Wave 1 = cut/mix/dip/FTB with mix on the **flat-list fast path** (both scenes' cells in one z-ordered list, incoming opacity ramped вЂ” zero compositor changes) and scene pre-render only for internally-overlapping scenes detected at arm time; Wave 2 wipes + DVE (finally wiring the modeled-but-ignored fit/crop/rotation), Wave 3 stinger; **T-bar = conflated idempotent absolute progress setter** (latest-wins, never one command per sample); flip-flop PGMв†”PVW swap on completion (default on, per-M/E), direct program punch, preview-transition, next-transition arming {background, key[1..n]} with DSK TIE/CUT/AUTO; degradation: an in-flight transition is **never shed mid-flight** (complete-then-degrade), under overload AUTO demotes to CUT at arm/admission time, the PVW compose sheds before program tiles *(Proposed)*
- [ADR-0056](ADR-0056.md) вЂ” Keyers: **upstream keyers** are M/E-scoped scene layers that participate in transitions; **downstream keyers** are a canvas-level post-M/E pre-FTB chain in explicit z-order; **FTB is the final master stage after the DSKs** (fades everything together, own rate, optional audio-follow per [ADR-0059](ADR-0059.md)); `KeySource{fill, key: Option, premultiplied}` + key types linear/fill+key В· luma В· chroma В· pattern В· dve_pip (MVP: linear/alpha + luma + dve_pip) with clip/gain/invert and a rectangular garbage matte; key alpha is computed at the **verified inv #8 insertion points** вЂ” luma on post-range-expand code-value Y, chroma in linear RGB after YUVв†’RGB вЂ” and multiplied in before the premultiplied linear-light `over`; fill+key = an optional second per-tile source reference (the GPU already binds per-tile texture-array layers); **multi-box composition** = the existing background+overlapping-z-ordered Layout model exposed as a bus-selectable source rendered as a drive-internal **same-tick pre-pass** (never a chained ProgramSet program вЂ” that costs a tick of latency + NV12 generation loss) *(Proposed)*
- [ADR-0057](ADR-0057.md) вЂ” Media library + media players: the **library** is desired-state asset storage (stills/clips/audio + a probe-at-import validate/reject-or-transcode pipeline) and **players** are a bounded set of bus-selectable source *channels* publishing into ordinary `TileStore`/`AudioStore`s (transport state engine-owned, mirrored read-only); transport extends the **production CLI file-ingest path** (`ingest_loop`/`PtsWallClock` pacing вЂ” explicitly NOT the unwired `multiview-input` `IngestPump` scaffold): cue = open + decode-first-frame + pause gated on `is_primed`, play/pause = anchor/suspend the pacer, seek = the zero-caller `Demuxer::seek`, loop = in-place seek-to-in-point on clean EOF (`Demuxer::seek` + decoder flush; a failed seek falls back to the supervised reconnect bracket), EOF policy {hold_last_frame В· loop В· black В· auto_off}, play-on-take, frame-accurate start; still store = decode-once + `HoldForever` (the proven EOF-hold path); protocol posture: native API only for v1 with cue/auto-chain semantics matching the open CasparCG AMCP precedent (VDCP/AMP RS-422-era external-server protocols out of scope) *(Proposed)*
- [ADR-0058](ADR-0058.md) вЂ” Alpha media path: an **optional NV12+A frame payload** вЂ” NV12 + one full-resolution R8 alpha plane, **2.5 B/px** vs 4.0 RGBA вЂ” for media-player/graphics-fill sources ONLY, riding the generic `TileStore<T>` unchanged (inv #2); never per-tile RGBA video (inv #5) and never bolted onto the input-decoupled overlay stack; alpha is **straight in the payload and premultiplied exactly once, in linear light, in-kernel** at the existing premultiply step (inv #8 order untouched); format policy verified against FFmpeg: accept ProRes 4444/4444 XQ, `qtrle`, PNG/TGA sequences, VP9-alpha via `libvpx-vp9` вЂ” **HEVC-with-alpha decodes opaque в†’ reject or import-transcode** (FFmpeg 8.0 added only libx265 alpha *encoding*); stinger mezzanine: pre-decoded at load into a bounded pool-allocated in-memory NV12+A ring indexed by tick offset (3 s 1080p60 в‰€ 0.93 GB vs в‰€ 1.49 GB RGBA; per-player + global budget, admission-accounted; decode-ahead ring beyond the cap) вЂ” zero decode/allocation on the take path *(Proposed)*
- [ADR-0059](ADR-0059.md) вЂ” Switcher audio: first slice is the **audio control seam** вЂ” an `AudioControlHandle` RCU slot (the subtitle seam's twin) drained once per `StreamItem` by the bake consumer, finally giving live commands a path to the BUILT-and-tested pop-free nucleus (per-sample equal-power `GainRamp` + `ProgramBus::repoint_crossfade`, today HELD with no production seam); **AFV** per input {fade_with_transition (default) В· hard_cut} with `ramp_frames = SampleClock::total_at(s + T) в€’ SampleClock::total_at(s)` (exact cumulative sample budgets вЂ” never the naive `T Г— samples_per_tick` product, off by up to a frame at NTSC), driven by the SAME switcher tick window as video; a master/program gain stage (one multiply pass in `ProgramBus::mix`) carries FTB audio fade + master gain; **gain-preserving per-strip mute** (today mute = unroute вЂ” clicky and gain-destroying); honor the currently-destructured-away `RouteIntent::Audio { gain_db, mute }`; per-bus audio = one `ProgramBus` per audio-bearing bus, with the single-cursor `AudioStore` needing a multi-cursor reader view flagged as THE real data-structure change; `Event::AudioMeter` wired (~30 Hz conflated; DSP built, zero emitters today) before the switcher UI ships; program-bus crossfade re-classified **Class-1** (this path is always decodeв†’mixв†’re-encode) and program audio becomes config-declared *(Proposed)*
