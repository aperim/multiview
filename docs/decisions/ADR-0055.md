## Context

No video transition machinery exists anywhere in the codebase: `set_layout`/`rebind_cell`
are atomic frame-boundary cuts (BUILT — `crates/multiview-engine/src/drive.rs:318,376`),
the only ramp in the repo is the audio equal-power `GainRamp`
(`crates/multiview-audio/src/mixer.rs:35`, reachable live only via tests), and the
compositor's `Tile` carries exactly an integer destination rect + one uniform opacity
(`crates/multiview-compositor/src/pipeline.rs:413-427`) — no masks, no transforms, no
sub-pixel placement. What IS load-bearing for a transition engine: **per-cell opacity is
wired end-to-end** (config `cell.opacity` → core `Cell.opacity`
(`crates/multiview-core/src/layout.rs:152`) → drive `Placement.opacity`
(`crates/multiview-engine/src/drive.rs:492`) → the premultiplied linear-light `over` on
both backends, GPU uniform at
`crates/multiview-compositor/src/gpu/uniforms.rs:289-290`), while **crop/rotation are
type-level only** (the config mapper spreads their defaults,
`crates/multiview-config/src/lib.rs:543-546`; no composite path consumes them) and the
GPU `TileParams` carries padding lanes where per-tile mask/transform parameters can ride
(`dst_size.zw` + `opacity_transfer.zw`,
`crates/multiview-compositor/src/gpu/shaders/composite.wgsl:17-21`). The tick is pure and
exact: `out_pts = f(tick)` (`crates/multiview-engine/src/clock.rs:222-227`), cadence an
exact `Rational` (`crates/multiview-core/src/time.rs:35`). This ADR pins the transition
engine that [ADR-0054](ADR-0054.md)'s switcher state machine evaluates and its render-plan
resolver renders.

## Decision

1. **Progress is a pure function of the tick index — nothing else.** For an in-flight
   transition armed at `start_tick` with duration `d` frames,
   `progress(tick) = clamp((tick.index − start_tick) / d, 0, 1)` as an **exact rational**
   (integer numerator/denominator); the optional easing/perceptual curve of
   [ADR-C007](ADR-C007.md) is applied to this scalar `t` — never to pixels. The one lossy
   conversion is the final `f32` handed to the compositor as a per-tile opacity/mask
   parameter, at the last moment (the same boundary discipline as the clock-hand angles
   in [ADR-0047](ADR-0047.md)). The switcher state machine receives the `Tick` at the
   widened frame-boundary hook ([ADR-0054](ADR-0054.md) Decision 2,
   `crates/multiview-engine/src/runtime.rs:430-439`), so progress is deterministic,
   replayable under a `ManualTimeSource`, and *sampled* — a transition can never pace,
   stall, or skip the clock (inv #1).

2. **Durations are integer frame counts at the exact-rational output cadence; the API
   accepts milliseconds and converts exactly.** A rate is stored as `frames: u32 ≥ 1`. A
   `duration_ms` input converts via the cadence rational
   (`frames = round(ms × fps_num / (1000 × fps_den))`, exact integer arithmetic on
   `Rational` (`time.rs:35`), minimum 1) and is **echoed back in frames** so the operator
   sees what will actually run. Never float fps, never float seconds (inv #3, safety
   rule #6). All derived timing — stinger trigger frames, AFV sample budgets via
   `SampleClock::total_at` deltas (`crates/multiview-audio/src/cadence.rs`,
   [ADR-0059](ADR-0059.md)) — comes from the same integer frame window. Arithmetic
   details and the NTSC cases are pinned in [ADR-T015](ADR-T015.md).

3. **Taxonomy and exact parameter sets** (generic industry vocabulary; behaviour labelled
   de-facto industry practice where no standard exists; all serde unions internally
   tagged per [ADR-0010](ADR-0010.md)):
   - **`cut`** — no parameters. The existing atomic frame-boundary swap
     (`rebind_cell`/scene swap); also the degenerate case of every timed transition.
   - **`mix`** (dissolve) — `{ rate }`. Linear-light premultiplied blend of outgoing and
     incoming scenes weighted by `t` ([ADR-C003](ADR-C003.md); domain decision in
     [ADR-C007](ADR-C007.md)).
   - **`dip`** — `{ rate, dip_source, switch_point }`. Two chained mixes through
     `dip_source` (any bus-selectable source, including solid-color generators —
     [ADR-0027](ADR-0027.md) synthetics): scene→dip over the first phase, dip→scene over
     the second. `switch_point` is an exact rational in `(0, 1)`, **default `1/2`**
     (adjustable dwell split is de-facto practice in open implementations); the phase
     boundary frame is `⌊d × switch_point⌋` — deterministic integer arithmetic, with `d`
     converting from `rate_ms` via [ADR-T015](ADR-T015.md) §2 (round-half-up, min 1) and
     the boundary/degenerate-`d` edge rule recorded in [ADR-T015](ADR-T015.md) §3 (dips
     validate `d ≥ 2`; the boundary clamps into `[1, d − 1]` so each phase gets at least
     one frame).
   - **`wipe`** — `{ pattern, softness, border_width, border_fill_source, position_xy,
     reverse, flip_flop }`. `pattern` names a closed-form SDF mask (the house primitive
     style — `rect_coverage`/`segment_distance`/`ring` in
     `crates/multiview-compositor/src/overlay/subpass.rs` — gives linear-edge, box,
     circle/ring wipes with analytic AA); `softness` widens the SDF edge; the optional
     border band fills from `border_fill_source`; `position_xy` offsets
     origin-positionable patterns; `reverse` mirrors direction; `flip_flop` alternates
     direction on successive takes (de-facto practice). Per-pixel mask alpha is evaluated
     exactly where uniform opacity applies today (the CPU fold loop and
     `composite.wgsl`'s blend step), parameters riding the verified `TileParams` padding
     lanes (`composite.wgsl:17-21`).
   - **`dve_push` / `dve_squeeze`** — `{ direction, rate }`. Animated per-tile affine
     transform (translate for push, scale for squeeze) with sub-pixel placement; requires
     the per-tile transform extension and finally **wires `FitMode`/crop/rotation**,
     which are modeled+validated but ignored by the render path today
     (`lib.rs:543-546`; `cell_dst_rect` uses the raw cell rect,
     `crates/multiview-engine/src/drive.rs:728`).
   - **`stinger`** (media transition) — `{ player, pre_roll, clip_duration,
     trigger_point, mix_rate }`, all timing in frames. The bound media player
     ([ADR-0057](ADR-0057.md)) rolls an alpha clip ([ADR-0058](ADR-0058.md) NV12+A) over
     the M/E; at `trigger_point` the underlying background mix of `mix_rate` frames
     runs while the clip covers. **Validation: `trigger_point + mix_rate ≤
     clip_duration`** (rejected at arm otherwise). A hard cut-point — the alternative
     semantic in some open implementations — is the **degenerate `mix_rate = 1` frame**
     case, so one parameter set covers both behaviours (de-facto practice synthesis).
   - **FTB** is deliberately *not* a transition: it is the master stage after the DSKs
     with its own rate ([ADR-0056](ADR-0056.md)); it shares this ADR's progress and
     duration rules.

4. **Delivery waves, gated by what the compositor verifiably supports.** **Wave 1
   (MVP):** cut, mix, dip, FTB — mix/dip ship on per-cell opacity alone. For scenes with
   **no internal overlap**, the *flat-list fast path*: both scenes' cells in one
   z-ordered placement list, the incoming scene's cells above the outgoing's with opacity
   `t` — the premultiplied linear-light `over` then yields `t·B + (1−t)·A` per pixel with
   **zero compositor changes** (the substrate is wired end-to-end, see Context). For
   scenes **with** internal overlap (PiP over base in one scene) the fast path
   double-blends — a scene must dissolve *as a unit* — so the resolver switches to
   **scene pre-render**: composite each scene to an NV12 intermediate (the compose output
   type-checks as a compose input), then blend two full-canvas tiles — ~2 extra
   composites per tick for the transition window plus one NV12 8-bit re-quantization per
   intermediate generation; on GPU the passes fuse on the pooled linear `Rgba16Float`
   canvas (the `composite_with_overlays` chaining precedent,
   `crates/multiview-compositor/src/gpu/compositor.rs:326`) with one readback. **The path
   is selected at arm time** by a pure overlap scan of both scenes' cell rects — never
   mid-transition. **Wave 2:** wipes (per-tile SDF masks) and DVE push/squeeze (per-tile
   affine + sub-pixel placement + fit/crop/rotation wiring) — the one substantial
   compositor contract extension, on both the CPU reference and wgpu paths (CPU oracle +
   GPU SSIM/PSNR per the existing test pattern). **Wave 3:** stinger (blocked on
   [ADR-0058](ADR-0058.md) alpha media).

5. **The T-bar is an idempotent absolute progress setter, conflated latest-wins.**
   `tbar(position)` sets manual progress to an absolute value in `[0, 1]` — never a
   relative nudge — so a replayed/duplicated/reordered command is harmless. Transport
   delivers it **conflated**: a latest-wins slot drained once per tick at the frame
   boundary (the route-coalescing precedent — last route per destination wins,
   `crates/multiview-engine/src/route.rs:31` — and the conflated-event lane,
   `crates/multiview-events/src/event.rs:1306`), **never one queued command per movement
   sample** (a fader emits hundreds of samples per second; the bounded command bus must
   not carry them — inv #10). Publisher-side cadence is pinned in
   [ADR-T015](ADR-T015.md)/[ADR-RT008](ADR-RT008.md). A T-bar write during an AUTO takes
   ownership (latest-wins, manual from that tick); reaching `1.0` completes exactly like
   an AUTO completing.

6. **Completion, flip-flop, and abort semantics.** At `progress = 1.0` (AUTO reaching
   `start_tick + d`, or the T-bar at `1.0`): the incoming scene becomes PGM, the
   transition state clears, armed keyer elements latch their new on-air state
   ([ADR-0056](ADR-0056.md)), and — with **flip-flop ON (the default, configurable per
   M/E)** — PGM and PVW swap so the outgoing scene sits on PVW ready for the return take
   (de-facto industry practice). With flip-flop OFF, PVW keeps its selection. **Direct
   program punch** (selecting a source straight onto PGM) is supported and is a plain
   cut. `cut()` during an in-flight transition completes it instantly (jump to `1.0`,
   same completion path — never a second transition stacked). **`abort()`** snaps
   progress to `0.0` at the next frame boundary: PGM unchanged, no swap, armed set
   retained (a timed reverse is simply driving the T-bar back — the absolute setter
   already provides it). Lifecycle events `transition.started/completed/aborted` are
   lossless; `transition.progress` is conflated ([ADR-RT008](ADR-RT008.md)).

7. **Next-transition arming is the set `{background} ∪ {key[1..n]}`.** Arming selects
   which elements the next transition moves: the background scene change, and/or any
   upstream keyers whose on-air state toggles through the transition — so **key-only
   transitions work** (background unarmed; the scene stays, a key mixes on/off through
   the selected transition type). DSKs are independent of the armed set, with their own
   TIE (join next transition) / CUT (instant) / AUTO (own rate) controls pinned in
   [ADR-0056](ADR-0056.md). The armed set is validated at arm (every armed key exists;
   at least one element armed) and is part of the switcher value-machine state
   ([ADR-0054](ADR-0054.md) Decision 2).

8. **Preview-transition rehearses on the PVW composite.** When engaged, the PVW bus
   renders `f(PVW scene, PGM scene, progress)` — the exact transition the take would
   produce — while PGM stays untouched; cost is the PVW bus's own admission-accounted
   budget ([ADR-0054](ADR-0054.md) Decision 8, [ADR-P007](ADR-P007.md)). Disengaging
   restores the plain PVW scene. No state escapes to PGM until a real take.

9. **Degradation interaction: never shed mid-transition; demote at arm.** An in-flight
   transition is **program-affecting** — the degradation loop (inv #9,
   [ADR-E007](ADR-E007.md)) must let it **complete, then degrade**
   (complete-then-degrade): shedding the PVW/pre-render passes mid-flight would visibly
   tear the program output, the one thing inv #1's spirit forbids. Under overload the
   demotion point is **arm/admission time**: an AUTO whose transient cost (scene
   pre-render, stinger decode-ahead) cannot be admitted **demotes to CUT** — announced in
   the arm/plan response ([ADR-W021](ADR-W021.md)) and as a lifecycle event, never
   silently. The transition window's transient cost is a first-class admission item
   (sized at arm: flat-list ≈ free; pre-render ≈ 2 composites/tick for `d` frames;
   stinger per [ADR-0058](ADR-0058.md) budgets); the PVW-bus rung
   ([ADR-0054](ADR-0054.md) Decision 8) sits below it in shed order. Source loss
   mid-transition holds last-good per tile and the transition completes on schedule
   ([ADR-R011](ADR-R011.md), inv #2).

## Rationale

- **`f(tick.index)` is the only progress law compatible with inv #1/#3.** It is exact
  (no accumulation, no drift — the same recompute-from-the-counter discipline as
  `pts_at`, `clock.rs:196-201`), deterministic under test, idempotent across replays,
  and structurally incapable of pacing: a missed control delivery changes *nothing*
  about which frame shows which progress.
- **The taxonomy is the cross-vendor consensus superset, expressed once.** Where open
  implementations disagree (dip dwell adjustable vs fixed; stinger triggered-mix vs
  hard-cut-point; rate units frames vs ms) this ADR picks the superset that degrades to
  every observed behaviour: adjustable `switch_point` defaulting to `1/2`,
  triggered-mix with the 1-frame degenerate cut, frames-internal with ms accepted at the
  boundary — each labelled de-facto industry practice, no vendor named.
- **The wave split tracks verified substrate, not ambition.** Wave 1 needs zero
  compositor changes (opacity is wired end-to-end — the one fully-plumbed per-tile
  parameter); Wave 2 is the honest single big compositor work item (masks + transforms +
  the dead fit/crop/rotation fields); Wave 3 is gated on the alpha path. Each wave is
  independently shippable and gate-able.
- **Conflating the T-bar is the only inv-#10-correct transport.** The bounded command
  bus exists to carry intents, not telemetry; an absolute, idempotent setter makes
  latest-wins conflation *lossless in effect* — the tick samples the newest position,
  exactly as the compositor samples the newest frame (inv #2's philosophy applied to
  control).

## Alternatives considered

- **A — Millisecond-based durations end-to-end.** Rejected (inv #3,
  [ADR-T003](ADR-T003.md)): ms does not divide NTSC cadences (`30000/1001`) exactly, so
  per-tick progress would need float seconds (drift, non-determinism) or hidden
  rounding the operator never sees. Frames at the output cadence are exact, replayable,
  and the operator-facing unit of de-facto practice; ms remains an accepted *input*
  converted exactly once at the boundary (Decision 2).
- **B — T-bar as a stream of per-sample commands on the bus.** Rejected: a fader emits
  O(100) samples/s; queued per-sample commands flood the bounded bus (squeezing real
  intents out — inv #10) and apply stale positions late. The absolute conflated slot
  (Decision 5) delivers the same operator experience with exactly one read per tick —
  the established route-coalescing/`is_conflated` pattern (`route.rs:31`,
  `event.rs:1306`).
- **C — Gamma-coded-domain mixing** (the de-facto perceptual midpoint of much broadcast
  practice). **Deferred to [ADR-C007](ADR-C007.md)**, which pins: the blend stays
  linear-light premultiplied `over` ([ADR-C003](ADR-C003.md), inv #8 — one blend domain,
  one code path) with an optional perceptual progress-curve mapping applied to `t` (not
  to pixels) approximating the gamma-domain midpoint. This ADR's engine is
  curve-agnostic by construction (Decision 1).
- **D — Wipes/transition graphics as overlay layers.** Rejected: the overlay bake runs
  off-thread on the egress consumer *after* the hot-loop tap publish
  (`crates/multiview-cli/src/pipeline.rs:2122`) — not frame-coordinated with on-clock
  transition state ([ADR-0054](ADR-0054.md) Decision 4), and overlays are
  input-decoupled by design (they may not sample sources). A wipe is per-tile compositor
  geometry evaluated in the same composite as the scenes it reveals (Decision 3's mask
  insertion point).
- **E — A generic keyframe/animation timeline engine** instead of typed transitions.
  Rejected for this layer: a free timeline cannot be admission-sized at arm, cannot be
  classified per inv #11, and re-opens the off-clock animation hazard
  ([ADR-0054](ADR-0054.md) Alternative C). The typed taxonomy is exhaustively
  `match`-able (absolute-typing pillar) and each variant carries a known cost model.
  Macro sequencing of discrete steps lives control-plane-side per
  [ADR-M012](ADR-M012.md)/[ADR-W021](ADR-W021.md).

## Consequences

- **Positive**
  - Transitions are exact, deterministic, and testable offline: the
    progress/arming/completion machine is a pure value machine over `Tick` — property
    tests and `ManualTimeSource` golden runs cover it without any engine running.
  - Wave 1 ships with zero compositor changes on the verified opacity substrate; the
    flat-list fast path makes simple-scene mixes nearly free (one merged placement
    list, no extra composite).
  - One parameter model covers the observed cross-implementation behaviour space
    (switch_point, trigger/mix, flip-flop, reverse), so presets in
    [ADR-M012](ADR-M012.md) round-trip without vendor-specific cases.
- **Negative / cost**
  - Scene pre-render (overlapping scenes) costs ~2 extra composites per tick for the
    transition window plus an NV12 re-quantization per intermediate generation —
    bounded by the transition duration and admission-gated at arm; GPU pass fusion
    (one readback) is the mitigation, and the CPU reference path needs a bench gate
    like the existing 40 ms composite gate.
  - Wave 2 is a genuine compositor contract change (per-tile mask + affine on both
    backends, wiring the dead fit/crop/rotation fields) — the single biggest work item
    this ADR creates, carried as its own backlog lane with CPU-oracle/GPU-SSIM gates.
  - Arm-time path selection (flat-list vs pre-render) and arm-time demotion (AUTO→CUT)
    introduce arm-result variability the UI must surface honestly
    ([ADR-W021](ADR-W021.md) plan/arm responses) — silent demotion is forbidden.
  - The conflated T-bar slot is a new engine-adjacent latest-wins structure; its
    chaos-gate obligation (a wedged writer never stalls the drain; the drain reads at
    most one value per tick) ships with it ([ADR-R011](ADR-R011.md)).
