# ADR-0005: Custom GPU-native compositor (CUDA / Metal / Vulkan·wgpu·libplacebo), not FFmpeg filters

- **Status:** Proposed
- **Area:** Core Engine
- **Date:** 2026-06-02
- **Source brief:** [core-engine.md](../research/core-engine.md)

## Decision

Implement compositing as a custom Compositor trait with CudaCompositor (fused kernels), MetalCompositor (compute), and a portable VulkanCompositor (ash or libplacebo, with wgpu/WGSL as the portable compute baseline). Do all fit/cover/crop/gaps/borders in-shader. Use FFmpeg/GStreamer compositing only as fallback references.

## Rationale

FFmpeg has no xstack_cuda (only chained two-input overlay_cuda); only Intel has true HW stack filters; no FFmpeg stack filter does per-cell fit/crop; base GStreamer compositor has only none/keep-aspect-ratio with no crop. A custom compositor is the only way to get cover/crop/asymmetric/PiP layouts with zero-copy on each vendor, and is the project's core value-add.

## Alternatives considered

FFmpeg overlay_cuda chains (rejected: vendor-uneven, extra latency, no fit/crop). GStreamer cudacompositor/glvideomixer (rejected: ties to GStreamer memory model/C ABI). libplacebo everywhere incl. native Metal (partly adopted as portable backend; on macOS it is MoltenVK-only, so native Metal is preferred for the fast path).

## Consequences

Must implement correct color management (matrix+range) and separable Lanczos in linear light; rounded corners/shadows are custom shader work. libplacebo dependency (LGPLv2.1) for the portable tier; wgpu HAL interop pinned and isolated behind the trait.
