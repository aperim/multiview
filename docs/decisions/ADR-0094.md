# ADR-0094: Subtitle layer enable/disable in the layout editor + per-output subtitle selection

- **Status:** Proposed
- **Area:** Web/API stack ↔ control ↔ overlay (operability; also a live defect)
- **Date:** 2026-06-13
- **Source brief:** [webui-operability-gaps.md §6](../research/webui-operability-gaps.md)
- **Extends:** [ADR-0019](ADR-0019.md) (native caption ingest + per-source selector
  `auto | off | teletext_page N | track id | embedded_cc | sidecar`), [ADR-R007](ADR-R007.md)
  (burn-in vs format-aware discrete passthrough), [ADR-W004](ADR-W004.md) (react-konva + dnd-kit
  layout editor)
- **Relates to:** [ADR-0093](ADR-0093.md) (the typed `<TrackSelect>` reused here),
  [decoupled-routing.md §5/§8](../research/decoupled-routing.md) (subtitle crosspoint re-point + class),
  [ADR-0059](ADR-0059.md) (the subtitle-seam precedent — `SubtitleRouteHandle` ArcSwap),
  [ADR-R010](ADR-R010.md) (Class-2 track-set change), [current-defects-2026-06-13.md §1.6](../development/current-defects-2026-06-13.md)

## Context

The layout editor offers no control to turn subtitles/captions on or off for a tile or the program. The
machinery is built — the per-source caption **selector** ([ADR-0019](ADR-0019.md)) and the ingest +
burn-in path (#47) — but there is **no UI binding** in the react-konva editor
([ADR-W004](ADR-W004.md)), and no per-output subtitle-track selection. This is both a feature gap and a
live defect ([current-defects §1.6](../development/current-defects-2026-06-13.md)).

## Decision

Bind the existing per-source selector to the layout editor and add per-output subtitle selection,
surfacing the burn-in-vs-passthrough capability — reusing the built selector, the typed `<TrackSelect>`
([ADR-0093](ADR-0093.md)), and the subtitle re-point seam.

1. **Per-tile subtitle control in the editor.** Each tile/cell gets a subtitle panel with an **on/off
   toggle** + a typed **track select** ([ADR-0093](ADR-0093.md)'s `<TrackSelect>` over that tile's
   source inventory). Because `CaptionSelector` is **source-global**, a per-tile toggle must **not**
   drive it directly — toggling one tile would otherwise disable caption ingest/render for every other
   tile sharing that source. Per-tile **render enablement** is therefore a **layout/subtitle-crosspoint**
   property (the tile's caption layer bound/unbound), and `CaptionSelector::Off` is reserved for
   **source-level ingest disable only**. The editor must **disambiguate** the two senses of "off"
   ([decoupled-routing.md](../research/decoupled-routing.md)): *don't ingest at all* (source selector
   `Off`, affecting all tiles on that source) vs *don't render this layer on this tile* (subtitle
   crosspoint unbound) — the panel labels which it does, and the per-tile toggle defaults to the
   layer-render sense.
2. **Per-output subtitle selection.** Where a transport can carry discrete subtitle tracks
   ([ADR-R007](ADR-R007.md): HLS/LL-HLS segmented WebVTT + in-band 608/708; MPEG-TS/SRT
   DVB-sub/teletext/608; RTMP = burn-in or single-608; NDI = burn-in), the output editor offers a typed
   per-output subtitle-track set with **capability-greyed** options and a clear **burn-in vs
   passthrough** badge, derived read-only from the [ADR-R007](ADR-R007.md) matrix — the subtitle
   analogue of [ADR-M004](ADR-M004.md)'s audio output-track ownership.
3. **Live-apply class, surfaced before apply.** A subtitle layer re-point onto an existing layer/track
   is **Class-1** (re-point the `Arc<CueStore>` the layer samples, clear-on-switch for the boundary
   frame — [decoupled-routing.md §5/§8](../research/decoupled-routing.md)); a subtitle **track-set**
   change that alters a pinned output's passthrough set is **Class-2** ([ADR-R010](ADR-R010.md)). The
   editor shows the class via `/plan` before applying. The per-source toggle rides the subtitle re-point
   seam (the [ADR-0059](ADR-0059.md) `SubtitleRouteHandle` ArcSwap precedent, drained off the hot path),
   so toggling captions is **live, not a restart**.
4. **Build-capability honesty.** Captions render only where the build includes the `overlay`
   (and `libass` for styled) feature; the deploy `nvidia` preset has historically omitted `overlay`. The
   subtitle control reflects the build capability via `LiveApplyCaps` / the capability report — a toggle
   over an absent feature is **greyed with a reason**, never a silent no-op (the
   [ADR-W022](ADR-W022.md) header-honesty discipline). Shipping the caption path in the deploy preset is
   a packaging prerequisite tracked alongside this work.

## Consequences

- **Positive.** Operators enable/disable subtitles and pick the subtitle source per tile and per output
  from the editor, see whether it is burned-in or passed through, and see the apply class before
  committing — captions become a first-class, live-operable layer.
- **Negative / cost.** New editor panels + an output subtitle-track set + generated types; the per-output
  passthrough surface depends on the [ADR-R007](ADR-R007.md) capability matrix being exposed read-only.
  The packaging caveat (deploy preset omits `overlay`) must be fixed or the toggle is honestly greyed.
- **Neutral.** Captions decode **only when shown** ([ADR-0019](ADR-0019.md)), so an editor toggle to
  `off` removes decode cost; cost scales with caption-bearing tiles, not sources.

## Alternatives considered

- **Config-file-only subtitle control (status quo).** Rejected: violates WebUI-manages-everything and
  forces download-edit-restart for a caption toggle.
- **A single global subtitles on/off.** Rejected: subtitles are per-source-selectable
  ([ADR-0019](ADR-0019.md)) and per-output-carriable ([ADR-R007](ADR-R007.md)); a global flag loses the
  per-tile/per-output expressiveness the model already has.
- **A restart-to-apply subtitle toggle.** Rejected: the subtitle re-point seam is Class-1 live
  ([ADR-0059](ADR-0059.md)/[decoupled-routing.md](../research/decoupled-routing.md)); a restart would be
  a needless interruption.

## How it honors the invariants

- **#1 (output clock).** A caption toggle/re-point rides the subtitle seam drained off the hot path; the
  compositor samples a bounded per-tile cue store per tick and holds/drops if late — never blocking on a
  caption source ([ADR-0019](ADR-0019.md)). A Class-2 track-set change goes through make-before-break
  ([ADR-R010](ADR-R010.md)), never a gap.
- **#10 (no back-pressure).** The selector write rides the bounded command bus / ArcSwap seam; the
  capability reads are cached; no new engine-inward channel. A stalled/absent caption source degrades to
  "no cue this frame", never back-pressuring the engine.
