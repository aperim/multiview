# ADR-0078: Switcher audio — mix-minus / IFB / talkback, with safety by construction

- **Status:** Proposed
- **Area:** Audio / Production switcher
- **Date:** 2026-06-13
- **Source brief:** [switcher-audio.md](../research/switcher-audio.md)
- **Extends:** [ADR-0077](ADR-0077.md) (input strips + buses + send matrix),
  [ADR-0059](ADR-0059.md) (per-bus audio = a `ProgramBus` per bus + `AudioReader` cursors)
- **Relates to:** [ADR-R005](ADR-R005.md) (program bus + per-output capability),
  [webrtc.md](../research/webrtc.md)/[ADR-0049](ADR-0049.md) (WebRTC returns),
  [sip-calling.md](../research/sip-calling.md)/[ADR-0085](ADR-0085.md) (SIP party returns),
  [aes67-delivery.md](../research/aes67-delivery.md)/[ADR-0033](ADR-0033.md) (AES67 return busses),
  [ADR-0034](ADR-0034.md) (crosspoints), [ADR-W021](ADR-W021.md) (REST), [ADR-RT008](ADR-RT008.md)
  (realtime), [ADR-T001](ADR-T001.md)/[ADR-RT004](ADR-RT004.md) (invariants #1/#10)

## Context

Remote production needs **N-1 / mix-minus** returns: the mix sent back to a participant must contain
what they need to hear **minus their own (delayed) audio**, or they hear themselves echoed.
[ADR-0059](ADR-0059.md) explicitly defers mix-minus ("post-MVP … with no design yet"). The bus
machinery now exists ([ADR-0077](ADR-0077.md): buses are `ProgramBus`es over shared stores via
`AudioReader` cursors), so mix-minus can be a **bus type**, not a pile of one-off routes. The same
machinery serves **IFB** (interruptible foldback) and **talkback** — and talkback is the single most
dangerous audio path in a live plant: a producer mic leaking to Program is a catastrophic, common
failure. Safety must be **structural**, not a checkbox.

## Decision

1. **Mix-minus is a first-class `AudioBusKind::MixMinus`** ([switcher-audio.md §9](../research/switcher-audio.md)):

   ```rust
   struct MixMinusBus { id, base_bus: AudioBusId, excluded_strips: Vec<AudioStripId>,
                        extra_sends: Vec<AudioSend>, talkback_sources: Vec<AudioStripId>,
                        duck_base_when_talkback: Option<DuckSettings> }
   ```

   Two construction modes: **explicit** (operator selects the return's sends — safest for broadcast)
   and **derived** (`return = base_bus − excluded_strips + extra_sends + talkback`, the default for
   remote-call workflows). A derived mix-minus is realized over the [ADR-0077](ADR-0077.md) send
   matrix as a bus whose strip sends are the base bus's minus the excluded strips — **not** a second
   decode (an `AudioReader` over each contributing store).
2. **Safety invariants (enforced at config-validation + apply, not advisory):**
   - A mix-minus bus **never feeds itself**; bus-to-bus routing has **cycle detection** (reject at
     422, [ADR-W015](ADR-W015.md) typed-validation precedent).
   - A remote source **never receives its own delayed signal** unless an explicit override is set —
     i.e. a participant's strip is auto-excluded from its own return by default.
   - **Talkback can never reach Program** unless a distinct, protected setting is set; talkback
     routing to Program is refused by the validator. The "talkback live" state is always visible.
   - Returns support per-destination delay / gain / limiter / mute.
3. **IFB & talkback.** Talkback sources are a strip class; momentary + latched talk; routed to
   selected IFB/mix-minus busses; optional `duck_base_when_talkback` dims the base mix on the target
   return while talk is active (a master-envelope duck on that bus, reusing the [ADR-0059](ADR-0059.md)
   §3 master ramp — Class-1, no new DSP).
4. **Return endpoints.** A return bus routes to a participant transport — WebRTC ([ADR-0049](ADR-0049.md)),
   SIP ([ADR-0085](ADR-0085.md)), AES67 ([ADR-0033](ADR-0033.md)), NDI, or SRT — via the
   [ADR-0077](ADR-0077.md) output-route model; default remote-guest template wires
   `guest_N → guest_N_return = pgm − guest_N + producer_talkback`. (NDI support stays opt-in/
   **runtime-loaded** — no SDK/spec redistribution, user supplies the licensed runtime; NDI® is a
   registered trademark of Vizrt NDI AB.)
5. **Invariant posture.** Each return bus is a consumer-side `ProgramBus`; the engine never awaits it;
   talkback/IFB control rides the [ADR-0059](ADR-0059.md) `AudioControlHandle` seam (drop-oldest, inv
   #1/#10).

## Rationale

- **A bus type, not hacks** — mix-minus expressed as `base − excluded + extras` over the existing
  send matrix is auditable, testable, and impossible to drift from the routing truth.
- **Safety is structural** — auto-exclude-self, cycle detection, and the talkback-to-Program refusal
  are validator rules, so the dangerous failures are *unrepresentable*, not merely discouraged.
- **Reuse** — ducking is the ADR-0059 master ramp; returns are ADR-0077 output routes; no new DSP.

## Alternatives considered

- **A — Mix-minus as a hand-built set of aux routes per guest.** Rejected: error-prone, no
  self-feed protection, no cycle detection — exactly the operator's "pile of one-off hacks" warning.
- **B — Talkback as just another send the operator wires.** Rejected: a single mis-click puts the
  producer mic on air; safety must be structural (self-exclusion + Program refusal), not operator
  discipline.
- **C — Per-return full re-decode.** Rejected per [ADR-0059](ADR-0059.md) §5 — `AudioReader` cursors
  give each return its own wait-free cursor over shared stores.

## Consequences

- **Positive.** Remote guests, IFB, foldback, and conferencing bridges get correct N-1 returns;
  catastrophic talkback/echo failures are designed out; closes the [ADR-0059](ADR-0059.md) mix-minus
  deferral.
- **Negative / cost.** Each return is a bus (a mix + encode, shed before Program by
  [ADR-E007](ADR-E007.md)); the validator gains cycle/self-feed/talkback rules; the UI must surface
  "talkback live" + return health.
- **Risks.** A dying excluded source still silence-fills (the return completes over silence, never
  stalls, [ADR-R011](ADR-R011.md)); chaos test: wedge a return consumer — Program + other returns
  keep ticking. Bus-graph cycle detection must run on every routing apply, not just at boot.
- **Deferred (named).** Multiple simultaneous guest returns + a dedicated IFB panel are post-MVP
  ([switcher-audio.md §20](../research/switcher-audio.md)); MVP ships **one** mix-minus template.
