# ADR-0004: Zero-copy islands per vendor; explicit copy at every cross-vendor/NDI/CPU boundary

- **Status:** Proposed
- **Area:** Core Engine
- **Date:** 2026-06-02
- **Source brief:** [core-engine.md](../research/core-engine.md)

## Decision

Treat each GPU vendor/device as an isolated zero-copy island (NVDEC->CUDA->NVENC; VideoToolbox->Metal->VideoToolbox; VAAPI/QSV->Vulkan via dma-buf) and budget exactly one explicit host/staged copy at any cross-vendor boundary, the CUDA->wgpu seam, and the NDI/CPU boundary.

## Rationale

Adversarial verification confirmed cross-vendor on-GPU zero-copy does not exist on desktop (foreign dma_buf into CUDA is Tegra-Thor-only; av_hwframe_map only aliases same-device contexts; wgpu has no stable external-memory import; NV12 unsupported on wgpu Metal; NDI is host-memory). Designing for cross-vendor zero-copy would break latency/density goals.

## Alternatives considered

Assume a universal portable zero-copy bus (rejected: does not exist). All-wgpu everywhere (rejected: copies on NVIDIA, NV12-on-Metal gap). Force single backend end-to-end always (rejected: loses best-per-stage flexibility).

## Consequences

On NVIDIA the compositor is CUDA-native (not wgpu) to keep the island intact. dma-buf paths require DRM-modifier negotiation + explicit sync and must be validated per driver with a copy fallback. Tests assert no silent host fallback within an island.
