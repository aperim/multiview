# Broadcast Multiviewer Capability Survey — Research Brief

> A standards-anchored, vendor-neutral survey of the established, near-universal capabilities of a professional broadcast multiviewer, mapped to Mosaic (a software, IP/streaming live mosaic generator). Every capability is expressed generically and anchored in an open standard. Products are not named, compared, or excluded; only open standards/protocols are referenced.

## Scope & framing

Mosaic already renders the mosaic, layout, overlays, audio (EBU R128), captions, colour, resilience state and the REST/WebSocket + preview control surface. The professional-multiviewer **delta** is the broadcast **monitoring/alarm engine + control plane** layered on top: content-aware fault probes with northbound notification, dynamic UMD/tally over an open label/tally protocol with external tally-bus integration, loudness logging + richer metering, salvo/scheduled automation, IP-facility discovery/connection control, and multi-head output.

Mosaic is software/IP. Baseband signal handling, genlock and physical display heads are reached only via an external IP gateway, so this brief targets the IP/streaming and protocol layers. The capabilities below are **table-stakes industry-wide and/or anchored in open standards** (SMPTE, EBU, ITU/ITU-T, AES, IEC, IETF/RFC, AMWA NMOS, SCTE, ETSI, and the openly-published control protocols TSL UMD, Ember+, SW-P-08).

## Taxonomy by category (with standards)

**1. Layouts, display modes & video walls.** Preset and free-form layouts (any size/aspect/position, overlap); instant preset recall/salvo; per-tile source crop/zoom (region-of-interest); round-robin cycling; auto-promote-on-fault; freeze/reference-still tiles; mixed-resolution tiles in one layout; output orientation (portrait/landscape) + per-tile rotation; **multi-head output** (independent per-head layout + resolution) and **video-wall spanning** with bezel compensation. *Standards:* AFD aspect handling **SMPTE ST 2016-1** (format), **ST 2016-3** (VANC mapping), carriage via **ETSI TS 101 154** / **ATSC A/53 Part 4**; safe-area graticules **SMPTE ST 2046-1** (action-safe 93 %, title-safe 90 % of the Production Aperture), alternate-aspect **RP 2046-2** (90/90), legacy **RP 218**, **ITU-R BT.1848**.

**2. UMD labels & tally.** Per-tile Under-Monitor Display text and tally (red = program, green = preview, amber = third state) driven *live* from an external switcher/router/automation. The open de-facto standard is the **TSL UMD protocol family**: v3.1 (RS-422/485 serial, also one-serial-packet-per-UDP-datagram; on/off tally + 2-bit brightness; fixed 16-char ASCII), v4.0 (appends CHKSUM/VBC/XDATA giving per-element LH/Text/RH colour tally 0 = off/1 = red/2 = green/3 = amber), v5.0 (native 16-bit IP/UDP, max 2048-byte packet, ASCII or UTF-16LE, screen + display addressing 0–0xFFFE with 0xFFFF broadcast, multiple variable-length displays per packet, DLE/STX framing only on TCP/byte-stream). The spec defines **no UDP port** (configurable; common conventions v5.0 = 40003, legacy = 40001). Multi-region/multi-level tally; a tally **arbiter** aggregating multiple buses; router/switcher **name-following** labels; **GPI/GPO** with the IP-native equivalent **AMWA NMOS IS-07** (event/tally over WebSocket/MQTT); **Ember+** and **SW-P-08** drivers; **NMOS IS-04** source/receiver labels.

**3. Audio metering & loudness.** Per-channel bargraphs (in/out of tile); selectable ballistics/scales — analog PPM Type I/IIa/IIb per **IEC 60268-10**, VU, digital sample-peak per **IEC TR 60268-18** (a Technical Report), true-peak dBTP per **ITU-R BS.1770**; phase-correlation + goniometer; multi-channel/surround grouping with Lo/Ro–Lt/Rt downmix metering per **ITU-R BS.775**. Loudness M (400 ms)/S (3 s)/I (gated) + LRA + max true-peak with selectable **EBU R128** (−23 LUFS; meters/LRA per **EBU Tech 3341/3342**) or **ATSC A/85** (−24 LKFS) target; **long-term loudness logging** with exportable compliance reports. Audio-follow-video monitor/PFL bus; channel shuffle/mapping (**NMOS IS-08**); embedded model **SMPTE ST 299-1 / AES3**, IP via **ST 2110-30 / AES67**.

**4. Fault/alarm & QC engine — the biggest delta.** Content-aware probes on decoded essence: loss-of-signal, **black**, **freeze** (both with detection-zone + dwell), audio silence/over/clip/phase/imbalance, loudness violation + dialnorm mismatch, caption presence-loss, format/standard change, AFD mismatch, colorimetry/HDR-transfer mismatch, no-reference QoE (**open** metrics **ITU-T P.1203/P.1204**, **ITU-T J.247/J.341**). For MPEG-TS inputs: **ETSI TR 101 290** Priority 1/2/3 (groupings verified against v1.3.1; Buffer/Empty_buffer/Data_delay are P3 in all editions). For RTP/IP: **Media Delivery Index (IETF RFC 4445)** + **SMPTE ST 2022-7** dual-path health. **SCTE 35 / SCTE 104** ad-marker monitoring. Alarm state machine: per-probe threshold + dwell-up/down + hysteresis; **ITU-T X.733** perceived severity (Critical/Major/Minor/Warning/Cleared) + probe→tile→group→system roll-up; latch/acknowledge; virtual/Boolean alarm grouping; auto-promote-on-fault; alarm logging (error-seconds, availability); northbound **SNMP traps** (IETF RFC 1157/3411 + MIB), **syslog (RFC 5424)**, email, webhook.

**5. Overlays, clocks, timers, metadata.** Multi-field UMD/OMD; NTP (**RFC 5905**) / **PTP (IEEE 1588 / SMPTE ST 2059-2)**-disciplined analog + digital clocks + multi-timezone; count-up/down/down-then-up timers (API/GPI-triggered, wrap colour); embedded source timecode display per **SMPTE ST 12-1/-2/-3 / RP 188 / ST 2110-40**; on-tile format/codec/bitrate/colorimetry readouts; AFD + safe-area markers; caption/teletext decode + presence (**CEA-608/708**, DVB subtitling **ETSI EN 300 743**, teletext **EN 300 706**, **OP-47 / SMPTE RDD 8**, WebVTT); SCTE cue indicators; flash-a-tile IDENTIFY; data-bound custom widgets.

**6. Inputs.** Native IP uncompressed **SMPTE ST 2110-20/-30/-40** (+ **-21** shaping, **-31** AES3); lightly-compressed **ST 2110-22** (codec-agnostic, JPEG XS common, **VSF TR-08**, **ISO/IEC 21122**); legacy **ST 2022-6**; **ST 2022-7** hitless dual-path; **PTP/ST 2059-2** timing; **NMOS IS-04/05/07/08**. Compressed MPEG-TS over UDP/RTP/SRT (**ISO/IEC 13818-1**, **RTP RFC 3550**, H.264/HEVC, full PSI/SI parsing, **VSF TR-01** J2K). OTT/contribution **SRT**, **RTMP**, **WebRTC (W3C/IETF)**, **HLS (RFC 8216)/LL-HLS**, **MPEG-DASH (ISO/IEC 23009-1)**, **RTSP**. ANC/VANC extraction (**ST 2110-40 / ST 2038 / ST 291**) for captions, AFD, timecode, SCTE. Per-input HDR/WCG (**ITU-R BT.2100** PQ/HLG, **BT.2020**, S-Log3) tone-mapped onto one wall (**ITU-R BT.2446**).

**7. Outputs, scaling, redundancy.** Multi-head **mosaic-as-a-stream** (the converged IP-multiviewer model — Mosaic's native strength), adding **ST 2110-20/-22** + **ST 2022-7** hitless + RIST/WebRTC; UHD canvas + per-tile HDR/SDR conversion; documented density (inputs × PiPs × heads) and a low-latency profile; HA via hot-standby/N+1, dual-NIC, heartbeat/failover.

**8. Control & operations.** Salvos (atomic multi-tile recall with arm/take); time/event/**SCTE-104/35**/**BXF (SMPTE 2021)**-scheduled layout automation; soft/hardware control panels + browser/mobile remote; **NMOS IS-12/MS-05** device control + the receiver-monitoring status model; **NMOS IS-10 (OAuth 2.0 RFC 6749 / JWT RFC 7519)** security; **Ember+**, **SW-P-08**, **** router control + route-follow; RBAC multi-user + audit log; config-as-code templates + scripting/webhook hooks.

## Mosaic: have vs. gap

**Already designed (enhance only):** layout presets / grid + absolute / fit / borders / z-order / transitions / hot-swap; overlays (label, clock, timecode, image/logo, tally_border, box, meter, alert_card, subtitle, lower_third); discrete + program audio + EBU R128; captions burn-in/passthrough; 4-axis colour + HDR/SDR tone-map + CICP tagging; per-tile LIVE/STALE/RECONNECTING/NO_SIGNAL state + slate cards; REST `/api/v1` + WebSocket + preview.

**Net-new gaps (the delta):** TSL UMD ingest/egress (v3.1/4.0/5.0) + dynamic UMD/multi-region tally; external tally-bus integration + arbiter + name-following + GPI/GPO (NMOS IS-07); the content-aware **monitoring/alarm engine** (black/freeze/silence/loudness-violation/caption-loss/format-change/AFD/TR 101 290/MDI/ST 2022-7) with X.733 severity, dwell/hysteresis, auto-promote, SNMP/syslog/email/webhook; **loudness logging** + ATSC A/85 profile + multi-standard ballistics + true-peak/phase/surround meters; **NMOS IS-04/05/07/08/IS-12/IS-10** + Ember+/SW-P-08 router control + salvos + scheduled automation; **multi-head/video-wall** output; **ST 2110/-22/2022-6/2022-7 + PTP** ingest/egress; AFD/safe-area overlays; SCTE-35/104 monitoring; count-up/down timers, analog/multi-timezone clocks; RBAC + audit + config-versioning enhancements.

## Verified protocol/spec notes (carry into implementation)

- **TSL UMD:** wire format field-verified against the primary spec. Spec defines **no port** — make it configurable. v5.0 text is variable-length (not fixed 16); 16-char ASCII applies to v3.1/v4.0 only. DLE/STX framing is **TCP/byte-stream only** (off on UDP). Displays-per-packet bounded by the 2048-byte ceiling, not a fixed 16. v4.0 and v5.0 are **not** wire-compatible (different CONTROL bit ordering). Tally colour palette 0/1/2/3 = off/red/green/amber for v4.0 and v5.0.
- **Safe areas:** cite **ST 2046-1** for the 93 %/90 % figures (of Production Aperture); RP 2046-2 is alternate-aspect protection (90/90); RP 218 is legacy.
- **AFD:** ST 2016-3 VANC packet DID = 0x41/SDID = 0x05; code meanings depend on 4:3 vs 16:9 coded frame. Primarily a read-for-fit/passthrough concern for IP/streaming.
- **TR 101 290:** P1/P2/P3 groupings verified against v1.3.1; Buffer/Empty_buffer/Data_delay are P3 in all editions.
- **Meter standards:** analog PPM = IEC 60268-10; digital sample-peak = **IEC TR 60268-18** (a Technical Report); **true-peak (dBTP) belongs to ITU-R BS.1770**, not 60268-18.
- **MDI severity bands (RFC 4445):** the metrics (Delay Factor, Media Loss Rate) are standardized; severity thresholds are implementation-defined and must be documented as such.
- **ST 2022-7:2019** ("Seamless Protection Switching of RTP Datagrams") operates at the RTP-datagram level, so it applies to any RTP essence including ST 2110-20/-30/-40.
- **PTP:** ST 2110-10 normatively *mandates* (shall support) the **ST 2059-2** profile of **IEEE 1588-2008**; it is not itself PTP.
- **NMOS IS-12/MS-05** receiver-monitoring model maps usefully onto per-tile health, but that mapping is an inference, not mandated by AMWA — document it as such.
