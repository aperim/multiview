- [SRT transport (input + output)](srt-transport.md) — IO/Transport (Secure Reliable Transport, the network transport — **not** subtitles: a tiered `SrtTransport` seam. Tier-0 keep the license-clean FFmpeg `srt://` caller baseline + conformance oracle and wire the **orphaned** typed `SrtConfig` (`multiview-input/src/srt.rs`) into the schema backward-compatibly, fixing the verified passphrase-`10..=80` + latency-ms→µs `×1000` bugs; Tier-1 own what FFmpeg structurally can't — link statistics→telemetry/`HealthWarning` + listener-mode stream-id multiplexing — over pure-Rust srt-tokio; TSBPD = input jitter buffer, never the output clock (inv #1/#2); all SRT→engine paths drop-oldest (inv #10); Tier-2 **defer** an owned `multiview-srt-sys` libsrt FFI leaf for bonding (ADR-0028 pattern, only-if-required); SRT-0..12 backlog; drives ADR-0039)
- [IP multicast transport (input + output)](multicast-transport.md) — IO/Transport (UDP-MPEG-TS + RTP/MP2T over IPv4/IPv6 multicast as a first-class TYPED transport — the media foundation SAP + AES67/ST2110 sit on: typed `SourceKind::Udp`/`Rtp` + `Output::Udp`/`Rtp` (group/port/SSM-sources/interface/ttl/buffer, mirroring `RtspOptions`, **not** a raw url), libav `udp`/`rtp` URL derived from typed config with SSM/interface/buffer defaults baked; wires the unreachable `PushProtocol::UdpTs` + new `::Rtp`; scoped to compressed TS-over-UDP/RTP (uncompressed ST 2110-20 out of scope, reuse only the st2110 RTP/2022-7/bounded-channel prior art); receive = input jitter buffer never a pacer (inv #1/#2), bounded drop-oldest both ways (inv #10). Plus the **bulletproof container-without-host-networking HealthWarning** (silent-bridge-multicast trap, ADR-0035 sense→detect→warn: conjunctive startup heuristic + authoritative runtime joined-but-no-traffic confirmation, never auto-fails, no false positives); MC-0..12 backlog; drives ADR-0040)
- [SAP discovery + announcement](sap-discovery.md) — IO/Transport (Session Announcement Protocol, RFC 2974 — VLC "Network Streams (SAP)", **not** subtitles — sitting on the multicast transport: SAP is discovery/announce only, the media rides multicast. Build **one** essence-agnostic SAP packet engine + **one** RFC 8866 SDP model (video/audio/MP2T) and collapse the planned AES67 SAP/SDP (AES67-5/-6) into thin profiles — supersedes ADR-0033 §6/§9, no second SAP; RECEIVE → bounded **untrusted** inventory the operator confirm-binds to a Source (Class-1, never auto-ingest); ANNOUNCE → tokio-timer emits SDP for our multicast outputs so VLC/Dante discover them; verified wire details (3-bit version, auth_len×4, ±1/3 jitter, max(10×period,1h) purge, reject E/inbound-T=1, full group set incl. de-facto 239.255.255.255); listener/announcer/publish off the data plane, bounded drop-oldest (inv #1/#10); SAP-0..10 backlog; drives ADR-0041)
- [IPv6-first networking](ipv6-first.md) — Networking/Cross-cutting (**all network-facing surfaces are IPv6-first**; IPv4 is legacy-only + on a deprecation path — the operator directive, previously undocumented. Concretely: bind **dual-stack `[::]`** not `0.0.0.0`, loopback `[::1]`; bracket IPv6 URL literals; SDP `c=IN IP6` first-class (**no TTL** — scope in the address, RFC 8866 §5.7); IPv6 multicast `ff00::/8`+SSM `FF3x::/32` via MLDv2 primary, IPv4 `239/8`/`232/8`+IGMPv3 legacy; examples lead IPv6. Audit found the IPv4-only sites — is05 SDP parser mangles `IN IP6`, WHEP answer hardcodes `IN IP4`, telemetry `0.0.0.0` binds, st2110 v4-only join, IPv4-only examples — captured as the IPV6-0..9 remediation backlog. Dual-stack defaults keep legacy IPv4 working through the deprecation window; no data-plane invariant moves; drives ADR-0042, canonical in conventions §10)
