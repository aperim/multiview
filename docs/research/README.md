- [SAP discovery + announcement](sap-discovery.md) — IO/Transport (Session Announcement Protocol, RFC 2974 — VLC "Network Streams (SAP)", **not** subtitles — sitting on the multicast transport: SAP is discovery/announce only, the media rides multicast. Build **one** essence-agnostic SAP packet engine + **one** RFC 8866 SDP model (video/audio/MP2T) and collapse the planned AES67 SAP/SDP (AES67-5/-6) into thin profiles — supersedes ADR-0033 §6/§9, no second SAP; RECEIVE → bounded **untrusted** inventory the operator confirm-binds to a Source (Class-1, never auto-ingest); ANNOUNCE → tokio-timer emits SDP for our multicast outputs so VLC/Dante discover them; verified wire details (3-bit version, auth_len×4, ±1/3 jitter, max(10×period,1h) purge, reject E/inbound-T=1, full group set incl. de-facto 239.255.255.255); listener/announcer/publish off the data plane, bounded drop-oldest (inv #1/#10); SAP-0..10 backlog; drives ADR-0041)
- [IPv6-first networking](ipv6-first.md) — Networking/Cross-cutting (**all network-facing surfaces are IPv6-first**; IPv4 is legacy-only + on a deprecation path — the operator directive, previously undocumented. Concretely: bind **dual-stack `[::]`** not `0.0.0.0`, loopback `[::1]`; bracket IPv6 URL literals; SDP `c=IN IP6` first-class (**no TTL** — scope in the address, RFC 8866 §5.7); IPv6 multicast `ff00::/8`+SSM `FF3x::/32` via MLDv2 primary, IPv4 `239/8`/`232/8`+IGMPv3 legacy; examples lead IPv6. Audit found the IPv4-only sites — is05 SDP parser mangles `IN IP6`, WHEP answer hardcodes `IN IP4`, telemetry `0.0.0.0` binds, st2110 v4-only join, IPv4-only examples — captured as the IPV6-0..9 remediation backlog. Dual-stack defaults keep legacy IPv4 working through the deprecation window; no data-plane invariant moves; drives ADR-0042, canonical in conventions §10)
