# IP multicast transport (UDP-TS + RTP) — implementation backlog

Dependency-ordered, PR-sized slices. Reflects only what survived adversarial review.
Drives [multicast-transport](../research/multicast-transport.md) / [ADR-0040](../decisions/ADR-0040.md).
The media-path foundation the **SAP** discovery layer ([SAP backlog](sap-discovery-backlog.md),
ADR-0041) depends on. Sizes: `S` ≤½ day, `M` ~1–2 days, `L` ~3–5 days, `XL` multi-week.
TDD-first: failing test committed separately, then green without touching the test.

> Everything here rides the **existing `ffmpeg` feature** (libav `udp`/`rtp` are already
> LGPL-clean per ADR-0031) — **no new feature flag, no GPL escalation**. The address model,
> URL derivation, and the container/host-net detector are **pure Rust** in the default GPU-free
> build.

## Sequencing notes

- **Critical path:** MC-0 → MC-1 → MC-2 → {MC-3 ingest, MC-4 egress} → MC-7 (runtime warning) →
  MC-9/MC-10 (gates). The warning's authoritative half (MC-7) depends on real ingest (MC-3); its
  advisory half (MC-5/MC-6) can land in parallel after the schema (MC-1).
- **Parallelizable:** MC-5 (pure detector) needs only MC-1; MC-8 (telemetry) and MC-11 (2022-7
  reuse) follow MC-3 independently; MC-12 (docs) follows MC-6.
- **The two media gaps the SAP layer depends on:** SAP **bind-to-Source** consumes **MC-3**
  (raw-UDP-TS ingest); SAP **announce** consumes **MC-4** (`Output::Udp` egress — there is no point
  announcing a stream Multiview cannot emit). The SAP backlog declares those cross-deps.
- **Out of scope this cycle (named, not stubbed):** uncompressed ST 2110-20; a manual `socket2`
  SSM `IP_ADD_SOURCE_MEMBERSHIP` path (libav `sources=` suffices for single-path SSM); full IPv6
  SSM fidelity beyond libav delegation; `SO_RCVBUFFORCE` (behind a `CAP_NET_ADMIN` opt-in);
  dual-path raw-UDP-TS redundancy (no per-packet seq).
- **Gating discipline:** every multicast→engine path (receive socket, egress sink) must pass the
  MC-9 chaos gate; the warning must never auto-fail or auto-disable a source (inv #1).
