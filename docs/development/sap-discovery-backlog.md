# SAP (Session Announcement Protocol) discovery + announcement — implementation backlog

Dependency-ordered, PR-sized slices. Reflects only what survived adversarial review.
Drives [sap-discovery](../research/sap-discovery.md) / [ADR-0041](../decisions/ADR-0041.md).
**SAP = Session Announcement Protocol (VLC "Network Streams (SAP)"), not subtitles.**
Sits on top of the IP-multicast transport ([multicast backlog](multicast-transport-backlog.md),
ADR-0040): the **media in/out** is owned there — SAP **bind** consumes **MC-3** (UDP-TS ingest) and
SAP **announce** consumes **MC-4** (`Output::Udp` egress). Sizes: `S` ≤½ day, `M` ~1–2 days,
`L` ~3–5 days. TDD-first: failing test committed separately, then green without touching the test.

> Everything here is pure-Rust big-endian byte work + a thin SDP serializer + supervised tokio
> tasks, off-by-default behind the **existing `st2110` feature** (`= [tokio/net]`, no FFI,
> `unsafe_code = forbid`, LGPL-clean). It lives strictly on the control/discovery plane.

## Sequencing notes

- **Critical path:** SAP-1/SAP-2 (codec + SDP, parallel, no deps) → SAP-3 → SAP-4 → SAP-5 → SAP-6
  (bind, needs **MC-3**) ; and SAP-8 (announce, needs **MC-4**) → SAP-9 → SAP-10. SAP-0 (the shared
  `channel_bridge` fix) precedes SAP-4's reuse.
- **Cross-backlog dependencies (honest):** SAP-6 **bind** cannot fully ship until **MC-3** (typed
  UDP-TS ingest) lands — though raw-UDP-TS binds via the existing `Ts{url}` today; SAP-8 **announce**
  cannot ship until **MC-4** (`Output::Udp` egress) lands (no stream to announce — the no-partial
  rule). The multicast backlog is the prerequisite for the end-to-end SAP loop.
- **Parallelizable:** SAP-1 and SAP-2 have no deps and unblock everything; SAP-7 (AES67 convergence)
  needs SAP-1/2/4 and is independent of the announce/bind path.
- **AES67 convergence timing:** land the `sap/`+`sdp/` extraction (SAP-1/SAP-2/SAP-4) **before or
  jointly with** AES67-5/-6 so SAP-7 stays a one-line re-point of ADR-0033 §6/§9 + the AES67 backlog
  rows rather than a refactor.
- **Out of scope this cycle (named, not stubbed):** SAP authentication (PGP/CMS — optional,
  unimplemented in practice) and encrypted (`E`) SAP (no SAPv2 algorithm; reject on receive); RTP-framed
  multicast **ingest** binding (the ADR-0040 gap — surfaced "unbindable"); auto-bridging SAP into the
  NMOS registry (kept separate by trust).
- **Gating discipline:** every SAP→engine path (listener, inventory publish, announce) must pass the
  SAP-9 chaos gate; discovered sessions are never auto-ingested.
