## Critical path & parallelism

Lanes per the design pass; **Lane A is SERIAL with the repo's LANE-CORE
`pipeline.rs` territory** — SW-1 (desugar consumer), SW-6/SW-7 (tap publication), SW-12
(player transport), and SW-13 (audio seam) all touch
`crates/multiview-cli/src/pipeline.rs` and are sequenced through the LANE-CORE owner,
never run in parallel with each other or with other LANE-CORE work.

- **Lane A — engine switcher core (the long pole, serial):**
  SW-0 → SW-3 → SW-4 → SW-5 → SW-6 → SW-7, with SW-1 early (config-side, one
  `pipeline.rs` touch). This is the MVP spine; everything else converges on it.
- **Lane B — compositor primitives (parallel from day one):** SW-8 → SW-9 → SW-10;
  post-MVP SW-24 → SW-25. SW-8 has no dependencies and should start immediately.
- **Lane C — media (parallel after SW-2):** SW-11 → SW-12 → (SW-26 jointly with Lane B).
- **Lane D — audio (parallel, independent start):** SW-13 → SW-14 (joins SW-4/SW-5 for
  the AFV/FTB windows).
- **Lane E — tally & routing (parallel, independent start):** SW-15 ∥ SW-16 (SW-16 needs
  SW-4/SW-6); SW-27 post-MVP.
- **Lane F — control plane + SPA:** SW-2 early (unblocks Lanes C/F); SW-17 → SW-18 →
  SW-20; SW-19 is parallel-early (pure SPA refactor, no backend dependency);
  SW-21/SW-22 after SW-17.
- **Convergence:** SW-20 (the operator panel) is the MVP integration point — it needs
  SW-17/SW-18 (surface), SW-19 (service), SW-6 (monitors), SW-14 (meters), SW-16 (tally).
  SW-23 closes the MVP wave: no MVP declaration without its gates green.
- **MVP wave = SW-0..SW-23** (one M/E designed for N — matches the briefs' phasing,
  [production-switcher.md §18](../research/production-switcher.md)). **Post-MVP waves: SW-24

## Hard acceptance gates (from adversarial review — non-negotiable)

1. **No transition without the `Tick`** — progress is `f(tick.index)` through the widened
   hook (SW-3/SW-4); a `ManualTimeSource` replay must be byte-identical; any wall-clock
   read inside switcher state is a defect, not a style issue.
2. **No flat-list mix on internally-overlapping scenes** — SW-5 must detect overlap at arm
   time and pre-render; the CPU oracle proves the through-blend defect otherwise.
3. **Never shed mid-transition** — complete-then-degrade; AUTO→CUT demotion happens at
   arm/admission time only; the PVW rung sheds before any program tile (SW-5/SW-6).
4. **No take without WARM-ON-ARM** — SW-7 prime-gates every PVW-driven take; the take
   frame is never a slate.
5. **T-bar is one conflated latest-wins slot** — never one command per position sample;
   per-tick drain work stays capped under a position storm (SW-17).
6. **DSK/FTB are on-clock** — a keyer or FTB toggle lands on the exact commanded frame
   boundary; the off-thread bake stays monitoring-only (SW-10).
7. **Alpha is NV12+A, premultiplied exactly once in linear light** — no per-tile RGBA
   path anywhere (inv #5/#8); both kernels pass the CPU-oracle/GPU-SSIM alpha gates
   (SW-8/SW-9).
8. **Audio is per-sample and exact** — AFV/FTB envelopes are per-sample `GainRamp`s with
   `SampleClock` integer budgets (never a per-tick scalar); the video completion tick
   equals the audio ramp end sample (SW-13/SW-14).
9. **No uncorrelated switcher 202s** — every switcher/media command carries a `CorrKey`
   variant and its outcome event echoes `corr` (SW-12/SW-17); the as-built `Route*`
   no-correlation gap is not repeated.
10. **TSL correctness = golden on-wire vectors transcribed from the published spec** —
    round-trip tests share one value model and are inadmissible as sole evidence (SW-15).
11. **Tally walks the render plan** — a source visible only as an on-air keyer's key
    input must PGM-tally; the untallied-overlay anti-pattern has a named held-out test
    (SW-16).
12. **Spec lockstep** — any API/event change updates utoipa + the `rest_routes` table +
    AsyncAPI + the regenerated SPA types in the same PR (SW-17/SW-18); drift fails CI.
