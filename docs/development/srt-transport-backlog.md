# SRT (Secure Reliable Transport) input + output — implementation backlog

Dependency-ordered, PR-sized slices. Reflects only what survived adversarial review.
Drives [srt-transport](../research/srt-transport.md) / [ADR-0039](../decisions/ADR-0039.md).
**SRT = the network transport (`libsrt`), not SubRip subtitles.**
Sizes: `S` ≤½ day, `M` ~1–2 days, `L` ~3–5 days, `XL` multi-week (soak/interop-gated).
TDD-first: failing test committed separately, then green without touching the test.

> **Tier map.** SRT-0..SRT-1 are pure (default build). SRT-2..SRT-3 wire the Tier-0 FFmpeg
> `srt://` byte path (existing `ffmpeg` feature, license-clean). SRT-4..SRT-10 build the Tier-1
> pure-Rust `srt-tokio` seam (off-by-default `srt` feature). SRT-11..SRT-12 are the deferred
> Tier-2 owned-FFI bonding work (`srt-libsrt` feature, only-if-required). Nothing here escalates
> the default build's license (libsrt is MPL-2.0, already on the `deny.toml` allow-list).

## Sequencing notes

- **Critical path:** SRT-0 → SRT-2 → SRT-3 (Tier-0 ships first, end-to-end, license-clean) →
  SRT-4 → SRT-5 → {SRT-6, SRT-7} → SRT-8 → SRT-9 → SRT-10. The product wins (typed config +
  observability + listener aggregation) all land by SRT-10 with **no license escalation and no
  new FFI**.
- **Parallelizable:** SRT-1 (stream-id grammar) needs only SRT-0 and unblocks SRT-9; SRT-6
  (encryption) and SRT-7 (stats) both follow SRT-5 and are independent of each other; SRT-8
  follows SRT-7; SRT-3 (Tier-0 loopback) runs alongside the SRT-4/5 seam work.
- **Long-pole / deferred:** SRT-11 (FFI + bonding) is the only `XL` and is **trigger-gated** —
  it ships **nothing** until a real requirement (bonding redundancy, HSv4 legacy interop, or
  proven-scale throughput) appears. Tier-0/1 cover all non-bonding SRT in the meantime.
- **Backward-compat join with ADR-0034:** SRT-0's "purely additive, no version bump" claim and
  ADR-0034's v2→v3 routing-table migration must be checked **together** — SRT-0/SRT-9 jointly
  verify the combined config golden (a completeness gap flagged in the brief).
- **Out of scope this cycle:** RIST (VSF TR-06; reopened only on a customer ST 2022-7-hardware
  requirement), FEC (`SRTO_PACKETFILTER`; validate-reject until implemented), AES-GCM outside
  Tier-2, balancing bonding (removed from 1.5.0, under development).
- **Gating discipline:** every engine→outside SRT channel (egress sink, stats poller, listener
  accept) must pass the SRT-10 chaos gate; Tier-1 srt-tokio is never production-trusted without
  the self-hosted interop run.
